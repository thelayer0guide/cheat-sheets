Grundlagen & Syntax
Einfacher Scan (Domain oder IP)
nikto -h TARGET
HTTPS-Ziel
nikto -h https://TARGET
Anderer Port
nikto -h TARGET -p 8080
Mehrere Ports gleichzeitig
nikto -h TARGET -p 80,443,8080
IP-Range (CIDR)
nikto -h 192.168.1.0/24
Mehrere Hosts aus Datei
nikto -h hosts.txt
SSL erzwingen / deaktivieren
nikto -h TARGET -ssl # SSL erzwingen nikto -h TARGET -nossl # SSL deaktivieren
Hinweis: nikto generiert erheblichen Traffic — ausschließlich auf Systemen einsetzen, für die eine schriftliche Genehmigung vorliegt.
🎛️
Tuning — Tests anpassen
Tuning-Kategorien
# -Tuning WERT 1 Interessante Dateien / Logs 2 Fehlkonfigurationen 3 Information Disclosure 4 Injection (XSS / Script) 5 Remote File Retrieval 6 DoS-Tests (gefährlich!) 7 Remote Source Inclusions 9 SQL Injection 0 Dateibasierte Tests
Kombinierter Scan (ohne DoS)
nikto -h TARGET -Tuning x13 # x = alle außer 6 (DoS)
Nur SQL-Injection-Tests
nikto -h TARGET -Tuning 9
Nur Information Disclosure
nikto -h TARGET -Tuning 3
x-Präfix: -Tuning x6 aktiviert alle Tests außer Kategorie 6 (DoS). Mehrere Kategorien ohne Trennzeichen kombinieren: -Tuning 1379
🔑
Authentifizierung & Session
HTTP Basic Auth
nikto -h TARGET -id user:pass
Default-Credentials testen
nikto -h TARGET -id admin:admin
Via Proxy (z. B. Burp Suite)
nikto -h TARGET \ -useproxy http://127.0.0.1:8080
Custom Cookie
nikto -h TARGET \ -Cookies "session=abc123"
Custom User-Agent
nikto -h TARGET \ -useragent "Mozilla/5.0 (compatible)"
Redirects folgen
nikto -h TARGET -maxredirects 5
Tipp: Burp als Proxy ermöglicht die Analyse aller nikto-Anfragen im Proxy-History-Tab für manuelle Nachuntersuchung.
🥷
Evasion-Techniken
Evasion-Methoden (-evasion N)
1 Zufällige URL-Enkodierung 2 Verzeichnis-Selbstreferenz (/./) 3 Vorzeitige URL-Abkürzung 4 Prepend langen Zufalls-String 5 Fake-Parameter anhängen 6 TAB statt Leerzeichen 7 URL großschreiben 8 Windows-Pfad-Delimiter (\)
Mehrere Evasions kombinieren
nikto -h TARGET -evasion 1 nikto -h TARGET -evasion 12 # 1+2
Evasion + Proxy (für WAF-Tests)
nikto -h TARGET \ -evasion 1 \ -useproxy http://127.0.0.1:8080
Hinweis: Evasions helfen beim Umgehen einfacher IDS/WAF-Regeln. Moderne WAFs erkennen nikto trotzdem anhand des Scan-Musters und User-Agents.

Output-Formate
Textdatei
nikto -h TARGET -o report.txt
HTML-Bericht
nikto -h TARGET \ -o report.html -Format html
CSV (Excel-Import)
nikto -h TARGET \ -o report.csv -Format csv
XML (Metasploit-Import)
nikto -h TARGET \ -o report.xml -Format xml
JSON
nikto -h TARGET \ -o report.json -Format json
Verbosity & Debug
nikto -h TARGET -Display V # alle Requests nikto -h TARGET -Display D # Debug nikto -h TARGET -nointeractive # kein Prompt
Format-Kurzform: nikto erkennt das Format auch am Dateinamen-Suffix — -o bericht.html ohne -Format erzeugt ebenfalls einen HTML-Report.
🧩
Plugins & Datenbank
Verfügbare Plugins auflisten
nikto -list-plugins
Bestimmte Plugins aktivieren
nikto -h TARGET \ -Plugins "headers;cookies"
Datenbank aktualisieren
nikto -update
DB-Integrität prüfen
nikto -dbcheck
Datenbank-Pfade
# Standard-Test-Datenbank: /usr/share/nikto/databases/db_tests # Eigene Tests hinzufügen: /usr/share/nikto/databases/db_user_defined
db_user_defined: Eigene Testeinträge im Format OSVDB-ID,"URI","Beschreibung" — nikto lädt diese Datei automatisch beim Start.
🔒
Zertifikat & SSL/TLS
SSL/TLS erzwingen
nikto -h TARGET -ssl
SSL-Warnungen auto-bestätigen
nikto -h TARGET -ssl -nointeractive
Verbindungs-Timeout setzen
nikto -h TARGET -timeout 30
Was nikto bei SSL prüft
# Automatisch geprüft: - Schwache Cipher (RC4, DES, NULL) - Veraltete Protokolle (SSLv2, SSLv3) - Abgelaufene Zertifikate (expired) - Selbst-signierte Zertifikate - Wildcard-Zertifikate - Unsichere Renegotiation
Tipp: Für detaillierte TLS-Analyse testssl.sh oder sslscan ergänzend einsetzen — nikto deckt nur die häufigsten SSL-Schwachstellen ab.
⚡
Quick Reference
Standard-Scan mit HTML-Report
nikto -h https://TARGET \ -o report.html -Format html
Via Burp Suite
nikto -h TARGET \ -useproxy http://127.0.0.1:8080
Ohne DoS-Tests
nikto -h TARGET -Tuning x6
Mehrere Ports scannen
nikto -h TARGET -p 80,443,8080,8443
Banner-Grabbing vorab (curl)
curl -k -I https://TARGET # Server-Header, X-Powered-By, etc.
Vollständiger Scan mit Auth + Output
nikto -h https://TARGET \ -id admin:admin \ -Tuning x6 \ -o full-report.xml -Format xml
Vorsicht: nikto ist kein Stealth-Tool — Scans erscheinen in Server-Logs. Immer nur auf autorisierten Zielen einsetzen (CTF, eigene Infrastruktur, Bug-Bounty-Scope).
