OSD-Flags — vollständig
Flags setzen / löschen / anzeigen
ceph osd set FLAG ceph osd unset FLAG ceph osd dump | grep flags ceph osd dump | grep -E "^flags"
Alle verfügbaren Flags
Flag Bedeutung noout OSDs die down gehen NICHT als out markieren noin OSDs nicht automatisch als in markieren noup OSDs nicht als up markieren nodown OSDs nicht als down markieren nobackfill Kein Backfill nach Recovery norebalance Kein Rebalancing bei CRUSH-Änderungen norecover Keine Recovery (gefährlich!) noscrub Kein normales Scrubbing nodeep-scrubKein Deep-Scrubbing pause Alle Client-I/Os pausieren full Cluster als voll markieren (blockt Writes) purged_snapdirsInterne Migration (nicht manuell setzen)
Wartungs-Workflow (Node-Neustart)
# VOR Wartung: ceph osd set noout ceph osd set norebalance # Optional für ruhigere Wartung: ceph osd set noscrub ceph osd set nodeep-scrub # NACH Wartung (Reihenfolge wichtig): ceph osd unset norebalance ceph osd unset noout ceph osd unset noscrub ceph osd unset nodeep-scrub # Danach: ceph -w und warten bis HEALTH_OK
Per-OSD flags (individuelle OSDs)
# Einzelnen OSD aus Rebalancing nehmen: ceph osd add-noout osd.3 ceph osd rm-noout osd.3 ceph osd add-noin osd.3 ceph osd rm-noin osd.3 ceph osd add-noup osd.3 ceph osd rm-noup osd.3 ceph osd add-nodown osd.3 ceph osd rm-nodown osd.3
⚠️ pause pausiert ALLE Client-I/Os — nur im Notfall verwenden!

OSD Management
Status & Überblick
ceph osd tree # CRUSH-Baum + Status ceph osd df tree # Belegung pro OSD ceph osd dump # Vollständiger OSD-Dump ceph osd ls # Liste aller OSD-IDs ceph osd find osd.5 # Wo liegt OSD 5? ceph osd metadata osd.5# Gerätedaten
OSD-Zustand ändern
ceph osd down osd.0 # OSD als down markieren ceph osd up osd.0 # OSD als up markieren ceph osd out osd.0 # Aus Cluster herausnehmen (löst Rebalance aus) ceph osd in osd.0 # Zurück in Cluster aufnehmen
OSD sicher entfernen (Workflow)
# 1. Prüfen ob sicher: ceph osd safe-to-destroy osd.5 # 2. OSD herausnehmen (löst Rebalance aus): ceph osd out osd.5 # 3. Warten bis HEALTH_OK, dann: ceph osd down osd.5 # 4. Daemon stoppen (auf dem Host): systemctl stop ceph-osd@5 # 5. OSD aus Cluster entfernen: ceph osd destroy osd.5 --yes-i-really-mean-it # 6. Auth + CRUSH bereinigen: ceph osd purge osd.5 --yes-i-really-mean-it
Scrubbing steuern
ceph osd scrub osd.0 ceph osd deep-scrub osd.0 ceph pg scrub 1.4a ceph pg deep-scrub 1.4a # Scrub-Zeiten einschränken (Config): ceph config set osd osd_scrub_begin_hour 22 ceph config set osd osd_scrub_end_hour 6
Performance & Throttling
ceph tell osd.0 bench ceph tell osd.* bench ceph daemon osd.0 perf dump # Recovery drosseln: ceph config set osd osd_recovery_max_active 2 ceph config set osd osd_max_backfills 1 ceph config set osd osd_recovery_sleep 0.1

ceph osd blocklist add 192.168.1.10:0/1234567890 ceph osd blocklist rm 192.168.1.10:0/1234567890 ceph osd blocklist ls # Alle Einträge löschen: ceph osd blocklist clear

ceph tell osd.* config set \ osd_recovery_max_active 3 ceph tell osd.0 config get \ osd_max_backfills ceph config show osd.0 # Aktive Config

ceph-volume — OSD Deployment
Verfügbare Geräte anzeigen
ceph-volume inventory ceph-volume inventory /dev/sdb ceph-volume lvm list # Existierende LVM-OSDs
OSD erstellen (LVM — empfohlen)
# Ganze Disk (auto-LVM): ceph-volume lvm create \ --bluestore \ --data /dev/sdb # Mit separatem WAL + DB (NVMe): ceph-volume lvm create \ --bluestore \ --data /dev/sdb \ --block.wal /dev/nvme0n1p1 \ --block.db /dev/nvme0n1p2
Prepare + Activate (getrennt)
# Schritt 1 — OSD vorbereiten: ceph-volume lvm prepare \ --bluestore --data /dev/sdb # Schritt 2 — OSD aktivieren: ceph-volume lvm activate \ OSD_ID OSD_FSID # Alle aktivieren (nach Reboot): ceph-volume lvm activate --all
OSD-Disk ersetzen (Replace-Workflow)
# 1. OSD herausnehmen + Daten migrieren: ceph osd out osd.7 # 2. Warten auf HEALTH_OK # 3. Daemon stoppen + OSD entfernen: systemctl stop ceph-osd@7 ceph osd purge osd.7 --yes-i-really-mean-it # 4. LVM-Volumes aufräumen: ceph-volume lvm zap /dev/sdb --destroy # 5. Neue Disk deployen: ceph-volume lvm create --bluestore \ --data /dev/sdb
Simple (RAW-Mode, kein LVM)
ceph-volume raw prepare \ --bluestore --data /dev/sdb ceph-volume raw activate \ --device /dev/sdb ceph-volume raw list

CRUSH Map — vollständig
CRUSH Map export / import / edit
# Binary-Map exportieren: ceph osd getcrushmap -o crushmap.bin # In Text decompilieren: crushtool -d crushmap.bin -o crushmap.txt # Text → Binary kompilieren: crushtool -c crushmap.txt -o crushmap-new.bin # Map zurückspielen (Achtung!): ceph osd setcrushmap -i crushmap-new.bin
CRUSH-Baum anzeigen
ceph osd crush dump # JSON-Dump ceph osd tree # Baumansicht ceph osd tree --format json ceph osd crush tree --show-shadow # CRUSH-Regeln anzeigen: ceph osd crush rule ls ceph osd crush rule dump ceph osd crush rule dump replicated_rule
Buckets hinzufügen / verschieben
# Neuen Host-Bucket erstellen: ceph osd crush add-bucket node04 host # Rack erstellen: ceph osd crush add-bucket rack01 rack # Datacenter: ceph osd crush add-bucket dc1 datacenter # Host unter Rack verschieben: ceph osd crush move node04 rack=rack01 # Rack unter Datacenter: ceph osd crush move rack01 \ datacenter=dc1 root=default
OSD zu CRUSH hinzufügen / entfernen
# OSD zu Host hinzufügen: ceph osd crush add osd.5 1.0 host=node04 # OSD-Gewicht anpassen (TB-Wert): ceph osd crush reweight osd.5 1.8 # Alle OSD-Gewichte neu berechnen: ceph osd crush reweight-all # OSD aus CRUSH entfernen: ceph osd crush rm osd.5 # Bucket entfernen (erst leeren): ceph osd crush rm node04
CRUSH-Regeln erstellen
# Replicated-Regel auf Host-Ebene: ceph osd crush rule create-replicated \ myrule default host # Auf Rack-Ebene (Rack Awareness): ceph osd crush rule create-replicated \ rack_rule default rack # Erasure-Coding-Regel: ceph osd crush rule create-erasure \ ec_rule myec_profile # Regel löschen: ceph osd crush rule rm myrule # Regel einem Pool zuweisen: ceph osd pool set mypool crush_rule rack_rule
Device Classes (NVMe, SSD, HDD)
# Klassen anzeigen: ceph osd crush class ls # Geräte pro Klasse: ceph osd crush class ls-osd ssd ceph osd crush class ls-osd hdd ceph osd crush class ls-osd nvme # Klasse manuell setzen: ceph osd crush set-device-class ssd osd.0 osd.1 ceph osd crush set-device-class hdd osd.2 # Klasse entfernen: ceph osd crush rm-device-class osd.0 # Neue Klasse erstellen: ceph osd crush class create myclass ceph osd crush class rm myclass
Klassen-basierte Regeln (z.B. nur SSD)
# SSD-Pool mit SSD-Regel: ceph osd crush rule create-replicated \ ssd_rule default host ssd ceph osd pool create fast-pool ceph osd pool set fast-pool \ crush_rule ssd_rule
CRUSH Map testen (crushtool)
# Placement simulieren: crushtool -i crushmap.bin --test \ --show-mappings \ --rule 0 --num-rep 3 \ --min-x 0 --max-x 100 # Verteilung prüfen: crushtool -i crushmap.bin --test \ --show-utilization # Map-Gültigkeit prüfen: crushtool --check crushmap.bin
💡 Rack Awareness: Replicas auf verschiedenen Racks verhindert Datenverlust bei Rack-Ausfall

PG-Zustände & Bedeutung
Zustand Bedeutung active+clean Normal — alles OK active+degradedReplica fehlt, aber I/O möglich active+recoveringRecovery läuft (Daten werden kopiert) active+backfillingBackfill auf neuen OSD active+remappedUmgemappt (z.B. nach OSD-Hinzufügen) peering PG handelt sich ab (kurz OK) stale Kein Status-Update — Primary OSD down? incomplete Zu wenige OSDs für Quorum (Datenverlust!) unknown Kein Report vom Primary inconsistent Scrub-Fehler gefunden → repair nötig repair Reparatur läuft gerade forced_backfillManuell priorisierter Backfill

Cinder-Backend (cinder.conf)
[ceph] volume_driver = \ cinder.volume.drivers.rbd.RBDDriver rbd_pool = volumes rbd_ceph_conf = /etc/ceph/ceph.conf rbd_flatten_volume_from_snapshot = false rbd_max_clone_depth = 5 rbd_user = cinder rbd_keyring_conf = \ /etc/ceph/ceph.client.cinder.keyring rbd_secret_uuid = LIBVIRT_SECRET_UUID
Glance-Config (glance-api.conf)
[glance_store] stores = rbd default_store = rbd rbd_store_pool = images rbd_store_user = glance rbd_store_ceph_conf = /etc/ceph/ceph.conf rbd_store_chunk_size = 8
Nova-Config (nova.conf)
[libvirt] images_type = rbd images_rbd_pool = vms images_rbd_ceph_conf = /etc/ceph/ceph.conf rbd_user = nova rbd_secret_uuid = LIBVIRT_SECRET_UUID # Libvirt-Secret erstellen: virsh secret-define secret.xml virsh secret-set-value \ --secret LIBVIRT_SECRET_UUID \ --base64 $(ceph auth get-key client.nova)

Diagnose & Debug
Logs
ceph log last 100 ceph -w # Live journalctl -u ceph-osd@0 -f journalctl -u ceph-mon@node01 -f journalctl -u ceph-mgr@node01 -f # Log-Level temporär erhöhen: ceph tell osd.0 injectargs \ --debug-osd 20
Daemon-Debugging
ceph daemon osd.0 status ceph daemon osd.0 config show ceph daemon osd.0 perf dump ceph daemon osd.0 dump_historic_ops ceph daemon osd.0 dump_ops_in_flight ceph daemon mon.node01 mon_status
Objekt lokalisieren
# Objekt → PG → OSDs: ceph osd map mypool myobject # PG direkt abfragen: ceph pg 1.4a query # Welche PGs auf OSD 3? ceph osd 3 pgs
Perf-Counters & I/O
ceph osd perf # Latenz aller OSDs ceph iostat 5 # I/O alle 5s ceph tell osd.* bench # Alle OSDs benchen ceph osd blocked-by # Blockierende OSDs
Crash-Reports
ceph crash ls # Crash-Liste ceph crash ls-new # Neue Crashes ceph crash info CRASH_ID ceph crash archive CRASH_ID ceph crash archive-all # Alle archivieren ceph crash prune \ --keep 30 # Älter als 30 Tage löschen
