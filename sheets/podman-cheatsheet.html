Container — Grundlagen
Starten / Stoppen
podman run -d --name web -p 8080:80 nginx podman run -it --rm alpine sh podman start|stop|restart|kill web podman rm -f web
Volumes & Mounts
podman run -v /host/path:/container/path:Z image # :Z = SELinux-Label setzen podman volume create mydata podman run -v mydata:/data image podman volume ls|inspect|rm mydata
Umgebung & Ressourcen
podman run -e MYVAR=value --env-file .env image podman run --memory 512m --cpus 2 image podman run --read-only --tmpfs /tmp image
Infos & Logs
podman ps -a podman logs -f web podman inspect web podman exec -it web bash podman stats --no-stream
📦
Images
Pull / Build / Push
podman pull docker.io/library/nginx:alpine podman build -t myapp:1.0 -f Dockerfile . podman push myapp:1.0 registry.example.com/myapp podman tag myapp:1.0 registry.example.com/myapp:latest
Image-Verwaltung
podman images podman rmi myapp:1.0 podman image prune -a podman save myapp:1.0 -o myapp.tar podman load -i myapp.tar
Registries
# /etc/containers/registries.conf: [registries.search] registries = ['registry.example.com', 'docker.io'] podman login registry.example.com podman search alpine

Pods
Pod erstellen & nutzen
podman pod create --name mypod -p 8080:80 podman run -d --pod mypod nginx podman run -d --pod mypod myapp podman pod ls podman pod inspect mypod podman pod start|stop|rm mypod
Kubernetes-Export
# Pod → Kubernetes YAML: podman generate kube mypod -f pod.yaml # YAML → Podman-Pod: podman play kube pod.yaml
⚙
Systemd-Integration
Systemd Unit generieren
# Für Container: podman generate systemd --new web \ -f ~/.config/systemd/user/container-web.service # Für Pod: podman generate systemd --new mypod \ -f ~/.config/systemd/user/pod-mypod.service systemctl --user enable --now container-web
Quadlet (Podman 4.4+)
# ~/.config/containers/systemd/nginx.container: [Container] Image=nginx:alpine PublishPort=8080:80 Volume=/srv/web:/usr/share/nginx/html:Z [Service] Restart=always [Install] WantedBy=default.target # Dann: systemctl --user daemon-reload

Netzwerk
Netzwerke
podman network create mynet podman network create --subnet 10.89.0.0/24 mynet podman run --network mynet --name db postgres podman network connect mynet existingcontainer podman network ls|inspect|rm mynet
Rootless Networking
# Rootless: slirp4netns / pasta (Standard) # Port < 1024 als Root oder mit: sysctl net.ipv4.ip_unprivileged_port_start=80 # DNS in Pods: aardvark-dns

Sicherheit / Rootless
Rootless-Konfiguration
# Rootless aktivieren (einmalig): podman system migrate # User-Namespace prüfen: cat /etc/subuid | grep $(whoami) # Laufende Infos: podman info --format '{{.Host.Security}}'
Podman Compose
podman-compose up -d podman-compose down podman-compose logs -f # Docker-Compose-Kompatibilität: podman compose --file docker-compose.yml up
