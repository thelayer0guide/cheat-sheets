Brendan Gregg Methodology
🖥️
top / htop — Prozessübersicht
top — wichtige Tastenkürzel
top # interaktiv starten top -b -n 1 > snapshot.txt # Snapshot in Datei top -p 1234,5678 # bestimmte PIDs Tasten: h=Hilfe, q=beenden, k=kill, r=renice 1=CPU-Kerne, m=Mem-Ansicht, t=Summary f=Felder wählen, o=Filter, W=Config speichern P=CPU sort, M=Mem sort, T=TIME sort
top — Header verstehen
load average: 1.23, 0.98, 0.87 → 1min 5min 15min us=user sy=kernel ni=nice id=idle wa=iowait hi=hw-irq si=sw-irq st=stolen(VM) VSZ=virtuell RSS=physisch resident SHR=geteilt
htop — nützliche Optionen
htop --tree # Prozessbaum htop -u nova # nur User 'nova' htop --filter python # vorgefilterter Start
💾
vmstat — Memory & CPU
vmstat — Verwendung
vmstat 1 # jede Sekunde aktualisieren vmstat 1 10 # 10 Mal, dann beenden vmstat -s # Memory-Statistik-Tabelle vmstat -d # Disk-Statistiken vmstat -t 1 # mit Timestamp
vmstat-Spalten verstehen
r = laufende Prozesse (> CPU-Kerne → überlastet) b = blockierte Prozesse (I/O-Wait) swpd = genutzter Swap (> 0 → Memory-Druck) si/so = Swap-In/Out (aktiv → KRITISCH) bi/bo = Block-In/Out (Disk-I/O in Blöcken/s) wa = CPU-iowait % (> 10-20% → Disk-Bottleneck) cs = context switches / sec
💿
iostat — Disk I/O
iostat — Verwendung
iostat -x 1 # extended stats, jede Sekunde iostat -x -d sda 1 # nur Disk sda iostat -h -x 1 # human-readable iostat -p sda # inkl. Partitionen
iostat -x Spalten verstehen
r/s w/s = Lese/Schreib-Operationen pro Sekunde rkB/s wkB/s = Durchsatz in KB/s await = mittl. I/O-Wartezeit in ms (<10ms gut) %util = Gerät-Auslastung (100% → gesättigt) svctm = Service-Zeit (veraltet, ignorieren) r_await/w_await = getrennte Read/Write-Latenz
📈
sar — Historische Daten
sar — Verwendung (sysstat-Paket)
sar -u 1 5 # CPU: 5x alle 1s sar -r 1 5 # Memory sar -b 1 5 # I/O sar -n DEV 1 5 # Netzwerk sar -q 1 5 # Load Average + Run Queue # Historische Daten von heute: sar -u -f /var/log/sysstat/sa$(date +%d) # Spezifische Uhrzeit: sar -u -s 10:00:00 -e 11:00:00
🔍
lsof & strace — Prozess-Diagnose
lsof — offene Dateien & Ports
lsof -p 1234 # offene Dateien von PID lsof -u nova # Dateien eines Users lsof -i :8080 # Port 8080 lsof -i tcp # alle TCP-Verbindungen lsof /var/log/nova.log # Prozesse mit Datei offen lsof -n -i :443 | grep LISTEN
strace — Syscalls verfolgen
strace -p 1234 # laufenden Prozess tracen strace -p 1234 -e trace=network strace -p 1234 -e trace=file strace -c -p 1234 # Syscall-Statistik strace -tt -f -p 1234 # mit Timestamps + Threads strace -o /tmp/trace.log -p 1234
⚡
perf — CPU-Profiling
perf stat & top
perf stat -p 1234 # CPU-Counters für PID perf stat -a sleep 5 # systemweit 5 Sekunden perf top # hot functions live perf top -p 1234 # nur PID
perf record & report (Flamegraphs)
perf record -g -p 1234 -- sleep 30 perf report perf script | flamegraph.pl > flame.svg # perf für OpenStack-Dienst: perf record -g -F 99 -p $(pgrep nova-compute) -- sleep 30

Netzwerk-Performance
Netzwerk-Diagnose
ss -s # Socket-Statistik ss -tlnp # TCP listening + PID netstat -s | grep -i error ip -s link # Interface-Statistiken (errors, drops) ethtool -S eth0 # NIC-Counters sar -n DEV 1 5 # rxpck/s txpck/s rxkB/s
Bandbreite & Latenz messen
# Bandbreite: iperf3 -s # Server iperf3 -c 10.0.0.1 # Client # Latenz: ping -c 100 10.0.0.1 | tail -2 # MTU testen: ping -M do -s 1472 10.0.0.1
💡 USE-Methode: Utilization, Saturation, Errors — für jeden Ressourcentyp prüfen

/proc — Kernel-Infos
Wichtige /proc-Einträge
cat /proc/meminfo # Speicher-Details cat /proc/cpuinfo # CPU-Details cat /proc/loadavg # Load + Run-Queue cat /proc/interrupts # IRQ-Verteilung cat /proc/net/sockstat # Socket-Statistiken cat /proc/diskstats # Disk-I/O-Rohwerte cat /proc/1234/status # Prozess-Details cat /proc/1234/limits # ulimits eines Prozesses ls -l /proc/1234/fd # offene Dateideskriptoren
free & uptime
free -h # Speicher human-readable free -h -s 2 # alle 2s aktualisieren uptime # load average w # users + load dmesg -T | tail -20 # Kernel-Meldungen dmesg -T | grep -i "oom\|error\|fail"
