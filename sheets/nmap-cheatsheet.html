Scan-Typen
TCP SYN Scan (Standard, benötigt root)
nmap -sS 192.168.1.1
TCP Connect Scan (kein root nötig)
nmap -sT 192.168.1.1
UDP Scan
nmap -sU 192.168.1.1
TCP ACK Scan (Firewall-Mapping)
nmap -sA 192.168.1.1
Ping-only (kein Port-Scan)
nmap -sn 192.168.1.0/24
TCP Window / FIN / Null / Xmas Scan
nmap -sW 192.168.1.1 # Window nmap -sF 192.168.1.1 # FIN nmap -sN 192.168.1.1 # Null nmap -sX 192.168.1.1 # Xmas
Hinweis: -sS (SYN) ist der schnellste und unauffälligste TCP-Scan, da keine vollständige Verbindung aufgebaut wird. Root-Rechte erforderlich.
🌐
Host Discovery
Ping Scan (kein Port-Scan)
nmap -sn 10.0.0.0/24
Host als online behandeln (kein Ping)
nmap -Pn 192.168.1.1
TCP SYN Discovery auf Port 443
nmap -PS443 192.168.1.0/24
TCP ACK Discovery auf Port 80
nmap -PA80 192.168.1.0/24
UDP Discovery auf Port 53
nmap -PU53 192.168.1.0/24
ICMP Echo, Timestamp & Netmask
nmap -PE 192.168.1.0/24 # Echo nmap -PP 192.168.1.0/24 # Timestamp nmap -PM 192.168.1.0/24 # Netmask
Ziele aus Datei einlesen
nmap -iL hosts.txt -sn
-Pn überspringt die Host-Discovery — sinnvoll bei Hosts, die auf Ping nicht antworten (z. B. Windows-Firewall).
🔌
Port-Angaben
Einzelner Port / Portbereich
nmap -p 22 192.168.1.1 nmap -p 22,80,443 192.168.1.1 nmap -p 1-1024 192.168.1.1
Alle 65535 Ports scannen
nmap -p- 192.168.1.1
Top N häufigste Ports
nmap --top-ports 100 192.168.1.1 nmap --top-ports 1000 192.168.1.1
Fast Scan (100 häufigste Ports)
nmap -F 192.168.1.1
Protokoll-spezifische Ports (TCP/UDP)
nmap -p T:80,U:53 192.168.1.1
Port-Bereich ausschließen
nmap -p 1-65535 --exclude-ports 9100 192.168.1.1
🏷️
Service & Version
Service-Version erkennen
nmap -sV 192.168.1.1
Default NSE-Scripts ausführen
nmap -sC 192.168.1.1
Version + Scripts kombiniert
nmap -sV -sC 192.168.1.1
Intensität der Versionserkennung (0–9)
nmap --version-intensity 5 192.168.1.1 # 0 = leicht, 9 = aggressiv nmap --version-light 192.168.1.1 # =2 nmap --version-all 192.168.1.1 # =9
-A aktiviert OS-Erkennung, Version, Scripts und Traceroute gleichzeitig.
💻
OS-Erkennung
OS-Fingerprinting
nmap -O 192.168.1.1
OS-Guess aggressiver machen
nmap -O --osscan-guess 192.168.1.1
OS-Scan auf günstige Ziele beschränken (mind. 1 offen+geschlossen)
nmap -O --osscan-limit 192.168.1.1
Aggressive Erkennung (OS+Version+Script)
nmap -A 192.168.1.1
Hinweis: OS-Erkennung benötigt mindestens einen offenen und einen geschlossenen Port. Root-Rechte sind erforderlich.
⏱️
Timing & Performance
Timing-Template (T0–T5)
nmap -T0 192.168.1.1 # paranoid (IDS-Umgehung) nmap -T1 192.168.1.1 # sneaky nmap -T2 192.168.1.1 # polite nmap -T3 192.168.1.1 # normal (default) nmap -T4 192.168.1.1 # aggressive nmap -T5 192.168.1.1 # insane
Minimale Paketrate (Pakete/Sekunde)
nmap --min-rate 1000 192.168.1.1
Maximale Paketrate begrenzen
nmap --max-rate 500 192.168.1.1
Maximale Wiederholungsversuche
nmap --max-retries 2 192.168.1.1
Parallele Hosts & Probes
nmap --min-hostgroup 64 10.0.0.0/24 nmap --min-parallelism 100 10.0.0.0/24
Timeout pro Host setzen
nmap --host-timeout 30s 192.168.1.1
📄
Output-Formate
Normal (lesbar), XML, Grepable
nmap -oN scan.txt 192.168.1.1 nmap -oX scan.xml 192.168.1.1 nmap -oG scan.gnmap 192.168.1.1
Alle Formate gleichzeitig (-oN/-oX/-oG)
nmap -oA ergebnis 192.168.1.1 # erzeugt: ergebnis.nmap, ergebnis.xml, ergebnis.gnmap
Script-Kiddie-Output (spaßig)
nmap -oS l33t.txt 192.168.1.1
Verbosität erhöhen
nmap -v 192.168.1.1 # verbose nmap -vv 192.168.1.1 # sehr verbose nmap -d 192.168.1.1 # debug
Paket-Trace ausgeben
nmap --packet-trace 192.168.1.1
-oA ist empfohlen: XML-Ausgabe kann mit ndiff oder xsltproc weiterverarbeitet werden.
📜
NSE Scripts
Default-Scripts (wie -sC)
nmap --script default 192.168.1.1
Script-Kategorien
nmap --script safe 192.168.1.1 nmap --script vuln 192.168.1.1 nmap --script auth 192.168.1.1 nmap --script discovery 192.168.1.1 nmap --script intrusive 192.168.1.1
Einzelne Scripts ausführen
nmap --script http-title 192.168.1.1 nmap --script ssh-hostkey 192.168.1.1 nmap --script smb-vuln-ms17-010 192.168.1.1
Script-Argumente übergeben
nmap --script http-brute \ --script-args userdb=users.txt,passdb=pass.txt \ -p 80 192.168.1.1
Wildcard / mehrere Scripts
nmap --script "http-*" 192.168.1.1 nmap --script http-title,http-headers 192.168.1.1
Script-Datenbank aktualisieren
nmap --script-updatedb
Scripts liegen in /usr/share/nmap/scripts/. Mit nmap --script-help vuln Beschreibungen anzeigen.
🛡️
Firewall-Umgehung
IP-Fragmentierung (8-Byte-Fragmente)
nmap -f 192.168.1.1 nmap -ff 192.168.1.1 # 16-Byte-Fragmente
Decoy-IPs (Tarnung durch Köder)
nmap -D 10.0.0.5,10.0.0.6,ME 192.168.1.1 nmap -D RND:10 192.168.1.1 # 10 zufällige Decoys
Quellport vortäuschen
nmap --source-port 53 192.168.1.1 nmap -g 443 192.168.1.1
Datenlänge auffüllen (IDS-Evasion)
nmap --data-length 25 192.168.1.1
MAC-Adresse fälschen
nmap --spoof-mac 0 192.168.1.1 # zufällig nmap --spoof-mac Apple 192.168.1.1
Checksummen-Fehler setzen
nmap --badsum 192.168.1.1
-D RND:10 erzeugt 10 zufällige Köder-IPs. ME als Position einfügen, um den echten Scan-Port-Eintrag zu steuern.
⚡
Typische Scan-Rezepte
Schneller Netzwerk-Überblick
nmap -sn -T4 10.0.0.0/24
Standard-Vollscan (alle Ports, Version, OS)
nmap -sS -sV -sC -O -p- -T4 -oA full 192.168.1.1
Schneller Top-100-Scan
nmap -F -T4 192.168.1.0/24
Stealth-Scan (langsam, IDS-freundlich)
nmap -sS -T1 -f --data-length 20 \ -D RND:5 192.168.1.1
Vulnerability-Scan
nmap -sV --script vuln -p- -T4 192.168.1.1
SMB-Schwachstellen prüfen
nmap -p 445 --script smb-vuln-* 192.168.1.1
Web-Dienste untersuchen
nmap -p 80,443,8080,8443 --script http-* 192.168.1.1
UDP-Hauptports (langsam!)
nmap -sU --top-ports 200 -T4 192.168.1.1
🎯
Ziel-Angaben
Einzelner Host / Hostname
nmap 192.168.1.1 nmap example.local
CIDR-Netz / Bereich / Wildcard
nmap 192.168.1.0/24 nmap 192.168.1.1-254 nmap 192.168.1.*
Mehrere Hosts & Netzwerke
nmap 192.168.1.1 10.0.0.1 172.16.0.0/16
Ziele aus Datei / zufällig
nmap -iL targets.txt nmap -iR 100 # 100 zufällige Hosts
Hosts ausschließen
nmap 192.168.1.0/24 --exclude 192.168.1.1 nmap 192.168.1.0/24 --excludefile excl.txt
--exclude schützt kritische Hosts (z. B. Produktions-Server) vor versehentlichem Scan.

Sonstiges & Diagnose
nmap-Version anzeigen
nmap -V
Scan ohne Ports-Scan (Host-Liste)
nmap -sL 192.168.1.0/24 # nur auflisten
IPv6-Scan
nmap -6 fe80::1
Traceroute
nmap --traceroute 8.8.8.8
Laufzeit-Statistik anzeigen
nmap --stats-every 10s 192.168.1.0/24
Scan-Ergebnisse vergleichen
ndiff scan1.xml scan2.xml
XML nach HTML konvertieren
xsltproc scan.xml -o scan.html
ndiff und xsltproc sind im nmap-Paket enthalten. ndiff eignet sich für Änderungs-Tracking über Zeit.
