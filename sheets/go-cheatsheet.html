Error Handling
Mehrere Rückgabewerte
func divide(a, b float64) (float64, error) { if b == 0 { return 0, fmt.Errorf("division by zero") } return a / b, nil }

Eigener Fehler

Goroutines & Channels
Goroutine starten
go doWork() go func() { fmt.Println("async") }()
Channel
ch := make(chan int, 10) // buffered ch <- 42 // senden val := <-ch // empfangen close(ch)
select
select { case msg := <-ch1: fmt.Println(msg) case ch2 <- value: fmt.Println("sent") default: fmt.Println("no activity") }
sync.WaitGroup
var wg sync.WaitGroup wg.Add(1) go func() { defer wg.Done() doWork() }() wg.Wait()
💡
defer, panic, recover
defer (LIFO)
func openFile() { f, _ := os.Open("x.txt") defer f.Close() // am Scope-Ende // arbeiten mit f... }
panic & recover
defer func() { if r := recover(); r != nil { fmt.Println("recovered:", r) } }() panic("something went wrong")
Tipp: Errors als Wert, nicht als Exceptions. defer für Cleanup immer direkt nach open/lock.
🔀
Kontrollstrukturen
for (einzige Schleife)
for i := 0; i < 10; i++ { ... } for _, v := range slice { ... } for k, v := range m { ... } for { /* endlos */ }
switch (kein fallthrough per default)
switch x { case 1, 2: fmt.Println("eins oder zwei") default: fmt.Println("andere") }
