🔑
Schlüssel generieren
Private Key erzeugen
wg genkey
Private Key in Datei speichern (umask schützen)
umask 077 wg genkey | tee /etc/wireguard/privatekey \ | wg pubkey > /etc/wireguard/publickey
Public Key aus Private Key ableiten
wg pubkey < /etc/wireguard/privatekey
Pre-shared Key (optionale Extra-Sicherheit)
wg genpsk > /etc/wireguard/preshared.key
Sicherheit: Private Keys niemals in Logs, Skripten oder Versionierung ablegen. Dateiberechtigungen: 600.
⚙️
Interface-Konfiguration (/etc/wireguard/wg0.conf)
Server-Seite — minimale Konfiguration
[Interface] Address = 10.0.0.1/24 ListenPort = 51820 PrivateKey = <SERVER_PRIVATE_KEY> [Peer] PublicKey = <CLIENT_PUBLIC_KEY> AllowedIPs = 10.0.0.2/32
Client-Seite — inkl. Pre-shared Key und DNS
[Interface] Address = 10.0.0.2/32 PrivateKey = <CLIENT_PRIVATE_KEY> DNS = 1.1.1.1 [Peer] PublicKey = <SERVER_PUBLIC_KEY> PresharedKey = <PSK> Endpoint = server.example.com:51820 AllowedIPs = 0.0.0.0/0, ::/0 PersistentKeepalive = 25
AllowedIPs: Definiert, welche Pakete durch den Tunnel geschickt werden. 0.0.0.0/0 = Full-Tunnel (gesamter Traffic).
⚡
wg-quick — Interface verwalten
Tunnel starten
wg-quick up wg0
Tunnel stoppen
wg-quick down wg0
Beim Systemstart aktivieren (systemd)
systemctl enable --now wg-quick@wg0
Status des Systemd-Service
systemctl status wg-quick@wg0
Alternative: manuelle ip-Befehle (ohne wg-quick)
ip link add wg0 type wireguard ip addr add 10.0.0.1/24 dev wg0 wg setconf wg0 /etc/wireguard/wg0.conf ip link set wg0 up

wg — Status & Diagnose
Alle aktiven WireGuard-Interfaces anzeigen
wg show
Bestimmtes Interface anzeigen
wg show wg0
Nur Peers auflisten
wg show wg0 peers
Transfer-Statistiken je Peer
wg show wg0 transfer
Letzter Handshake-Zeitpunkt je Peer
wg show wg0 latest-handshakes
Aktuelle Konfiguration ausgeben (laufendes Interface)
wg showconf wg0
➕
Peers zur Laufzeit verwalten
Peer hinzufügen (ohne Neustart)
wg set wg0 peer <PUBKEY> \ allowed-ips 10.0.0.3/32
Peer mit Endpoint und Keepalive hinzufügen
wg set wg0 peer <PUBKEY> \ endpoint 192.168.1.5:51820 \ allowed-ips 10.0.0.3/32 \ persistent-keepalive 25
Peer mit Pre-shared Key hinzufügen
wg set wg0 peer <PUBKEY> \ preshared-key /etc/wireguard/preshared.key \ allowed-ips 10.0.0.3/32
Peer entfernen
wg set wg0 peer <PUBKEY> remove
Laufende Konfiguration persistent speichern
wg-quick save wg0
Achtung: wg set ändert die laufende Konfiguration im RAM. wg-quick save schreibt den aktuellen Zustand in die .conf-Datei zurück.

Routing & IP-Forwarding
IP-Forwarding temporär aktivieren
sysctl -w net.ipv4.ip_forward=1 sysctl -w net.ipv6.conf.all.forwarding=1
IP-Forwarding dauerhaft (/etc/sysctl.d/99-wg.conf)
net.ipv4.ip_forward = 1 net.ipv6.conf.all.forwarding = 1
NAT für ausgehenden Internet-Traffic (iptables)
iptables -t nat -A POSTROUTING \ -o eth0 -j MASQUERADE iptables -A FORWARD \ -i wg0 -j ACCEPT iptables -A FORWARD \ -o wg0 -j ACCEPT
NAT in wg0.conf via PostUp/PreDown integrieren
[Interface] # ... PostUp = iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE PreDown = iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE
Split-Tunnel: Nur bestimmte Netze in AllowedIPs eintragen statt 0.0.0.0/0 — z. B. 10.0.0.0/8, 192.168.0.0/16.
🛡️
Firewall & Ports
UDP-Port in firewalld öffnen
firewall-cmd --permanent \ --add-port=51820/udp firewall-cmd --reload
UDP-Port mit iptables öffnen
iptables -A INPUT -p udp \ --dport 51820 -j ACCEPT
UFW-Regel (Ubuntu)
ufw allow 51820/udp
WireGuard-Interface in firewalld-Zone einordnen
firewall-cmd --permanent \ --zone=trusted \ --add-interface=wg0 firewall-cmd --reload
Standard-Port: WireGuard hat keinen offiziellen Port. 51820/udp ist die gängige Konvention.

Kernel-Modul & Installation
Installation (Debian/Ubuntu)
apt install wireguard wireguard-tools
Installation (RHEL/Rocky/Alma 8+)
dnf install epel-release dnf install wireguard-tools
Installation (Arch Linux)
pacman -S wireguard-tools
Kernel-Modul laden
modprobe wireguard lsmod | grep wireguard
WireGuard-Version prüfen
wg --version
Kernel ≥ 5.6: WireGuard ist seit Linux 5.6 im Mainline-Kernel enthalten. Auf älteren Kerneln wird das DKMS-Modul benötigt.

MTU & Performance
MTU explizit setzen (Standard: 1420 für IPv4+IPv6)
[Interface] MTU = 1420
MTU am laufenden Interface setzen
ip link set dev wg0 mtu 1420
Aktuelle MTU prüfen
ip link show wg0
Latenz und Durchsatz messen
ping -c 10 10.0.0.1 # Latenz iperf3 -s # Server iperf3 -c 10.0.0.1 -t 10 # Client
MTU-Berechnung: Physische MTU − 60 (IPv4) oder − 80 (IPv6) für den WireGuard-Overhead. Bei PPPoE: 1420 − 8 = 1412.

Diagnose & Fehlerbehebung
Handshake-Alter prüfen (0 = kein Kontakt)
wg show wg0 latest-handshakes
UDP-Traffic auf WireGuard-Port beobachten
tcpdump -i eth0 -n udp port 51820
Kernel-Logs (wireguard-Modul)
dmesg | grep -i wireguard journalctl -k | grep wireguard
wg-quick-Log via journald
journalctl -u wg-quick@wg0 -f
Routing-Tabelle prüfen
ip route show table main ip route show table all | grep wg0
Verbindung zum Peer testen (via Tunnel)
ping -I wg0 10.0.0.2
Kein Handshake: Häufigste Ursachen — falscher Public Key, Endpoint nicht erreichbar (Firewall/UDP-Port), falsches AllowedIPs.

Multi-Peer / Road-Warrior
Server-Konfiguration mit mehreren Clients
[Interface] Address = 10.0.0.1/24 ListenPort = 51820 PrivateKey = <SERVER_PRIVKEY> PostUp = iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE PreDown = iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE [Peer] # Laptop PublicKey = <LAPTOP_PUBKEY> AllowedIPs = 10.0.0.2/32 [Peer] # Handy PublicKey = <PHONE_PUBKEY> AllowedIPs = 10.0.0.3/32
Jeder Client erhält eine eindeutige IP in AllowedIPs (/32). Clients kommunizieren über den Server, nicht direkt.
🔗
Site-to-Site VPN
Standort A (192.168.10.0/24) — wg0.conf
[Interface] Address = 10.10.10.1/30 PrivateKey = <SITE_A_PRIVKEY> ListenPort = 51820 [Peer] PublicKey = <SITE_B_PUBKEY> Endpoint = site-b.example.com:51820 AllowedIPs = 10.10.10.2/32, 192.168.20.0/24
Standort B (192.168.20.0/24) — wg0.conf
[Interface] Address = 10.10.10.2/30 PrivateKey = <SITE_B_PRIVKEY> ListenPort = 51820 [Peer] PublicKey = <SITE_A_PUBKEY> Endpoint = site-a.example.com:51820 AllowedIPs = 10.10.10.1/32, 192.168.10.0/24 PersistentKeepalive = 25
IP-Forwarding muss auf beiden Seiten aktiviert sein. Routen zu den Remote-Netzen werden durch AllowedIPs automatisch gesetzt.

Nützliche Snippets
Alle Peer-Verbindungszeiten anzeigen (lesbar)
wg show wg0 latest-handshakes | \ awk '{cmd="date -d @"$2" +\"%F %T\""; cmd | getline d; close(cmd); print $1, d}'
Alle WireGuard-Interfaces auf einmal hochbringen
for conf in /etc/wireguard/*.conf; do wg-quick up "$(basename "${conf%.conf}")" done
Konfiguration validieren (Syntax-Check)
wg-quick strip wg0 > /dev/null && echo OK
Endpunkt eines Peers live aktualisieren
wg set wg0 peer <PUBKEY> \ endpoint new-host.example.com:51820
wg-quick strip gibt die Konfiguration ohne wg-quick-spezifische Felder (PostUp, MTU, DNS …) aus — nützlich für manuelle ip/wg-Befehle.

Kryptographie-Überblick
Protokoll-Stack
Key Exchange : Curve25519 (ECDH) Verschlüss. : ChaCha20-Poly1305 Hashing : BLAKE2s Handshake : Noise Protocol Framework (IKpsk2-Pattern)
Key-Längen
Private Key : 32 Byte (256 Bit) — Base64 (44 Zeichen) Public Key : 32 Byte (256 Bit) — Base64 (44 Zeichen) Pre-shared : 32 Byte (256 Bit) — Base64 (44 Zeichen)
Handshake-Intervall
# Automatisch alle 3 Minuten (180 s) # Kein manueller Eingriff nötig wg show wg0 latest-handshakes
Perfect Forward Secrecy: Ephemere Session-Keys werden beim Handshake ausgehandelt und alle 3 Minuten erneuert. Kompromittierte Long-Term-Keys enthüllen keine alten Sessions.
📋
Konfigurationsfelder — Referenz
[Interface]-Felder
PrivateKey # Pflicht — eigener Private Key Address # IP des Interfaces (CIDR) ListenPort # UDP-Port (Server) DNS # Nameserver (wg-quick) MTU # Paketgröße (Standard: 1420) Table # Routing-Tabelle (off|auto|N) PreUp/PostUp # Hook-Skripte PreDown/PostDown # Hook-Skripte
[Peer]-Felder
PublicKey # Pflicht — Public Key des Peers AllowedIPs # Pflicht — erlaubte Quell-IPs Endpoint # host:port des Peers PresharedKey # Optionaler PSK PersistentKeepalive # Intervall in Sekunden (NAT)
Table = off: WireGuard setzt keine automatischen Routen — nützlich für benutzerdefiniertes Routing (policy routing, VRF).
