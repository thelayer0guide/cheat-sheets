Grundbefehle
Status & Reload
systemctl status firewalld firewall-cmd --state # running/not running firewall-cmd --reload # Konfiguration neu laden firewall-cmd --complete-reload # Kernel-Module neu laden
Temporär vs. Permanent
# Ohne --permanent: gilt nur bis zum Reload/Reboot firewall-cmd --add-port=80/tcp # Mit --permanent: dauerhaft gespeichert firewall-cmd --permanent --add-port=80/tcp firewall-cmd --reload # danach noch reload!
Alles anzeigen
firewall-cmd --list-all firewall-cmd --list-all-zones firewall-cmd --get-active-zones
🌍
Zonen
Vordefinierte Zonen
drop # alles verworfen block # abgelehnt mit icmp-*-prohibited public # Standard für öffentliche Netze external # für Router mit Masquerade dmz # Demilitarisierte Zone work # Arbeitsumgebung home # Heimnetz internal # internes Netz trusted # alles erlaubt
Zone für Interface
firewall-cmd --get-zone-of-interface=eth0 firewall-cmd --permanent --zone=internal \ --add-interface=eth1 firewall-cmd --permanent --zone=public \ --change-interface=eth0
Default Zone
firewall-cmd --get-default-zone firewall-cmd --set-default-zone=public

Services & Ports
Services hinzufügen
firewall-cmd --permanent --add-service=http firewall-cmd --permanent --add-service=https firewall-cmd --permanent --add-service=ssh firewall-cmd --permanent --zone=public \ --add-service=postgresql firewall-cmd --permanent --remove-service=telnet
Ports
firewall-cmd --permanent --add-port=8080/tcp firewall-cmd --permanent --add-port=5432/tcp firewall-cmd --permanent --add-port=3000-3100/tcp firewall-cmd --permanent --remove-port=8080/tcp firewall-cmd --list-ports
Verfügbare Services
firewall-cmd --get-services ls /usr/lib/firewalld/services/
Eigener Service: /etc/firewalld/services/myapp.xml erstellen, dann --add-service=myapp.
📋
Rich Rules
Rich Rule Syntax
# rule [family] [source] [destination] # [service|port|protocol] [log] [audit] [action] firewall-cmd --permanent \ --add-rich-rule='rule family="ipv4" source address="10.0.0.0/24" service name="ssh" accept'
Rate Limiting
firewall-cmd --permanent --add-rich-rule=\ 'rule service name="ssh" limit value="5/m" accept'
IP blockieren
firewall-cmd --permanent --add-rich-rule=\ 'rule family="ipv4" source address="1.2.3.4" reject' # Oder mit ipset (erstellen + Eintrag): firewall-cmd --permanent --new-ipset=blocklist --type=hash:ip firewall-cmd --permanent \ --ipset=blocklist --add-entry=1.2.3.4
Logging
firewall-cmd --permanent --add-rich-rule=\ 'rule family="ipv4" source address="192.168.1.0/24" port port=22 protocol=tcp log prefix="SSH:" level="info" accept'

NAT & Weiterleitung
Masquerade (SNAT)
firewall-cmd --permanent --zone=external \ --add-masquerade firewall-cmd --query-masquerade
Port-Weiterleitung
# Lokal: 8080 → 80 firewall-cmd --permanent \ --add-forward-port=port=8080:proto=tcp:toport=80 # Zu anderem Host: firewall-cmd --permanent \ --add-forward-port=port=3306:proto=tcp:toport=3306:toaddr=10.0.0.5
IP-Forwarding
echo 'net.ipv4.ip_forward=1' >> /etc/sysctl.d/99-firewalld.conf sysctl -p /etc/sysctl.d/99-firewalld.conf
