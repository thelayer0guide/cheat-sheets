Proxy & Intercept
Browser-Proxy konfigurieren
Proxy-Host: 127.0.0.1 Proxy-Port: 8080 # In Firefox: Einstellungen → Proxy # Burp-Browser: eingebaut, kein Setup
CA-Zertifikat installieren (HTTPS)
http://burpsuite # oder http://burp # → "CA Certificate" herunterladen # → Im Browser als Trusted CA importieren
Intercept ein-/ausschalten
Proxy → Intercept [Intercept is on] → Requests anhalten [Intercept is off] → Alles durchlassen
Request-Aktionen im Intercept
Forward # Request weitersenden Drop # Request verwerfen Action → Send to Repeater (Ctrl+R) Action → Send to Intruder (Ctrl+I)
Match & Replace (Proxy → Options)
# Automatisch Requests/Responses ändern Type: Request header Match: User-Agent: .* Replace: User-Agent: BurpScanner/1.0
HTTP History filtern
Filter-Leiste: Show only in-scope items Hide: CSS, Images, JS (Noise reduzieren) Search: Ctrl+F → Stichwort in Responses
Tipp: Burps eingebauter Chromium-Browser (Proxy → Open Browser) hat das CA-Zertifikat bereits installiert — ideal für schnellen Start.

Repeater
Request senden (Shortcut)
Ctrl+R # Send to Repeater (aus History/Intercept) Ctrl+Space # Send (im Repeater) # oder: Button "Send"
Request bearbeiten
# Direkt im Request-Feld tippen GET /api/user?id=1 HTTP/1.1 Host: example.com Cookie: session=abc123 # Parameter, Header, Body frei editierbar
Responses vergleichen
# Mehrere Tabs nebeneinander öffnen Tab 1: id=1 → Response A Tab 2: id=2 → Response B # Comparer: Action → "Compare responses"
URL-Encode beim Schreiben
# Rechtsklick im Request → "URL-encode" # Oder manuell: Leerzeichen = %20 # Sonderzeichen: & = %26, = = %3D
Tipp: Mit Ctrl+Z lassen sich Änderungen im Request-Editor rückgängig machen. Tabs lassen sich umbenennen (Doppelklick auf Tab-Namen).

Intruder
Attack Types
Sniper # 1 Payload-Set, eine Position Battering Ram # 1 Set → alle Positionen gleich Pitchfork # n Sets → jeweils eine Position Cluster Bomb # n Sets → Kreuzprodukt aller Pos.
Payload-Positionen markieren
POST /login HTTP/1.1 ... username=§admin§&password=§pass§ # §...§ = Payload-Marker (Auto-markieren möglich)
Payload-Typen (Payloads-Tab)
Simple list # Wortliste (manuell / aus Datei) Numbers # Zahlenbereich (z. B. 1–1000) Brute forcer # Zeichenkombinationen Runtime file # Datei zur Laufzeit lesen Username generator # Variationen eines Namens
Grep-Match (Treffer markieren)
# Settings → Grep - Match Ausdruck: "Invalid password" # Spalte "Invalid password" = true/false # Erleichtert Treffer-Identifikation
Grep-Extract (Wert aus Response)
# Settings → Grep - Extract # Regex oder Delimiter-basiert # z. B. CSRF-Token aus Response extrahieren
Community-Limit: Intruder ist in der Community-Edition auf 1 Request/Sekunde gedrosselt. Für schnelle Brute-Force-Angriffe wird die Pro-Version benötigt.

Scanner / Active Scan (Pro)
Scan starten
# Methode 1: Aus HTTP History Rechtsklick → "Scan" # Methode 2: Dashboard → New Scan URL eingeben → Scan-Typ wählen
Scan-Typen
Crawl # Seitenstruktur entdecken Audit # Schwachstellen testen Crawl & Audit # Beides kombiniert
Scan-Konfiguration
# Dashboard → New Scan → Scan config Audit checks: All / Fast / Light Insertion points: Parameter, Headers, Cookies, Body Concurrency: Threads / Requests/s
Passive Scan (Community & Pro)
# Automatisch für alle Proxy-Requests # Findet: Missing headers, Info-Leaks # Target → Issue Activity ansehen
Issue-Severity-Filter
High / Medium / Low / Information # Target → Site map → Issues-Tab # "Issue definitions" = komplette Liste
Scope einhalten: Vor einem aktiven Scan immer den Scope definieren, um unbeabsichtigte Tests auf fremde Hosts zu vermeiden.

Target & Scope
Host zu Scope hinzufügen
# Methode 1: HTTP History Rechtsklick auf Request → "Add to scope" # Methode 2: Target → Scope → Include URL: https://example.com/.*
Scope-Regeln (Regex)
# Include: Protocol: https Host: example\.com File: .* # Exclude (z. B. Logout-Seite): File: /logout
Proxy: Nur Scope-Traffic intercepten
# Proxy → Options → Intercept Client Requests Regel: URL Is in target scope # Reduziert Noise aus Drittanbieter-Requests
Site Map
# Target → Site map Baumstruktur aller gesehenen URLs Filter: Show only in-scope items Rechtsklick → Spider/Crawl this host (Pro)
Tipp: Mit aktiviertem Scope-Filter in der HTTP-History bleibt nur der relevante Traffic sichtbar — wichtig bei Tests mit vielen Drittanbieter-Ressourcen.

Decoder & Comparer
Decoder — Encode-Typen
URL encode / decode HTML encode / decode Base64 encode / decode Hex encode / decode Gzip decompress Hash MD5 / SHA-1 / SHA-256
Decoder verwenden
# Decoder-Tab öffnen Text einfügen → Decode as → URL # Kette: Base64 → dann URL-decode # "Smart decode" erkennt Format automatisch
Comparer — Requests/Responses vergleichen
# Methode 1: Rechtsklick → "Send to Comparer" # Methode 2: Comparer-Tab → Paste Compare: Words / Bytes # Unterschiede farblich hervorgehoben
Typische Decoder-Anwendungsfälle
# JWT dekodieren: Header.Payload.Sig → Base64-decode # Cookie-Wert analysieren: URL-decode → Base64-decode # XSS-Payload enkodieren: <script> → HTML-encode
Tipp: Im Repeater oder Intruder lassen sich markierte Werte direkt per Rechtsklick in den Decoder senden: Send to Decoder.

Häufige Checks
IDOR (Insecure Direct Object Reference)
GET /api/invoice?id=1001 # → id auf andere Werte ändern (Repeater) # → Benutzer-ID in JWT / Cookie tauschen # → Intruder: Numbers 1000–1100
SQL-Injection
id=1' # Error testen id=1 AND 1=1-- # Boolean-based id=1 ORDER BY 5-- # Spaltenanzahl # Scanner (Pro) findet SQLi automatisch
XSS (Reflected / Stored)
param=<script>alert(1)</script> param="><img src=x onerror=alert(1)> param=javascript:alert(1) # Response: Payload im HTML-Body sichtbar?
SSRF (Server-Side Request Forgery)
url=http://169.254.169.254/latest/meta-data/ url=http://localhost/admin url=http://burp-collaborator-id.oastify.com # Antwort: Interner Inhalt / DNS-Lookup
Auth-Bypass / Broken Auth
# Session-Token in Repeater austauschen # JWT: alg=none, Key-Confusion (RS→HS) # Admin-Funktion mit Low-Priv-Token aufrufen # Password-Reset-Token: Entropy prüfen
Security-Header-Check (Passiv)
# Passive Scan / Response-Inspektion: X-Frame-Options # Clickjacking Content-Security-Policy # XSS-Schutz Strict-Transport-Security # HSTS X-Content-Type-Options # MIME-Sniffing
Wichtig: Alle Tests nur auf explizit autorisierten Systemen durchführen. Burp Suite ist ein professionelles Penetrationstesting-Tool — unautorisierter Einsatz ist illegal.
