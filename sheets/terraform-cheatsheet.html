Grundbefehle
Initialisieren
terraform init terraform init -upgrade # Provider aktualisieren terraform init -reconfigure
Plan erstellen
terraform plan terraform plan -out=myplan.tfplan terraform plan -target=openstack_compute_instance_v2.web terraform plan -destroy # Destroy-Plan
Anwenden
terraform apply terraform apply myplan.tfplan terraform apply -auto-approve
Zerstören
terraform destroy terraform destroy -auto-approve terraform destroy -target=openstack_compute_instance_v2.web
Formatieren & Validieren
terraform fmt -recursive terraform validate
💡 tofu ist ein Drop-in-Ersatz für terraform (OpenTofu fork)
📋
State Management
State anzeigen
terraform state list terraform state show openstack_compute_instance_v2.web terraform show
Ressource aus State entfernen
terraform state rm openstack_compute_instance_v2.web
Ressource umbenennen im State
terraform state mv \ openstack_compute_instance_v2.old \ openstack_compute_instance_v2.new
Bestehende Ressource importieren
terraform import \ openstack_compute_instance_v2.web \ SERVER_UUID terraform import \ openstack_networking_network_v2.net \ NETWORK_UUID
State aktualisieren & Refresh
terraform refresh terraform apply -refresh-only
State-Datei manuell bearbeiten
terraform state pull > backup.tfstate terraform state push backup.tfstate
📝
Variables & Outputs
Variable deklarieren (variables.tf)
variable "flavor" { type = string default = "m1.small" description = "Instance flavor" } variable "instance_count" { type = number default = 1 }
Output deklarieren (outputs.tf)
output "server_ip" { value = openstack_compute_instance_v2.web.access_ip_v4 description = "Public IP address" }
Variablen übergeben
terraform plan -var "flavor=m1.large" terraform plan -var-file=prod.tfvars # Env-Variable: export TF_VAR_flavor=m1.large
Outputs anzeigen
terraform output terraform output server_ip terraform output -json
locals.tf — Berechnete Werte
locals { name_prefix = "${var.env}-${var.project}" tags = { environment = var.env } }

OpenStack Provider
Provider konfigurieren (provider.tf)
terraform { required_providers { openstack = { source = "terraform-provider-openstack/openstack" version = "~> 2.0" } } } provider "openstack" { auth_url = "https://keystone:5000/v3" user_name = var.os_user password = var.os_password tenant_name = var.os_project region = "RegionOne" }
Alternativ: clouds.yaml / openrc.sh
provider "openstack" { cloud = "mycloud" # aus ~/.config/openstack/clouds.yaml }
Compute Instance (Nova)
resource "openstack_compute_instance_v2" "web" { name = "web-01" image_name = "ubuntu-22.04" flavor_name = var.flavor key_pair = "mykey" security_groups = ["web-sg"] network { name = "mynet" } }
Netzwerk (Neutron)
resource "openstack_networking_network_v2" "net" { name = "mynet" admin_state_up = true } resource "openstack_networking_subnet_v2" "subnet" { name = "mysubnet" network_id = openstack_networking_network_v2.net.id cidr = "192.168.1.0/24" ip_version = 4 dns_nameservers = ["8.8.8.8"] }
Volume (Cinder) & anhängen
resource "openstack_blockstorage_volume_v3" "vol" { name = "data-vol" size = 50 } resource "openstack_compute_volume_attach_v2" "va" { instance_id = openstack_compute_instance_v2.web.id volume_id = openstack_blockstorage_volume_v3.vol.id }
Floating IP
resource "openstack_networking_floatingip_v2" "fip" { pool = "public" } resource "openstack_compute_floatingip_associate_v2" "fip_assoc" { floating_ip = openstack_networking_floatingip_v2.fip.address instance_id = openstack_compute_instance_v2.web.id }
Security Group
resource "openstack_networking_secgroup_v2" "web_sg" { name = "web-sg" } resource "openstack_networking_secgroup_rule_v2" "http" { direction = "ingress" ethertype = "IPv4" protocol = "tcp" port_range_min = 80 port_range_max = 80 security_group_id = openstack_networking_secgroup_v2.web_sg.id }
💡 Env-Variablen OS_* werden automatisch vom OpenStack Provider gelesen

Module & Provider
Modul aufrufen
module "webserver" { source = "./modules/openstack-vm" # oder Registry: source = "git::https://github.com/org/module.git" version = "1.0.0" flavor = var.flavor image = "ubuntu-22.04" }
Provider-Versionen sperren
terraform providers lock \ -platform=linux_amd64
Modul-Ausgaben referenzieren
output "web_ip" { value = module.webserver.ip_address }
Iteration über mehrere Ressourcen
# count — Index-basiert: resource "openstack_compute_instance_v2" "vms" { count = 3 name = "vm-${count.index}" } # for_each — Key-basiert (nicht mit count mischbar): resource "openstack_compute_instance_v2" "nodes" { for_each = toset(["web", "db", "cache"]) name = "${each.key}-node" }

Workspaces
Workspaces verwalten
terraform workspace list terraform workspace new staging terraform workspace select staging terraform workspace show terraform workspace delete staging
Workspace-Name in Config nutzen
locals { env = terraform.workspace } resource "openstack_compute_instance_v2" "web" { name = "${local.env}-web-01" }

Backends (Remote State)
S3-Backend (z.B. Ceph RGW)
terraform { backend "s3" { bucket = "terraform-state" key = "infra/terraform.tfstate" region = "us-east-1" # Für Ceph RGW: endpoint = "https://rgw.example.com" skip_credentials_validation = true skip_metadata_api_check = true force_path_style = true } }
HTTP-Backend
terraform { backend "http" { address = "https://gitlab.com/api/v4/projects/1/terraform/state/prod" lock_address = "...lock" unlock_address = "...lock" } }
State zwischen Backends migrieren
terraform init -migrate-state

Debugging & Lifecycle
Log-Level setzen

Lifecycle-Regeln im Code
resource "openstack_compute_instance_v2" "web" { # ... lifecycle { prevent_destroy = true create_before_destroy = true ignore_changes = [user_data] } }

terraform graph | dot -Tsvg > graph.svg
