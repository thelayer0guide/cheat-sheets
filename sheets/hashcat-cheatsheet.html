Grundsyntax
Allgemeines Schema
hashcat -m HASHTYPE -a ATTACKMODE \ HASHFILE WORDLIST [OPTIONEN]
Einzelner Hash direkt angeben
hashcat -m 0 -a 0 \ '5d41402abc4b2a76b9719d911017c592' \ /usr/share/wordlists/rockyou.txt
Hash-Datei (eine Hash pro Zeile)
hashcat -m 1000 -a 0 ntlm.txt rockyou.txt
Passwörter einer Hash-Datei anzeigen
hashcat -m 1000 ntlm.txt --show
Hinweis: HASHFILE kann auch ein einzelner Hash-String sein (ohne Anführungszeichen wenn kein Sonderzeichen enthalten). Bei Sonderzeichen einfache Anführungszeichen verwenden.
🗡
Angriffsmodi (-a)
-a 0 — Wordlist (Dictionary Attack)
hashcat -m 0 -a 0 hash.txt rockyou.txt
-a 1 — Combinator (zwei Wordlists kombinieren)
hashcat -m 0 -a 1 hash.txt \ words1.txt words2.txt
-a 3 — Brute-Force / Masken-Angriff
hashcat -m 0 -a 3 hash.txt '?l?l?l?l?d?d'
-a 6 — Wordlist + Maske (Suffix-Angriff)
hashcat -m 0 -a 6 hash.txt rockyou.txt '?d?d?d?d'
-a 7 — Maske + Wordlist (Präfix-Angriff)
hashcat -m 0 -a 7 hash.txt '?d?d' rockyou.txt
Empfehlung: -a 0 mit Rules (-r) ist oft effektiver als Brute-Force. Combinator (-a 1) eignet sich für Passphrasen wie "CorrectHorseBattery".
🔑
Wichtige Hash-Typen (-m)
Häufige Hashes
# -m 0 MD5 # -m 100 SHA-1 # -m 1400 SHA-256 # -m 1700 SHA-512 # -m 3200 bcrypt $2*$ # -m 500 md5crypt $1$ # -m 1800 sha512crypt $6$
Windows / Active Directory
# -m 1000 NTLM (LM:NT aus SAM/NTDS) # -m 3000 LM-Hash # -m 5500 NetNTLMv1 (Responder) # -m 5600 NetNTLMv2 (Responder) # -m 13100 Kerberoast TGS-REP RC4 # -m 18200 AS-REP Roast # -m 19600 Kerberoast AES-128 # -m 19700 Kerberoast AES-256
Netzwerk / WLAN
# -m 22000 WPA-PBKDF2-PMKID+EAPOL # -m 2500 WPA/WPA2 (veraltet) # -m 16500 JWT RS256/HS256
Anwendungen / Web
# -m 400 phpBB3 / WordPress # -m 3711 MediaWiki B type # -m 7400 sha256crypt $5$ # -m 1710 sha512($pass.$salt)
Hash-Typ automatisch erkennen
hashcat --identify hashes.txt # oder: hashcat --example-hashes | grep -A2 NTLM
📋
Wordlists & Rules
Mit Regel-Datei angreifen
hashcat -m 1000 -a 0 hash.txt rockyou.txt \ -r rules/best64.rule
Mehrere Regel-Dateien kombinieren
hashcat -m 0 -a 0 hash.txt words.txt \ -r rules/best64.rule \ -r rules/toggles1.rule
Mitgelieferte Regel-Dateien (Auswahl)
rules/best64.rule # 64 häufige Mutationen rules/rockyou-30000.rule # aus RockYou abgeleitet rules/d3ad0ne.rule # umfassend rules/dive.rule # sehr umfassend (langsam) rules/Kaonashi.rule # Fokus Zahlen-Suffix
Kandidaten auf stdout ausgeben (kein Cracking)
hashcat -a 0 --stdout words.txt \ -r rules/best64.rule | head -20
Regelformat — häufige Operatoren
# : keine Aktion (Passwort unverändert) # l alles Kleinbuchstaben # u alles Großbuchstaben # c erstes Zeichen groß # r Passwort umkehren # $1 Zeichen '1' anhängen # ^! Zeichen '!' voranstellen # D3 Zeichen an Pos 3 löschen # i3X Zeichen 'X' an Pos 3 einfügen
Tipp: best64.rule ist ein guter Startpunkt. rockyou-30000.rule erzeugt Milliarden von Kandidaten — nur bei schnellen Hashes (MD5, NTLM) sinnvoll.
🎭
Masken & Brute-Force (-a 3)
Eingebaute Zeichenklassen
?l # Kleinbuchstaben [a-z] ?u # Großbuchstaben [A-Z] ?d # Ziffern [0-9] ?s # Sonderzeichen [ !"#$%&...] ?a # ?l + ?u + ?d + ?s ?b # 0x00 - 0xff (alle Bytes)
Masken-Beispiele
# 8 Zeichen: Groß + 5× Klein + 2× Ziffer hashcat -a 3 hash.txt '?u?l?l?l?l?l?d?d' # PIN 4-stellig hashcat -a 3 hash.txt '?d?d?d?d' # Typisches Windows-PW: Wort + Jahreszahl hashcat -a 6 hash.txt words.txt '?d?d?d?d'
Benutzerdefinierter Zeichensatz (-1 bis -4)
# -1 definiert ?1 als eigene Klasse hashcat -a 3 -1 '?l?d' hash.txt '?1?1?1?1?1?1' # Nur Hex-Zeichen [0-9a-f] hashcat -a 3 -1 '0123456789abcdef' \ hash.txt '?1?1?1?1?1?1?1?1'
Inkrementeller Modus (variable Länge)
hashcat -a 3 --increment \ --increment-min 4 --increment-max 8 \ hash.txt '?a?a?a?a?a?a?a?a'
Warnung: ?a?a?a?a?a?a?a?a hat 95^8 ≈ 6,6 Billionen Kombinationen. Nur für schnelle Hashes (NTLM) auf leistungsstarker GPU sinnvoll.
🖥
GPU-Einstellungen & Performance
Verfügbare Devices anzeigen
hashcat -I # zeigt alle OpenCL/CUDA-Geräte
Bestimmtes Device auswählen
hashcat -d 1 -m 1000 -a 0 hash.txt words.txt # mehrere Devices: hashcat -d 1,2 -m 1000 -a 0 hash.txt words.txt
Workload-Profile (-w 1–4)
-w 1 # Low — Desktop bleibt nutzbar -w 2 # Default -w 3 # High — GPU Vollauslastung -w 4 # Nightmare — System kaum nutzbar
Optimierter Kernel (-O)
hashcat -O -m 1000 -a 0 hash.txt words.txt # schneller, aber max. Passwortlänge ~31
--force (Sicherheitsprüfungen überspringen)
hashcat --force -m 1000 hash.txt words.txt # nur in VMs/Containern nötig; kann # falsche Ergebnisse produzieren
Empfehlung: -O und -w 3 für maximale Geschwindigkeit in dedizierten Cracking-Umgebungen. --force nur bei Treiberproblemen in VMs.
💾
Session & Restore
Session benennen
hashcat -m 1000 -a 0 hash.txt rockyou.txt \ --session myjob
Unterbrochene Session fortsetzen
hashcat --session myjob --restore
Fortschritt während des Laufs
# Im laufenden Prozess: s # Status anzeigen p # Pause / Fortsetzen b # Bypass (Wordlist überspringen) q # Beenden (Session wird gespeichert)
Restore-Dateien verwalten
ls ~/.local/share/hashcat/sessions/ # oder je nach Installation: ls /usr/share/hashcat/sessions/
Laufzeit begrenzen
hashcat --session job1 --runtime 3600 \ -m 1000 -a 0 hash.txt words.txt # stoppt nach 3600 Sekunden (1 Stunde)
📄
Ergebnis & Output
Ergebnis in Datei schreiben
hashcat -m 1000 -a 0 hash.txt words.txt \ --outfile cracked.txt
Ausgabeformat (--outfile-format)
--outfile-format 1 # Hash --outfile-format 2 # Klartext-Passwort --outfile-format 3 # Hash:Klartext (default) --outfile-format 4 # HEX-Klartext --outfile-format 6 # Hash:HEX
Gecrackte Hashes anzeigen
hashcat -m 1000 hashes.txt --show # Ausgabe: hash:passwort
Nicht gecrackte Hashes anzeigen
hashcat -m 1000 hashes.txt --left
Potfile-Pfad anpassen
hashcat --potfile-path /tmp/my.pot \ -m 1000 -a 0 hash.txt words.txt # Potfile deaktivieren: hashcat --potfile-disable ...
Hinweis: Hashcat speichert gecrackte Hashes im Potfile (~/.local/share/hashcat/hashcat.potfile). --show liest aus dem Potfile, ohne neu zu cracken.
📊
Benchmark & Diagnose
Benchmark für einen Hash-Typ
hashcat -b -m 1000 # zeigt MH/s (Mega-Hashes pro Sekunde)
Alle Hash-Typen benchmarken
hashcat --benchmark-all # gibt Übersicht aller unterstützten # Hash-Typen mit Geschwindigkeit aus
Device-Informationen anzeigen
hashcat -I # zeigt OpenCL/CUDA-Plattformen, # GPU-Modell, VRAM, Treiber-Version
Hash-Typ eines unbekannten Hashes ermitteln
hashcat --identify unknown_hash.txt # schlägt passende -m-Werte vor
Beispiel-Hash für einen -m-Wert anzeigen
hashcat --example-hashes -m 13100 # zeigt Beispiel-Hash + Crack-Befehl
Selbsttest (Korrektheitsprüfung)
hashcat --self-test-disable # deaktivieren # Standardmäßig führt hashcat bei # jedem Start einen Selbsttest durch

Häufige Kombinationen
WPA2 PMKID / Handshake cracken
# .hc22000 (hcxdumptool/hcxpcapngtool): hashcat -m 22000 -a 0 capture.hc22000 \ rockyou.txt -r rules/best64.rule
NTLM aus SAM/NTDS (Impacket-Dump)
# Format: user:RID:LM:NT::: cut -d: -f4 ntds.dit.out > ntlm_only.txt hashcat -m 1000 -a 0 ntlm_only.txt \ rockyou.txt -r rules/best64.rule -O
Kerberoasting TGS-REP Tickets cracken
# Tickets aus Impacket GetUserSPNs.py: hashcat -m 13100 -a 0 kerb_tickets.txt \ rockyou.txt -r rules/best64.rule
AS-REP Roasting (kein Kerberos Pre-Auth)
hashcat -m 18200 -a 0 asrep_hashes.txt \ rockyou.txt -r rules/best64.rule
NetNTLMv2 aus Responder cracken
# Responder-Logs: /var/log/Responder/ hashcat -m 5600 -a 0 netntlmv2.txt \ rockyou.txt -r rules/best64.rule
LinkedIn / SHA-1 unsalted (Leaks)
hashcat -m 100 -a 0 linkedin.txt \ rockyou.txt -r rules/rockyou-30000.rule -O
Hinweis: hcxdumptool + hcxpcapngtool sind für WPA2-Captures empfohlen. Das ältere .cap-Format benötigt -m 2500 (veraltet ab hashcat 6.0).
