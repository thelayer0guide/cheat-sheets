Installation & Vorbereitung
Kolla-Ansible installieren (Deployment-Node)
pip install kolla-ansible # oder aus Git für bleeding-edge: pip install git+https://opendev.org/openstack/kolla-ansible # Konfigurationsdateien anlegen: mkdir -p /etc/kolla cp $(kolla-ansible --version-data 2>/dev/null | grep share | awk '{print $2}')/kolla-ansible/etc/kolla/globals.yml /etc/kolla/ # Einfacher mit pip-Pfad: cp /usr/local/share/kolla-ansible/etc_examples/kolla/globals.yml /etc/kolla/ cp /usr/local/share/kolla-ansible/etc_examples/kolla/passwords.yml /etc/kolla/
Passwörter generieren
kolla-genpwd # oder einzelnes Passwort lesen: grep keystone_admin_password /etc/kolla/passwords.yml
📋
Inventory — Multinode
Inventory-Template kopieren
cp /usr/local/share/kolla-ansible/ansible/inventory/multinode ./inventory cp /usr/local/share/kolla-ansible/ansible/inventory/all-in-one ./inventory
multinode Inventory — Struktur
[control] controller01 ansible_user=ubuntu ansible_become=true controller02 controller03 [network] controller01 controller02 [compute] compute01 compute02 [storage] storage01 storage02 [monitoring] controller01 [deployment] localhost ansible_connection=local
⚙️
globals.yml — Schlüsseloptionen
Basis-Konfiguration
# /etc/kolla/globals.yml kolla_base_distro: "ubuntu" # centos|ubuntu|debian openstack_release: "2024.1" # Zed, 2023.x, 2024.x kolla_internal_vip_address: "10.0.0.100" kolla_external_vip_address: "192.168.1.100" network_interface: "eth0" neutron_external_interface: "eth1" # kein IP! kolla_enable_tls_internal: "yes" kolla_enable_tls_external: "yes"
Dienste aktivieren/deaktivieren
enable_openstack_core: "yes" # Nova,Neutron,Glance,Cinder enable_ceph: "yes" enable_heat: "yes" enable_octavia: "yes" enable_barbican: "yes" enable_designate: "yes" enable_prometheus: "yes" enable_grafana: "yes" enable_haproxy: "yes" # interner LB nova_compute_virt_type: "kvm" # kvm|qemu
Docker & Storage
docker_registry: "registry.example.com" docker_registry_insecure: "no" ceph_nova_user: "cinder" cinder_backend_ceph: "yes" nova_backend_ceph: "yes"

Deploy-Workflow
Vollständiger Deploy-Ablauf
# 1. Ansible-Verbindung testen kolla-ansible -i ./inventory ping # 2. SSH + Dependencies auf Nodes installieren kolla-ansible -i ./inventory bootstrap-servers # 3. Konfiguration vorab prüfen kolla-ansible -i ./inventory prechecks # 4. Container-Images pullen kolla-ansible -i ./inventory pull # 5. OpenStack deployen kolla-ansible -i ./inventory deploy # 6. Admin-Credentials generieren kolla-ansible -i ./inventory post-deploy source /etc/kolla/admin-openrc.sh
💡 --tags nova,neutron — nur bestimmte Dienste deployen
🔄
Reconfigure, Upgrade & Destroy
Konfiguration anwenden (ohne Re-Deploy)
# Konfigurationsänderung in globals.yml → anwenden: kolla-ansible -i ./inventory reconfigure # Nur bestimmte Dienste: kolla-ansible -i ./inventory reconfigure --tags nova
Upgrade auf neue OpenStack-Version
# globals.yml: openstack_release auf neue Version setzen kolla-ansible -i ./inventory pull kolla-ansible -i ./inventory prechecks kolla-ansible -i ./inventory upgrade
Dienste neu starten & zerstören
kolla-ansible -i ./inventory stop # Container stoppen kolla-ansible -i ./inventory deploy # neu starten kolla-ansible -i ./inventory destroy --yes-i-really-really-mean-it
🔧
Konfiguration überschreiben
Service-spezifische Overrides
# Eigene config-Snippets unter: /etc/kolla/nova/nova.conf /etc/kolla/neutron/neutron.conf /etc/kolla/keystone/keystone.conf # Beispiel nova Override: # /etc/kolla/nova/nova.conf [DEFAULT] cpu_allocation_ratio = 4.0 ram_allocation_ratio = 1.5
Extra-Variablen übergeben
kolla-ansible -i ./inventory deploy \ -e "nova_compute_virt_type=qemu" \ -e "@/path/to/extra_vars.yml"
🐛
Debugging & Troubleshooting
Verbose & Check-Mode
kolla-ansible -i ./inventory deploy -v # verbose kolla-ansible -i ./inventory deploy -vvv # debug kolla-ansible -i ./inventory deploy --check # dry-run kolla-ansible -i ./inventory deploy --diff # config diff
Container-Logs & Status prüfen
# Auf einem Node: docker ps -a # alle Container docker logs nova_compute --tail 50 docker exec -it nova_compute bash docker inspect nova_compute | jq '.[0].State' # Kolla-Log-Verzeichnis: ls /var/log/kolla/nova/
Einzelnen Node targetieren
kolla-ansible -i ./inventory deploy --limit compute01 kolla-ansible -i ./inventory deploy --limit "compute01,compute02"
💡 kolla-ansible certificates — TLS-Zertifikate für API-Endpunkte generieren
