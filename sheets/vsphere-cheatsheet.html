vmkfstools — VMDK & Storage
VMDK erstellen & klonen
vmkfstools -c 100G /vmfs/volumes/ds01/myvm/myvm.vmdk vmkfstools -i source.vmdk dest.vmdk # klonen vmkfstools -i source.vmdk -d thin dest.vmdk # thin clone vmkfstools -i source.vmdk -d zeroedthick dest.vmdk
VMDK-Operationen
vmkfstools -X 200G myvm.vmdk # vergrößern vmkfstools -K myvm.vmdk # defragmentieren / shrink vmkfstools -e myvm.vmdk # Extents auflisten vmkfstools -P /vmfs/volumes/ds01/ # VMFS-Info vmkfstools -v 10 -C vmfs6 /vmfs/devices/disks/naa.XXX:1 # VMFS formatieren
RDM (Raw Device Mapping)
vmkfstools -r /vmfs/devices/disks/naa.XXX rdm.vmdk # physical RDM vmkfstools -z /vmfs/devices/disks/naa.XXX rdm.vmdk # virtual RDM

REST API
Session-Token
curl -ks -X POST \ "https://vc01/api/session" \ -H "Authorization: Basic $(echo -n 'admin:pass' | base64)" \ -H "Content-Type: application/json" # Response: "token-string" → für alle weiteren Requests
VMs per REST API
TOKEN="xxxx" # Alle VMs: curl -ks "https://vc01/api/vcenter/vm" \ -H "vmware-api-session-id: $TOKEN" | jq . # VM-Details: curl -ks "https://vc01/api/vcenter/vm/vm-123" \ -H "vmware-api-session-id: $TOKEN" | jq . # VM einschalten: curl -ks -X POST "https://vc01/api/vcenter/vm/vm-123/power?action=start" \ -H "vmware-api-session-id: $TOKEN"
DCLI (Datacenter CLI)
