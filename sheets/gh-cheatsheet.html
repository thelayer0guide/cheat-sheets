Authentication
Interactive login (browser / token)
gh auth login gh auth login --hostname github.company.com # GHES
Login mit Token (non-interactive)
echo $GH_TOKEN | gh auth login --with-token
Aktuellen Auth-Status prüfen
gh auth status gh auth status --show-token
Token ausgeben (für Skripte)
gh auth token
Token-Scopes erweitern
gh auth refresh --scopes read:org,write:packages
Ausloggen
gh auth logout
Env-Variable: GH_TOKEN oder GITHUB_TOKEN werden automatisch erkannt — kein Login nötig.

Repository
Repo klonen
gh repo clone owner/repo gh repo clone owner/repo -- --depth=1
Neues Repo erstellen
gh repo create my-project --public --clone gh repo create my-project --private --description "Desc"
Repo forken
gh repo fork owner/repo --clone gh repo fork --remote-name upstream
Repo-Infos anzeigen
gh repo view gh repo view owner/repo --web
Repos auflisten
gh repo list # eigene gh repo list owner --limit 50 gh repo list --fork --language go
Repo archivieren / löschen
gh repo archive owner/repo gh repo delete owner/repo --yes
Repo-Topics setzen
gh repo edit --add-topic devops,cli gh repo edit --description "New description"

Pull Requests
PR erstellen
gh pr create --title "Fix bug" --body "Details" gh pr create --fill # aus Commits gh pr create --draft --web
PR-Liste anzeigen
gh pr list gh pr list --state closed --author @me gh pr list --label bug --json number,title
PR anzeigen
gh pr view 42 gh pr view --web
PR auschecken
gh pr checkout 42 gh pr checkout feature-branch
PR mergen
gh pr merge 42 --squash gh pr merge --rebase --delete-branch gh pr merge --auto --squash
PR schließen / wieder öffnen
gh pr close 42 gh pr reopen 42
PR reviewen
gh pr review 42 --approve gh pr review 42 --request-changes --body "Needs work" gh pr review 42 --comment --body "LGTM"
PR-Status (CI-Checks)
gh pr checks 42 gh pr checks --watch
--fill: Titel und Body werden automatisch aus den Commit-Messages des Branches befüllt.

Issues
Issue erstellen
gh issue create --title "Bug: crash" --body "Steps..." gh issue create --label bug,priority:high gh issue create --assignee @me --milestone "v1.0"
Issues auflisten
gh issue list gh issue list --state closed --label enhancement gh issue list --assignee @me --limit 20
Issue anzeigen
gh issue view 12 gh issue view 12 --web
Issue schließen / öffnen
gh issue close 12 --reason completed gh issue reopen 12
Issue zuweisen
gh issue edit 12 --add-assignee alice,bob gh issue edit 12 --remove-assignee alice
Label hinzufügen / entfernen
gh issue edit 12 --add-label bug gh issue edit 12 --remove-label wontfix
Kommentar hinzufügen
gh issue comment 12 --body "Fixed in #45"

GitHub Actions
Workflow-Runs auflisten
gh run list gh run list --workflow ci.yml --limit 10 gh run list --status failure --branch main
Run-Details anzeigen
gh run view 123456789 gh run view --log
Run live verfolgen
gh run watch 123456789 gh run watch # letzten Run wählen
Run erneut starten
gh run rerun 123456789 gh run rerun --failed # nur fehlgeschlagene Jobs
Run abbrechen
gh run cancel 123456789
Workflow manuell triggern
gh workflow run deploy.yml gh workflow run deploy.yml --ref main gh workflow run deploy.yml -f env=prod
Workflow-Artefakte herunterladen
gh run download 123456789 gh run download 123456789 --name build-output gh run download --dir ./artifacts
Workflows auflisten / aktivieren
gh workflow list gh workflow enable ci.yml gh workflow disable ci.yml
Logs streamen: gh run watch && gh run view --log gibt den vollständigen Log nach Abschluss aus.

SSH & GPG Keys
SSH-Keys auflisten
gh ssh-key list
SSH-Key hinzufügen
gh ssh-key add ~/.ssh/id_ed25519.pub --title "Work laptop" gh ssh-key add ~/.ssh/id_ed25519.pub --type signing
SSH-Key löschen
gh ssh-key delete KEY_ID --yes
GPG-Keys auflisten
gh gpg-key list
GPG-Key hinzufügen
gpg --armor --export KEY_ID | gh gpg-key add -
GPG-Key löschen
gh gpg-key delete KEY_ID --yes
Signing-Keys: SSH-Keys können seit gh 2.33 auch als Commit-Signing-Keys hinterlegt werden (--type signing).

gh alias
Alias erstellen
gh alias set prc 'pr create --fill --web' gh alias set co 'pr checkout'
Shell-Alias (bash/sh)
gh alias set --shell prl 'gh pr list | fzf | cut -f1 | xargs gh pr checkout'
Alle Aliases anzeigen
gh alias list
Alias löschen
gh alias delete prc
Alias-Scope: Aliases werden in ~/.config/gh/config.yml gespeichert und gelten für alle Repos.

Output & Formatierung
JSON-Ausgabe
gh pr list --json number,title,state gh issue list --json number,title,labels
Mit jq filtern
gh pr list --json number,title --jq '.[].title' gh run list --json status --jq '[.[] | select(.status=="completed")]'
Go-Template
gh pr list --template '{{range .}}#{{.number}} {{.title}}{{"\n"}}{{end}}'
API direkt aufrufen
gh api repos/{owner}/{repo}/topics gh api graphql -f query='{ viewer { login } }' gh api --paginate /repos/{owner}/{repo}/issues
API POST / PATCH
gh api --method POST /repos/{owner}/{repo}/issues \ -f title="New issue" -f body="Details"
Paginierung
gh api --paginate /orgs/myorg/repos \ --jq '.[].full_name'
Felder erkunden: Verfügbare JSON-Felder via gh pr view --help ermitteln. --json ohne Feldliste gibt einen Fehler zurück.

Config & Extensions
Konfiguration anzeigen / setzen
gh config list gh config set editor vim gh config set browser firefox gh config set git_protocol ssh
Konfiguration pro Host
gh config set -h github.company.com editor vim
Extensions installieren
gh extension install github/gh-copilot gh extension install dlvhdr/gh-dash gh extension install mislav/gh-branch
Extensions verwalten
gh extension list gh extension upgrade --all gh extension remove github/gh-copilot
Eigene Extension erstellen
gh extension create my-ext --precompiled go
Config-Datei: ~/.config/gh/config.yml — enthält Hosts, Aliases, Einstellungen.

Workflows & Tipps
PR für aktuellen Branch anlegen
git push -u origin HEAD && gh pr create --fill
Offene Review-Requests anzeigen
gh pr list --search "is:open review-requested:@me"
Issue im Browser öffnen
gh issue view 42 --web
Letzten fehlgeschlagenen Run-Log
gh run list --status failure --limit 1 \ --json databaseId --jq '.[0].databaseId' | \ xargs gh run view --log-failed
Alle PRs eines Users suchen
gh search prs --author alice --state open gh search issues --label bug --repo owner/repo
Repository-Secrets
gh secret list gh secret set MY_SECRET gh secret set MY_SECRET --body "value" gh secret delete MY_SECRET
gh search: gh search prs, gh search issues und gh search repos unterstützen die vollständige GitHub-Suchsyntax.
