Status & Überblick
Wichtigste Befehle
chronyc tracking # Synchronisationsstatus chronyc sources -v # NTP-Quellen verbose chronyc sourcestats # Quell-Statistiken chronyc ntpdata # Detaillierte NTP-Daten chronyc activity # Online-Status der Quellen
tracking — Ausgabe verstehen
Reference ID: Aktive NTP-Quelle (IP/Name) Stratum: Schicht (1=Atom, 2=Sync von Stratum 1) Ref time (UTC): Zeitstempel des letzten Updates System time: Offset zur NTP-Zeit Last offset: Letzter Versatz RMS offset: Durchschnittlicher Versatz Frequency: Drift-Korrektur (ppm) Leap status: Normal / Schaltsekunde
sources — Ausgabe verstehen
# Spalten: # M = Mode: * (synced), + (candidate), - (rejected) # S = State: * aktive Quelle # Name/IP # Stratum # Poll: Abfrageintervall (2^n Sekunden) # Reach: Oktal-Bitmaske (377 = 8/8 erfolgreich) # LastRx: Letzte Antwort in Sekunden # Last sample: Offset ±Fehler chronyc sources -v
⚙
/etc/chrony.conf
Server-Konfiguration
# NTP-Server (iburst = schnelle erste Sync): server 0.de.pool.ntp.org iburst server 1.de.pool.ntp.org iburst server 2.de.pool.ntp.org iburst prefer # Lokales NTP-Server-Setup: pool ntp.example.com iburst maxsources 4 # Als NTP-Server dienen: allow 10.0.0.0/8 allow 192.168.0.0/16
Wichtige Parameter
driftfile /var/lib/chrony/drift makestep 1.0 3 # Sprung wenn offset > 1s (max 3x) rtcsync # Hardware-Clock sync logdir /var/log/chrony log tracking sources statistics maxupdateskew 100.0 # Max. Skew-Änderung stratumweight 0.001 # Stratum-Gewichtung keyfile /etc/chrony.keys
OpenStack-spezifisch
# Controller (NTP-Server für Nodes): server pool.ntp.org iburst allow 10.0.0.0/8 # Management-Netz local stratum 10 # Fallback ohne externes NTP # Compute/Storage Nodes: server controller.example.com iburst prefer # Keine allow-Direktive auf non-Controller
🔧
chronyc — Interaktive Befehle
Quellen verwalten
chronyc add server ntp.example.com iburst chronyc delete ntp.example.com chronyc online # Alle Quellen online setzen chronyc offline # Alle Quellen offline chronyc burst 4/4 # Burst-Modus: 4 Samples
Manuelle Zeitkorrektur
# Sofortiger Sprung: chronyc makestep # Manuell erzwingen (als Root): chronyd -q # Einmalige Synchronisation + Exit chronyd -Q # Nur anzeigen, kein Setzen # Drift manuell prüfen: chronyc waitsync 10 0.01 0.01 1
systemd & Service
systemctl enable --now chronyd systemctl status chronyd journalctl -u chronyd -f # Zeitsynchronisation gesamtsystem: timedatectl status timedatectl set-ntp true timedatectl set-timezone Europe/Berlin
🔍
Diagnose & Troubleshooting
Zeitprobleme diagnostizieren
# Offset anzeigen: chronyc tracking | grep "System time" # NTP-Konnektivität testen: ntpdate -d -u pool.ntp.org ntpstat # chrony-Log analysieren: tail -f /var/log/chrony/tracking.log
Häufige Probleme
# "No NTP sources" — Firewall-Check: firewall-cmd --add-service=ntp --permanent firewall-cmd --reload # Port 123/UDP öffnen: iptables -A OUTPUT -p udp --dport 123 -j ACCEPT # "Leap status: Not synchronised": # → makestep ausführen oder iburst warten
💡 OpenStack: exakte Zeitsynchonisation ist kritisch für Keystone-Tokens und Ceph-OSD-Daten
