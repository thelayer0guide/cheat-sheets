Grundbefehle
GET-Request
curl https://example.com curl -s https://api.example.com/users # silent curl -L https://example.com # Redirects folgen curl -o file.html https://example.com # in Datei curl -O https://example.com/file.zip # Original-Name curl -I https://example.com # Nur Header
HTTP-Methoden
# POST mit JSON: curl -X POST \ -H "Content-Type: application/json" \ -d '{"name":"Alice","age":30}' \ https://api.example.com/users # PUT: curl -X PUT -d '{"name":"Alice"}' \ https://api.example.com/users/1 # DELETE: curl -X DELETE https://api.example.com/users/1 # PATCH: curl -X PATCH -d '{"active":false}' URL
Header senden
curl -H "Authorization: Bearer TOKEN" URL curl -H "Accept: application/json" \ -H "X-Request-ID: abc123" URL # Response-Header anzeigen: curl -D - https://example.com # Header + Body curl -D headers.txt https://example.com
🔒
Authentifizierung
Auth-Methoden
# Basic Auth: curl -u user:password URL curl --user user:password URL # Bearer Token: curl -H "Authorization: Bearer $TOKEN" URL # API-Key im Header: curl -H "X-API-Key: $API_KEY" URL # Digest Auth: curl --digest -u user:pass URL
OpenStack Keystone Token
# Token holen: TOKEN=$(curl -s -X POST \ -H "Content-Type: application/json" \ -d '{ "auth":{"identity":{"methods":["password"], "password":{"user":{"name":"admin", "domain":{"name":"Default"},"password":"SECRET"}}}, "scope":{"project":{"name":"admin","domain":{"name":"Default"}}}}}' \ http://keystone:5000/v3/auth/tokens \ -D - | grep "X-Subject-Token" | awk '{print $2}') curl -H "X-Auth-Token: $TOKEN" http://nova:8774/v2.1/servers
🔐
TLS / SSL-Optionen
Zertifikate
# SSL-Fehler ignorieren (Test only!): curl -k https://self-signed.example.com # Eigene CA nutzen: curl --cacert /etc/ssl/ca.crt https://example.com # Client-Zertifikat (mutual TLS): curl --cert client.crt --key client.key \ https://example.com # TLS-Version erzwingen: curl --tlsv1.3 https://example.com
TLS debuggen
curl -v https://example.com 2>&1 | grep -E 'SSL|TLS|cert' curl --verbose --silent https://example.com 2>&1
📦
Daten & Upload
Form-Daten / File-Upload
# Form-POST: curl -d "user=alice&pass=secret" URL # Multipart/Form-Data: curl -F "file=@image.png" \ -F "name=myphoto" URL # JSON aus Datei: curl -X POST \ -H "Content-Type: application/json" \ -d @payload.json URL
Download / Resume
# Fortgesetzter Download: curl -C - -O https://example.com/large.iso # Parallel mit Fortschritt: curl -Z URL1 URL2 URL3 # Rate-Limit: curl --limit-rate 1M -O https://example.com/file
⏱
Timing & Proxy
Timing-Ausgabe
curl -w "\n DNS: %{time_namelookup}s Connect: %{time_connect}s TLS: %{time_appconnect}s Start: %{time_starttransfer}s Total: %{time_total}s Code: %{http_code} Size: %{size_download} bytes\n" \ -s -o /dev/null URL
Proxy
curl -x http://proxy:3128 URL curl --proxy-user user:pass -x proxy:3128 URL # SOCKS5: curl --socks5 127.0.0.1:1080 URL # Proxy umgehen: curl --noproxy "localhost,127.0.0.1" URL
Timeouts & Retry
curl --connect-timeout 5 URL # Connect-Timeout curl --max-time 30 URL # Gesamt-Timeout curl --retry 3 --retry-delay 5 URL curl --retry 3 --retry-connrefused URL
📝
curl-Konfiguration
~/.curlrc (Defaults)
# ~/.curlrc — persistente Optionen: silent location connect-timeout = 10 max-time = 30 user-agent = "MyCLI/1.0" # Kein SSL-Verify (nur intern!): insecure
Nützliche Aliase
# JSON-API-Kurzform: alias jcurl='curl -s -H "Content-Type: application/json" -H "Accept: application/json"' # Mit pretty-print via jq: curl -s https://api.example.com | jq '.'
💡 curl -v zeigt alle Request/Response-Header — ideal zum Debuggen von API-Problemen
