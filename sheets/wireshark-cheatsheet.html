Display-Filter Syntax
IP-Adresse (Quelle oder Ziel)
ip.addr == 192.168.1.1
Quell-IP / Ziel-IP
ip.src == 10.0.0.1 ip.dst == 10.0.0.2
TCP- / UDP-Port
tcp.port == 443 udp.port == 53
Protokoll-Filter
http dns tls icmp ssh ftp smtp arp
Protokolle ausblenden (NOT-Verknüpfung)
!arp && !dns
Nur SYN-Pakete (kein SYN-ACK)
tcp.flags.syn == 1 && tcp.flags.ack == 0
Große Pakete filtern
frame.len > 1000
Tipp: Display-Filter wirken nur auf die Anzeige, nicht auf das Capture. Logik: &&, ||, !. Vergleich: ==, !=, contains, matches.

Capture-Filter Syntax
Host filtern
host 192.168.1.1
Netzwerk / Subnetz
net 192.168.0.0/24
Port / TCP-Port
port 80 tcp port 443
Port ausschließen
not port 22
Nur Source-Host
src host 10.0.0.1
Port-Bereich
portrange 1000-2000
Nur ICMP
icmp
Capture-Filter (BPF-Syntax) werden vor dem Capture angewendet und reduzieren den Buffer-Verbrauch. Sie sind effizienter als Display-Filter.

Protokoll-spez. Filter
HTTP-Methode / Response-Code
http.request.method == "GET" http.response.code == 200
DNS-Query enthält String
dns.qry.name contains "evil"
TLS Client Hello
tls.handshake.type == 1
FTP-Passwort / SMTP-Auth
ftp.request.command == "PASS" smtp.req.command == "AUTH"
SSH-Version
ssh.protocol contains "OpenSSH"
TCP-Retransmissions
tcp.analysis.retransmission
Protokoll-Felder sind hierarchisch: http.request.method, dns.qry.name, tls.handshake.type. Tab-Vervollständigung im Filter-Feld hilft beim Erkunden.

tshark (CLI)
Live-Capture auf Interface
tshark -i eth0
pcap-Datei lesen
tshark -r capture.pcap
Capture in Datei schreiben
tshark -i eth0 -w out.pcap
Display-Filter anwenden
tshark -r file.pcap -Y "http"
Felder extrahieren (-T fields)
tshark -r file.pcap -T fields \ -e ip.src -e http.host
HTTP-Statistik (Tree-Format)
tshark -r file.pcap -z http,tree
Output nach Stichwort durchsuchen
tshark -r file.pcap | grep "password"
tshark ist die Kommandozeilen-Version von Wireshark. Mit -T json lässt sich die Ausgabe maschinell weiterverarbeiten.

Decryption
TLS — Pre-Master-Secret Log setzen
Edit → Preferences → Protocols → TLS → (Pre)-Master-Secret log filename
Browser-Keys exportieren (Linux/macOS)
SSLKEYLOGFILE=/tmp/keys.log firefox SSLKEYLOGFILE=/tmp/keys.log chromium
WPA2 entschlüsseln — Einstellungen
Edit → Preferences → Protocols → IEEE 802.11 → Decryption keys
WPA2-Key-Format
wpa-pwd:Passwort:SSID # Klartext wpa-psk:HEX_PSK_64_ZEICHEN # Hex
Kerberos — Keytab-Datei hinterlegen
Edit → Preferences → Protocols → Kerberos → Keytab-Datei
SSLKEYLOGFILE funktioniert mit Firefox, Chromium und Node.js (via NODE_OPTIONS=--tls-keylog). Die Datei direkt in Wireshark als TLS-Keylog eintragen.

Capture-Optionen & pcap-Tools
Interface-Auswahl & Promiscuous-Mode
Capture → Options # GUI tshark -D # Interfaces auflisten
Ring-Buffer (Größe / Zeitlimit)
Capture → Options → Output → Ring buffer: 10 files × 100 MB
pcap-Metadaten anzeigen
capinfos file.pcap
Mehrere pcap-Dateien zusammenführen
mergecap -w out.pcap a.pcap b.pcap
Pakete ausschneiden (Nummernbereich)
editcap -r in.pcap out.pcap 1-1000
Zeitstempel verschieben (in Sekunden)
editcap -t 3600 in.pcap out.pcap
System-Capture mit tcpdump
tcpdump -i eth0 -w out.pcap
capinfos, mergecap, editcap und tshark werden mit dem Wireshark-Paket mitgeliefert und sind im PATH verfügbar.

Quick Reference — Häufige Filter
Klartext-Credentials (mehrere Protokolle)
ftp || http || smtp || pop || imap
DNS-Exfiltration (lange Subdomain-Labels)
dns && dns.qry.name.len > 40
SYN-Scan erkennen
tcp.flags == 0x002 && !tcp.analysis.retransmission
ARP-Spoofing erkennen
arp.duplicate-address-detected
Große Transfers (nahe MTU)
frame.len > 1400 && ip
HTTP POST mit "password"-Feld
http.request.method == "POST" && http contains "password"
Hinweis: Display-Filter sind case-sensitiv. Der contains-Operator prüft auf Bytes/Strings im Payload. matches akzeptiert Perl-kompatible Regex (PCRE).
