Grundlagen & Syntax
Einfacher Crack (auto-detect Format)
john hash.txt
Wordlist-Angriff
john --wordlist=/usr/share/wordlists/rockyou.txt hash.txt
Format explizit angeben
john --format=FORMAT hash.txt
Gecrackte Passwörter anzeigen
john --show hash.txt john --show --format=FORMAT hash.txt
Alle unterstützten Formate auflisten
john --list=formats john --list=formats | grep -i sha
Hinweis: John speichert gecrackte Passwörter in ~/.john/john.pot. --show liest daraus, ohne erneut zu cracken. Das Pot-File kann mit --pot=DATEI angepasst werden.
🔑
Hash-Formate (--format)
Linux / Unix Hashes
# md5crypt — Linux MD5 ($1$) # sha256crypt — SHA-256 ($5$) # sha512crypt — SHA-512 ($6$) # bcrypt — bcrypt ($2a$, $2b$)
Windows / Active Directory
# nt — Windows NTLM # lm — Windows LM (veraltet) # mscash2 — Domain Cached Credentials v2 # krb5tgs — Kerberoasting-Hashes
Archiv / Dokument
# zip — ZIP-Archiv # rar — RAR-Archiv # office — MS-Office-Dokumente # pdf — PDF-Passwort # keepass — KeePass KDBX
Beispiel: Format-spezifischer Crack
john --format=nt --wordlist=rockyou.txt hashes.txt john --format=sha512crypt hash.txt
Tipp: --list=formats --format=dynamic zeigt alle dynamischen Formate. Bei unbekannten Hashes hilft hash-identifier oder hashid beim Erkennen des Typs.
📋
Wordlist-Angriffe
Standard-Wordlist
john --wordlist=rockyou.txt hash.txt
Mit Standard-Regeln
john --wordlist=rockyou.txt --rules hash.txt
Mit benannten Regeln
john --wordlist=rockyou.txt --rules=Jumbo hash.txt john --wordlist=rockyou.txt --rules=All hash.txt
Wordlist von stdin
john --wordlist=- hash.txt < words.txt
Häufige Wordlist-Pfade
/usr/share/wordlists/rockyou.txt /usr/share/john/password.lst /usr/share/seclists/Passwords/
Empfehlung: --rules=Jumbo erweitert jedes Wort um hunderte Mutationen (Großschreibung, Zahlenanhänge, Leet-Speak). Deutlich effektiver als reiner Wordlist-Angriff.
⚙
Regel-basierte Angriffe
Eingebaute Regel-Sets
# --rules=Single Single-Crack-Regeln # --rules=Wordlist Standard-Wordlist-Regeln # --rules=Extra Erweiterte Mutationen # --rules=Jumbo JtR-Jumbo-Regeln # --rules=KoreLogic Komplexe Transformationen # --rules=All Alle Regel-Sets kombiniert
Eigene Regeln in john.conf
# ~/.john/john.conf öffnen und ergänzen: [List.Rules:Custom] # c — erstes Zeichen Großbuchstabe # Az"123" — Suffix "123" anhängen # l — alles Kleinbuchstaben # r — Wort umkehren
Benutzerdefinierte Regeln verwenden
john --wordlist=rockyou.txt \ --rules=Custom hash.txt
Hinweis: Die Konfigurationsdatei liegt standardmäßig unter /etc/john/john.conf oder ~/.john/john.conf. Benutzer-Einstellungen haben Vorrang.
🔓
Brute-Force & Incremental
Vollständiger Brute-Force
john --incremental hash.txt
Vordefinierte Zeichensätze
john --incremental=Alpha hash.txt # a-z A-Z john --incremental=Digits hash.txt # 0-9 john --incremental=Alnum hash.txt # a-z A-Z 0-9
Masken-Angriff
john --mask='?a?a?a?a?a?a' hash.txt john --mask='?l?l?d?d' hash.txt
Masken-Zeichenklassen
?l # Kleinbuchstaben [a-z] ?u # Großbuchstaben [A-Z] ?d # Ziffern [0-9] ?s # Sonderzeichen ?a # alle Zeichen (?l+?u+?d+?s)
Warnung: Incremental-Mode ist extrem langsam bei langen Passwörtern. Für einfache Hashes (MD5, NTLM) Hashcat mit GPU nutzen — John ist CPU-optimiert.
🔧
Hash-Extraktion (Helper-Tools)
Linux-System-Hashes
unshadow /etc/passwd /etc/shadow > hashes.txt
Archiv-Passwörter extrahieren
zip2john archive.zip > zip.hash rar2john archive.rar > rar.hash
Dokument-Passwörter extrahieren
pdf2john file.pdf > pdf.hash office2john doc.docx > office.hash
Schlüssel & Datenbank-Passwörter
keepass2john keepass.kdbx > kp.hash ssh2john id_rsa > ssh.hash bitlocker2john image.img > bl.hash
Extrahierten Hash direkt cracken
zip2john secret.zip > z.hash john --wordlist=rockyou.txt z.hash
Hinweis: Alle *2john-Tools sind in der Jumbo-Edition enthalten. Der Pfad liegt typischerweise unter /usr/share/john/ oder ist direkt im PATH verfügbar.
💾
Session-Management
Session benennen und starten
john --session=mysession \ --wordlist=rockyou.txt hash.txt
Unterbrochene Session fortsetzen
john --restore=mysession john --restore # letzte Session
Aktuellen Fortschritt anzeigen
john --status john --status=mysession
Session-Dateien verwalten
ls ~/.john/*.rec # alle Sessions # Ctrl+C → Session wird automatisch # gespeichert und kann fortgesetzt werden
Tipp: Sessions werden als .rec-Dateien in ~/.john/ gespeichert. Bei langen Cracking-Jobs immer eine Session benennen — so lässt sich die Arbeit jederzeit unterbrechen und fortsetzen.
🖧
Distributed Cracking
Keyspace auf mehrere Nodes aufteilen
john --node=1/3 --wordlist=rockyou.txt hash.txt john --node=2/3 --wordlist=rockyou.txt hash.txt john --node=3/3 --wordlist=rockyou.txt hash.txt
MPI-Unterstützung (Jumbo Edition)
mpirun -n 4 john --wordlist=rockyou.txt hash.txt # 4 parallele Prozesse auf lokalem System
Node-Aufteilung — Konzept
# Node 1/3 → crackt Keyspace 0–33% # Node 2/3 → crackt Keyspace 33–66% # Node 3/3 → crackt Keyspace 66–100% # Alle Nodes laufen unabhängig, kein # gemeinsamer Netzwerkstatus nötig
Tipp: --node funktioniert für alle Angriffsmodi (Wordlist, Incremental, Mask). Pot-Dateien sollten nach Abschluss aller Nodes zusammengeführt werden.
📄
Output & Wordlist-Generierung
Ergebnisse in Datei speichern
john --show hash.txt > cracked.txt john --show=left hash.txt # ungecrackte
Eigene Pot-Datei verwenden
john --pot=custom.pot \ --wordlist=rockyou.txt hash.txt
Passwort-Kandidaten ausgeben (kein Crack)
john --stdout --incremental | head -20 john --stdout --wordlist=rockyou.txt \ --rules | head -50
Charset-Datei für Incremental erzeugen
john --make-charset=custom.chr # Erzeugt .chr-Datei aus gecr. Passwörtern # für optimierten Incremental-Mode
Hinweis: --stdout mit Pipe zu hashcat --stdin kombinierbar: John generiert Kandidaten mit Regeln, Hashcat crackt GPU-beschleunigt.
🚀
Quick Reference
Linux-System-Hashes cracken
unshadow /etc/passwd /etc/shadow | \ john --wordlist=rockyou.txt --stdin
NTLM-Hashes (Windows)
john --format=nt --wordlist=rockyou.txt hashes.txt john --format=nt --show hashes.txt
Kerberoast-Hashes (AD)
john --format=krb5tgs \ --wordlist=rockyou.txt tgs.txt
ZIP-Archiv knacken
zip2john secret.zip > z.hash john --wordlist=rockyou.txt --rules z.hash john --show z.hash
SSH-Schlüssel (passphrase)
ssh2john id_rsa > ssh.hash john --wordlist=rockyou.txt --rules=Jumbo ssh.hash
Format-Auto-Detection
john hash.txt # --format=auto ist Standard # John erkennt viele Formate automatisch # bei Problemen: --format=FORMAT angeben
Merksatz: *2john → Hash extrahieren · john --wordlist + --rules → cracken · john --show → Ergebnisse anzeigen.
