🖥
virsh — VMs auflisten & Status
VMs auflisten
virsh list # laufende VMs virsh list --all # alle VMs (auch gestoppte) virsh list --state-shutoff virsh dominfo myvm # VM-Details virsh domstate myvm virsh domstats myvm # CPU/Mem/I/O Statistiken
VM starten / stoppen
virsh start myvm # starten virsh shutdown myvm # ACPI graceful shutdown virsh destroy myvm # sofort stoppen (hard) virsh reboot myvm virsh reset myvm # hard reset virsh suspend myvm virsh resume myvm
Autostart & Löschen
virsh autostart myvm # beim Host-Boot starten virsh autostart --disable myvm virsh undefine myvm # VM-Definition löschen virsh undefine myvm --remove-all-storage
⚙️
virsh — Konfiguration & Konsole
Konfiguration anzeigen & bearbeiten
virsh dumpxml myvm # XML-Konfig ausgeben virsh edit myvm # XML-Editor öffnen virsh define myvm.xml # VM aus XML definieren
CPU & RAM anpassen
virsh setvcpus myvm 4 --live --config virsh setmem myvm 4G --live --config virsh vcpuinfo myvm virsh vcpupin myvm 0 2 # vCPU 0 → pCPU 2 pinnen
Konsole & VNC
virsh console myvm # serielle Konsole (Ctrl+] zum Verlassen) virsh vncdisplay myvm # VNC-Port anzeigen virsh send-key myvm KEY_CTRL KEY_ALT KEY_DELETE
Netzwerk-Interfaces der VM
virsh domiflist myvm # Netzwerk-Interfaces virsh domifstat myvm vnet0 # Interface-Statistiken virsh domifaddr myvm # IP-Adressen (QEMU-Agent)
💡 virsh --connect qemu+ssh://user@host/system list — remote libvirt
📸
virsh — Snapshots
Snapshots verwalten
virsh snapshot-create-as myvm snap1 \ --description "vor Update" --disk-only virsh snapshot-list myvm virsh snapshot-info myvm snap1 virsh snapshot-revert myvm snap1 virsh snapshot-delete myvm snap1
Checkpoint (für inkrementelles Backup)
virsh checkpoint-create-as myvm chk1 virsh checkpoint-list myvm virsh backup-begin myvm --checkpoint chk1
🌐
virsh — Netzwerk
Netzwerke verwalten
virsh net-list --all virsh net-info default virsh net-start default virsh net-autostart default virsh net-dumpxml default virsh net-destroy default virsh net-undefine default
Netzwerk definieren (XML)
# /tmp/mynet.xml <network> <name>mynet</name> <bridge name="virbr1"/> <ip address="192.168.100.1" prefix="24"> <dhcp><range start="192.168.100.10" end="192.168.100.254"/></dhcp> </ip> </network> virsh net-define /tmp/mynet.xml
💾
virsh — Storage Pool & Volumes
Storage Pools
virsh pool-list --all virsh pool-info default virsh pool-start default virsh pool-refresh default virsh pool-define-as mypool dir --target /data/vms virsh pool-build mypool virsh pool-autostart mypool
Volumes verwalten
virsh vol-list default virsh vol-info default myvm.qcow2 virsh vol-create-as default myvm.qcow2 20G \ --format qcow2 virsh vol-delete default myvm.qcow2 virsh vol-resize /var/lib/libvirt/images/myvm.qcow2 30G
Disk anhängen / trennen
virsh attach-disk myvm \ /var/lib/libvirt/images/data.qcow2 vdb \ --persistent --driver qemu --subdriver qcow2 virsh detach-disk myvm vdb --persistent
🛠
virt-install — VM erstellen
VM aus ISO installieren
virt-install \ --name myvm \ --memory 2048 \ --vcpus 2 \ --disk size=20,format=qcow2 \ --cdrom /iso/debian-12.iso \ --network bridge=virbr0 \ --os-variant debian12 \ --graphics vnc
VM aus Cloud-Image (OpenStack-Stil)
virt-install \ --name cloud-vm \ --memory 2048 \ --vcpus 2 \ --disk path=/var/lib/libvirt/images/ubuntu.qcow2 \ --cloud-init user-data=user-data.yaml \ --import \ --os-variant ubuntu22.04 \ --noautoconsole
OS-Varianten auflisten
osinfo-query os | grep ubuntu osinfo-query os --fields=short-id,name | grep debian
💡 --graphics none --console pty für reine CLI-VMs ohne VNC
🖼
qemu-img — Disk-Images
Image erstellen & Info
qemu-img create -f qcow2 disk.qcow2 20G qemu-img create -f qcow2 -b base.qcow2 -F qcow2 \ overlay.qcow2 # Copy-on-Write Overlay qemu-img info disk.qcow2
Konvertieren & Komprimieren
qemu-img convert -f vmdk -O qcow2 vm.vmdk vm.qcow2 qemu-img convert -O qcow2 -c raw.img compressed.qcow2 qemu-img convert -O raw disk.qcow2 disk.raw
Größe ändern & prüfen
qemu-img resize disk.qcow2 +10G qemu-img resize disk.qcow2 30G # absolute Größe qemu-img check disk.qcow2 qemu-img snapshot -l disk.qcow2 # interne Snapshots
💡 Nach qemu-img resize: Partition/Dateisystem im Gast ebenfalls vergrößern (growpart, resize2fs)
📋
virt-clone & Migration
VM klonen
virt-clone \ --original myvm \ --name myvm-clone \ --auto-clone virt-clone \ --original myvm \ --name myvm-clone \ --file /var/lib/libvirt/images/clone.qcow2
VM migrieren (live)
virsh migrate --live myvm \ qemu+ssh://target-host/system virsh migrate --live --persistent \ --undefinesource myvm \ qemu+ssh://target/system
virt-sysprep (Image entpersonalisieren)
virt-sysprep -d myvm virt-sysprep -a disk.qcow2 \ --hostname newhost --root-password password:secret
