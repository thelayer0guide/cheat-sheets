rabbitmqctl — Status & Verwaltung
Status & Übersicht
rabbitmqctl status rabbitmqctl environment rabbitmqctl node_health_check rabbitmqctl list_nodes rabbitmqctl cluster_status
Queues, Exchanges, Connections
rabbitmqctl list_queues -p / # in vHost / rabbitmqctl list_queues name messages consumers rabbitmqctl list_exchanges -p openstack rabbitmqctl list_bindings rabbitmqctl list_connections rabbitmqctl list_channels rabbitmqctl list_consumers
Queues bereinigen
rabbitmqctl purge_queue nova -p openstack rabbitmqctl delete_queue nova.dead -p openstack

vHosts & Benutzer
vHosts anlegen
rabbitmqctl add_vhost openstack rabbitmqctl list_vhosts rabbitmqctl delete_vhost openstack
Benutzer verwalten
rabbitmqctl add_user openstack SECRET rabbitmqctl change_password openstack NEWPASS rabbitmqctl delete_user openstack rabbitmqctl list_users # Tags: administrator, monitoring, management rabbitmqctl set_user_tags openstack administrator
Berechtigungen setzen
# set_permissions -p VHOST USER CONF WRITE READ rabbitmqctl set_permissions -p openstack openstack ".*" ".*" ".*" rabbitmqctl list_permissions -p openstack rabbitmqctl clear_permissions -p openstack openstack

Clustering

rabbitmqctl forget_cluster_node rabbit@node3 rabbitmqctl change_cluster_node_type disc rabbitmqctl change_cluster_node_type ram

Exchange-Typen & Routing
Exchange-Typen
direct # routing key == binding key (default) fanout # broadcast an alle gebundenen Queues topic # wildcard routing: nova.*.create, *.*.# headers # routing nach AMQP-Header-Attributen
Topic-Routing Wildcards
* # ersetzt genau ein Wort (nova.*.create) # # ersetzt null oder mehr Wörter (nova.#) # Beispiel: nova.compute.create # Passt auf: nova.*.*, nova.#, #.create
Queue-Eigenschaften
durable # überlebt Broker-Neustart exclusive # nur eine Verbindung auto-delete# löscht sich wenn leer x-message-ttl # Nachrichtenablauf in ms x-dead-letter-exchange # DLX für failed msgs

OpenStack — RabbitMQ-Konfiguration
Transport URL (nova.conf, neutron.conf etc.)
# Single Node transport_url = rabbit://openstack:SECRET@10.0.0.1:5672/openstack # HA Cluster (3 Nodes) transport_url = rabbit://openstack:SECRET@10.0.0.1:5672,openstack:SECRET@10.0.0.2:5672,openstack:SECRET@10.0.0.3:5672/openstack
Weitere oslo.messaging Optionen
[oslo_messaging_rabbit] ssl = false rabbit_retry_interval = 1 rabbit_retry_backoff = 2 rabbit_max_retries = 0 # unbegrenzt heartbeat_timeout_threshold = 60 heartbeat_rate = 2
OpenStack vHost & User anlegen
rabbitmqctl add_vhost openstack rabbitmqctl add_user openstack RABBIT_PASS rabbitmqctl set_permissions -p openstack openstack ".*" ".*" ".*"
