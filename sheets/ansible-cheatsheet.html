Ad-hoc Befehle
Ping / Verbindung testen
ansible all -m ping ansible webservers -m ping ansible all -m ping -i inventory.ini
Befehl ausführen
ansible all -m command -a "uptime" ansible all -m shell -a "df -h | grep /dev" ansible all -m raw -a "uname -r"
Paket installieren / Service
ansible all -m apt \ -a "name=nginx state=present" -b ansible all -m service \ -a "name=nginx state=started" -b
Datei kopieren / Facts sammeln
ansible all -m copy \ -a "src=file.conf dest=/etc/file.conf" ansible all -m setup ansible all -m setup -a "filter=ansible_os*"
Wichtige Flags
-i hosts Inventory-Datei -u user Remote-Benutzer -b become (sudo) --become-user root Als User -k SSH-Passwort abfragen -K sudo-Passwort abfragen -f 10 Forks (Parallelität) -v / -vvv Verbose

ansible-playbook — Ausführen
Grundlegende Ausführung
ansible-playbook playbook.yml ansible-playbook playbook.yml -i hosts ansible-playbook playbook.yml -v
Ausführung einschränken
ansible-playbook site.yml \ --limit webservers ansible-playbook site.yml \ --limit host1,host2 ansible-playbook site.yml \ --tags deploy,config ansible-playbook site.yml \ --skip-tags debug
Variablen übergeben
ansible-playbook site.yml \ -e "env=prod version=1.2" ansible-playbook site.yml \ -e @vars/extra.yml
Testen & Prüfen
ansible-playbook site.yml --check dry-run ansible-playbook site.yml --diff Änderungen ansible-playbook site.yml --syntax-check ansible-playbook site.yml --list-tasks ansible-playbook site.yml --list-hosts ansible-playbook site.yml --step interaktiv
Ab bestimmtem Task starten
ansible-playbook site.yml \ --start-at-task "Install nginx"
💡 --check --diff kombinieren für sicheres Testen vor dem Ausrollen

Inventory
INI-Format (hosts / inventory.ini)
# Einzelne Hosts web1.example.com web2.example.com ansible_port=2222 # Gruppen [webservers] web1.example.com web2.example.com [dbservers] db1.example.com # Gruppe von Gruppen [production:children] webservers dbservers # Gruppen-Variablen [webservers:vars] ansible_user=deploy http_port=80
YAML-Format (inventory.yml)
all: children: webservers: hosts: web1.example.com: ansible_user: deploy web2.example.com: dbservers: hosts: db1.example.com:
Inventory prüfen
ansible-inventory --list ansible-inventory --graph ansible-inventory --host web1
Verbindungs-Variablen
ansible_host IP / Hostname ansible_port SSH-Port (default 22) ansible_user SSH-Benutzer ansible_password SSH-Passwort ansible_ssh_private_key_file ansible_become true/false ansible_become_user z.B. root ansible_python_interpreter /usr/bin/python3

Playbook — Struktur
Vollständiges Playbook-Beispiel
--- - name: Webserver einrichten hosts: webservers become: true vars: http_port: 80 app_name: myapp pre_tasks: - name: System aktualisieren apt: update_cache: true tasks: - name: Nginx installieren apt: name: nginx state: present notify: Restart nginx - name: Config deployen template: src: nginx.conf.j2 dest: /etc/nginx/nginx.conf handlers: - name: Restart nginx service: name: nginx state: restarted post_tasks: - name: Smoke test uri: url: http://localhost status_code: 200
💡 Handler laufen immer am Ende des Plays — egal wie oft notifiziert

Variablen & Facts
Variablen definieren
# Im Playbook vars: app_port: 8080 app_name: myapp # Als Datei einbinden vars_files: - vars/main.yml - vars/secrets.yml # Interaktiv abfragen vars_prompt: - name: username prompt: Benutzername?
Variablen verwenden (Jinja2)
# Einfache Verwendung dest: "/etc/{{ app_name }}.conf" # Filter anwenden {{ app_name | upper }} {{ liste | join(', ') }} {{ var | default('fallback') }} {{ pfad | basename }} {{ zahl | int }} # Bedingung {{ 'prod' if env == 'production' else 'dev' }}
Ansible Facts
{{ ansible_hostname }} Hostname {{ ansible_os_family }} Debian / RedHat {{ ansible_distribution }} Ubuntu / CentOS {{ ansible_default_ipv4.address }} {{ ansible_memtotal_mb }} RAM gesamt {{ ansible_processor_vcpus }} CPU-Kerne {{ ansible_env.HOME }} Env-Variablen
Variable Priorität (niedrig → hoch)
role defaults → inventory → playbook vars → vars_files → set_fact → -e (extra vars)
Variable registrieren
- name: Ausgabe speichern command: whoami register: cmd_output - debug: msg: "{{ cmd_output.stdout }}"
💡 group_vars/all.yml und host_vars/hostname.yml für strukturierte Vars

Tasks — Steuerung & Schleifen
Bedingungen (when)
- name: Nur auf Debian apt: name: nginx when: ansible_os_family == "Debian" - name: Mehrere Bedingungen command: echo ok when: - env == "production" - ansible_memtotal_mb > 1024
Schleifen (loop)
- name: Pakete installieren apt: name: "{{ item }}" state: present loop: - nginx - curl - git - name: Dict-Schleife user: name: "{{ item.name }}" state: "{{ item.state }}" loop: - { name: alice, state: present } - { name: bob, state: absent }
Fehlerbehandlung
- name: Fehler ignorieren command: /bin/false ignore_errors: true - name: Eigene Fehlerbedingung command: check_status.sh register: result failed_when: result.rc != 0 - name: Immer als changed markieren command: deploy.sh changed_when: false
Blöcke & Rescue
block: - name: Task 1 command: riskytask.sh - name: Task 2 command: nexttask.sh rescue: - name: Fehler-Handler debug: msg: "Etwas ging schief" always: - name: Aufräumen command: cleanup.sh
💡 loop_control: label: "{{ item.name }}" für lesbarere Ausgabe

Dateien & Verzeichnisse

Roles — Struktur & Verwendung
Verzeichnisstruktur einer Role
roles/ myrole/ tasks/ main.yml Task-Einstieg handlers/ main.yml Handler templates/ Jinja2-Vorlagen (.j2) files/ Statische Dateien vars/ main.yml Unveränderliche Variablen defaults/ main.yml Überschreibbare Defaults meta/ main.yml Abhängigkeiten tests/ Molecule-Tests
Role erstellen & verwenden
ansible-galaxy init myrole # Im Playbook einbinden roles: - myrole - role: geerlingguy.nginx vars: nginx_port: 8080 # Als Task (ansible 2.4+) tasks: - include_role: name: myrole
Abhängigkeiten (meta/main.yml)
dependencies: - role: common - role: nginx vars: nginx_port: 80

Ansible Vault — Secrets
Datei verschlüsseln / entschlüsseln
ansible-vault encrypt secrets.yml ansible-vault decrypt secrets.yml ansible-vault view secrets.yml ansible-vault edit secrets.yml ansible-vault rekey secrets.yml
Einzelnen Wert verschlüsseln
ansible-vault encrypt_string \ 'supersecret' --name db_password # Ausgabe in vars/main.yml einfügen: db_password: !vault | $ANSIBLE_VAULT;1.1;AES256 ...
Mit Vault-Passwort ausführen
ansible-playbook site.yml \ --ask-vault-pass ansible-playbook site.yml \ --vault-password-file .vault_pass # .vault_pass in .gitignore eintragen!
Mehrere Vault-IDs
ansible-vault encrypt \ --vault-id prod@prompt prod.yml ansible-playbook site.yml \ --vault-id prod@.prod_pass \ --vault-id dev@.dev_pass
💡 Vault-Passwortdatei: chmod 600 .vault_pass und in .gitignore eintragen

Ansible Galaxy & Collections
Roles installieren

roles: - name: geerlingguy.nginx version: "3.2.0" collections: - name: community.general - name: ansible.posix version: ">=1.3"
Collections verwalten
ansible-galaxy collection install \ community.general ansible-galaxy collection install \ -r requirements.yml ansible-galaxy collection list

ansible.cfg — Konfiguration
Wichtige Einstellungen
[defaults] inventory = ./inventory remote_user = deploy private_key_file = ~/.ssh/id_rsa host_key_checking = False retry_files_enabled= False forks = 20 gathering = smart roles_path = ./roles:~/.ansible/roles stdout_callback = yaml [privilege_escalation] become = True become_method = sudo become_user = root [ssh_connection] pipelining = True control_path = /tmp/ansible-ssh-%%h-%%p-%%r
Konfiguration prüfen
ansible-config list ansible-config dump --only-changed ansible-config view

Nützliche Muster & Tricks
Task nur einmal ausführen (run_once)
- name: DB Migration command: python manage.py migrate run_once: true delegate_to: db01.example.com
Task lokal ausführen
- name: Lokal ausführen command: ./local_script.sh delegate_to: localhost connection: local
Rolling Updates (serial)
- hosts: webservers serial: 2 2 Hosts gleichzeitig serial: "30%" 30% gleichzeitig max_fail_percentage: 0
Datei aus Template mit lookup
vars: ssh_key: "{{ lookup('file','~/.ssh/id_rsa.pub') }}" env_val: "{{ lookup('env','HOME') }}" secret: "{{ lookup('password','/tmp/pass length=20') }}"
Inventory dynamisch testen
ansible all --list-hosts ansible webservers --list-hosts ansible 'web*' --list-hosts ansible '!dbservers' --list-hosts ansible 'web1:web2' --list-hosts
💡 ANSIBLE_STDOUT_CALLBACK=yaml für lesbare Ausgabe ohne ansible.cfg
