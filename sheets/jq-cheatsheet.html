jq — Grundlagen
Basis-Zugriff
# Identity (pretty-print): jq '.' file.json # Feld-Zugriff: jq '.name' file.json jq '.user.email' file.json jq '.items[0]' file.json jq '.items[1:3]' file.json # Slice jq '.items[-1]' file.json # Letztes Element
Arrays iterieren
# Array-Iterator: jq '.[]' file.json # Feld aus Array: jq '.[] | .name' file.json # Kompakt (-c) + Raw (-r für Strings): jq -r '.[] | .name' file.json jq -c '.items[]' file.json
Objekte konstruieren
# Neues Objekt bauen: jq '.[] | {name: .name, age: .age}' file.json # Array bauen: jq '[.[] | .name]' file.json # String-Interpolation: jq -r '.[] | "\(.name) - \(.age)"' file.json
🔍
jq — Filtern & Suchen
select / map / if
# Filtern: jq '.[] | select(.age > 30)' file.json jq '.[] | select(.name == "Alice")' file.json jq '.[] | select(.name | test("^Al"))' file.json # map (auf Arrays): jq 'map(select(.active == true))' file.json jq 'map(.name)' file.json
Nützliche Funktionen
jq 'length' file.json # Länge jq 'keys' file.json # Objekt-Keys jq 'values' file.json # Werte jq 'has("name")' file.json # Key vorhanden? jq '.name // "default"' # Alternative jq 'unique_by(.name)' # Deduplizierung jq 'group_by(.type)' # Gruppieren jq 'sort_by(.age)' # Sortieren
Aggregation & Reduce
jq '[.[] | .price] | add' # Summe jq '[.[] | .price] | max' # Maximum jq '[.[] | .price] | min' # Minimum jq 'reduce .[] as $x (0; . + $x.price)'
🔄
jq — Mutation & Update
Felder ändern / hinzufügen
# Feld setzen: jq '.name = "Bob"' file.json # Update-Operator |=: jq '.items[] |= .price * 1.1' file.json # Feld löschen: jq 'del(.password)' file.json jq 'del(.[] | .secret)' file.json # Mergen: jq '. + {"newkey": "value"}' file.json
Multi-Input
# Mehrere Dateien: jq -n '[inputs]' a.json b.json # Aus Pipe: curl -s api/endpoint | jq '.data[]' # Env-Variablen in jq: jq -n --arg name "$NAME" '{"name": $name}'
📄
yq — YAML-Verarbeitung
Grundbefehle (yq v4 — mikefarah)
yq '.' file.yaml yq '.metadata.name' pod.yaml yq '.spec.containers[0].image' pod.yaml yq '.spec.containers[].name' pod.yaml # Mehrere Dokumente: yq 'select(.kind == "Deployment")' manifest.yaml
Werte setzen & löschen
# In-place update (-i): yq -i '.metadata.labels.env = "prod"' pod.yaml yq -i 'del(.metadata.annotations)' pod.yaml # Neuen Eintrag hinzufügen: yq -i '.spec.replicas = 3' deploy.yaml
Konvertierung
# YAML → JSON: yq -o=json file.yaml # JSON → YAML: yq -p=json file.json # TOML → YAML: yq -p=toml -o=yaml config.toml # Merge mehrerer YAML-Dateien: yq '. *+ load("override.yaml")' base.yaml

Kubernetes + OpenStack Use Cases
OpenStack API-Antworten verarbeiten
# Server-Liste filtern: openstack server list -f json | \ jq '.[] | select(.Status=="ACTIVE") | .Name' # IPs extrahieren: openstack server show myvm -f json | \ jq '.addresses | to_entries[] | .value[].addr' # Volumes ohne Attachment: openstack volume list -f json | \ jq '.[] | select(.Attachments==[]) | .ID'
Kubernetes Manifests
# Image-Version aller Pods: kubectl get pods -o json | \ jq '.items[] | {name:.metadata.name, img:.spec.containers[0].image}' # Pod-Status filtern: kubectl get pods -o json | \ jq '.items[] | select(.status.phase!="Running") | .metadata.name'
💡 jq -r gibt Raw-Strings aus (ohne Anführungszeichen) — gut für Shell-Pipes
