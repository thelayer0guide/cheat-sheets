Stack auflisten & anzeigen
Übersicht
heat stack-list heat stack-list --show-deleted heat stack-list --show-hidden heat stack-list -f status=FAILED heat stack-show mystack
Template & Outputs
heat template-show mystack heat output-list mystack heat output-show mystack server_ip
🚀
Stack erstellen & aktualisieren
Stack erstellen
heat stack-create \ -f mystack.yaml \ mystack # Mit Parametern: heat stack-create \ -f mystack.yaml \ -P "image=ubuntu-22.04;flavor=m1.small;network=private" \ mystack # Mit Umgebungsdatei: heat stack-create \ -f mystack.yaml \ -e env.yaml \ --poll \ mystack
Stack aktualisieren
heat stack-update \ -f mystack.yaml \ -P "flavor=m1.large" \ mystack # Vorschau (dry-run): heat stack-preview \ -f mystack.yaml \ mystack
Stack löschen
heat stack-delete mystack heat stack-delete --yes mystack
📊
Events & Resources
Events verfolgen
heat event-list mystack heat event-list -n 3 mystack # nested bis Tiefe 3 heat event-show mystack \ myserver-resource EVENT-UUID
Ressourcen
heat resource-list mystack heat resource-list -n 2 mystack # nested heat resource-show mystack myserver heat resource-signal mystack myserver heat resource-mark-unhealthy \ mystack myserver "hardware failure"
Tipp: heat stack-show mystack | grep stack_status_reason zeigt den Fehlergrund direkt nach einem fehlgeschlagenen Stack-Deployment.
⏸
Suspend, Resume & Abandon
Lifecycle
heat action-suspend mystack # VMs pausieren heat action-resume mystack # VMs fortsetzen heat action-check mystack # Status prüfen heat stack-cancel-update mystack # Update abbrechen
Abandon & Adopt
# Stack entfernen, Ressourcen behalten: heat stack-abandon mystack # Aus bestehenden Ressourcen adoptieren: heat stack-adopt \ -f mystack.yaml \ --adopt-file abandon_data.json \ mystack
✅

Validierung
heat template-validate -f mystack.yaml heat template-validate \ -f mystack.yaml \ -e env.yaml

heat resource-type-list heat resource-type-list \ | grep OS::Nova heat resource-type-show \ OS::Nova::Server

Minimales HOT-Template
heat_template_version: 2021-04-06 description: Simple server stack parameters: image: type: string default: ubuntu-22.04 flavor: type: string default: m1.small network: type: string default: private resources: myserver: type: OS::Nova::Server properties: image: { get_param: image } flavor: { get_param: flavor } networks: - network: { get_param: network } key_name: mykey outputs: server_ip: description: IP-Adresse value: get_attr: [myserver, first_address]

HOT Intrinsic Functions
Gängige Funktionen
# Parameter-Wert: { get_param: image_name } # Resource-Attribut: { get_attr: [myserver, first_address] } # Resource-ID: { get_resource: myserver } # String zusammenführen: { str_replace: template: "Hello $name" params: $name: { get_param: name } } # Bedingungen: conditions: is_prod: { equals: [{get_param: env}, prod] } # Abhängigkeit: depends_on: mynetwork
ResourceGroup: OS::Heat::ResourceGroup erzeugt N identische Ressourcen per count-Parameter — ideal für Auto-Scaling-Gruppen.
