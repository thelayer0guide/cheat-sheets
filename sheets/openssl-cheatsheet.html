OpenSSL 3.x
TLS · CSR · CA
Zertifikate · PKCS12
Prüfen · Debuggen
🔑
Schlüssel generieren
Private Keys
# RSA 4096: openssl genrsa -out server.key 4096 # RSA mit Passwort: openssl genrsa -aes256 -out server.key 4096 # EC (schneller, moderner): openssl ecparam -genkey -name prime256v1 \ -out server.key # Ed25519: openssl genpkey -algorithm ed25519 -out server.key
Key-Informationen
openssl rsa -in server.key -text -noout openssl rsa -in server.key -pubout # Public Key openssl pkey -in server.key -text -noout
📋
CSR (Certificate Signing Request)
CSR erstellen
openssl req -new \ -key server.key \ -out server.csr \ -subj "/C=DE/ST=NRW/L=Berlin/O=MyOrg/CN=example.com" # Mit SAN (Subject Alt Names): openssl req -new -key server.key -out server.csr \ -subj "/CN=example.com" \ -addext "subjectAltName=DNS:example.com,DNS:www.example.com"
CSR prüfen
openssl req -in server.csr -text -noout openssl req -in server.csr -verify
🏛
Self-Signed & CA
Self-Signed Zertifikat
openssl req -x509 -newkey rsa:4096 \ -keyout key.pem -out cert.pem \ -sha256 -days 365 -nodes \ -subj "/CN=localhost" # Mit SAN: openssl req -x509 -newkey rsa:4096 \ -keyout key.pem -out cert.pem \ -days 365 -nodes \ -addext "subjectAltName=DNS:example.com,IP:192.168.1.1"
Root CA erstellen
# Root CA Key + Zertifikat: openssl genrsa -aes256 -out ca.key 4096 openssl req -x509 -new -key ca.key \ -sha256 -days 3650 -out ca.crt \ -subj "/CN=MyRoot CA/O=MyOrg"
Zertifikat mit CA signieren
openssl x509 -req -in server.csr \ -CA ca.crt -CAkey ca.key \ -CAcreateserial \ -out server.crt \ -days 365 -sha256 \ -extfile <(echo "subjectAltName=DNS:example.com")
🔍
Zertifikate prüfen
Zertifikat-Informationen
openssl x509 -in server.crt -text -noout openssl x509 -in server.crt -dates -noout openssl x509 -in server.crt -subject -noout openssl x509 -in server.crt -fingerprint -noout openssl verify -CAfile ca.crt server.crt
Remote TLS prüfen
openssl s_client -connect example.com:443 openssl s_client -connect example.com:443 \ -servername example.com # SNI # Nur Zertifikat anzeigen: openssl s_client -connect host:443 \ </dev/null 2>&1 | openssl x509 -text -noout # Ablaufdatum remote prüfen: openssl s_client -connect host:443 \ </dev/null 2>&1 | openssl x509 -dates -noout
🔄
Format-Konvertierung
PEM ↔ DER ↔ PKCS12
# PEM → DER: openssl x509 -in cert.pem -outform DER -out cert.der # DER → PEM: openssl x509 -in cert.der -inform DER -outform PEM -out cert.pem # PEM → PKCS12 (für Java/Windows): openssl pkcs12 -export \ -in server.crt -inkey server.key \ -out server.p12 -name "myserver" # PKCS12 → PEM: openssl pkcs12 -in server.p12 \ -out combined.pem -nodes
🛡
Verschlüsselung & Hashing
Datei ver-/entschlüsseln
# Verschlüsseln: openssl enc -aes-256-cbc -salt \ -in plain.txt -out encrypted.bin -pbkdf2 # Entschlüsseln: openssl enc -aes-256-cbc -d -pbkdf2 \ -in encrypted.bin -out decrypted.txt
Hashes & Checksums
openssl dgst -sha256 file.txt openssl dgst -sha512 file.txt # HMAC: openssl dgst -sha256 -hmac "secretkey" file.txt # Passwort-Hash: openssl passwd -6 'mypassword' # SHA-512
💡 openssl s_client ist das beste Tool zum Live-Debuggen von TLS-Verbindungen
