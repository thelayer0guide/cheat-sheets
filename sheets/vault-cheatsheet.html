Init, Unseal & Login
Vault initialisieren
vault operator init vault operator init \ -key-shares=5 \ -key-threshold=3
Vault unsealen
vault operator unseal UNSEAL_KEY_1 vault operator unseal UNSEAL_KEY_2 vault operator unseal UNSEAL_KEY_3 vault status
Login
vault login ROOT_TOKEN vault login -method=userpass \ username=admin vault login -method=ldap \ username=myuser export VAULT_ADDR=https://vault:8200 export VAULT_TOKEN=hvs.xxx
Auto-Unseal (Transit)
# In vault.hcl: seal "transit" { address = "https://vault-primary:8200" token = "TRANSIT_TOKEN" disable_renewal = "false" key_name = "autounseal-key" mount_path = "transit/" }
🗝
KV Secrets Engine
KV v2 aktivieren
vault secrets enable -path=secret kv-v2 vault secrets list
Secrets schreiben / lesen
vault kv put secret/myapp \ password=secret123 \ api_key=abc123 vault kv get secret/myapp vault kv get -field=password secret/myapp vault kv get -format=json secret/myapp
Versionen & History
vault kv list secret/ vault kv metadata get secret/myapp vault kv get -version=2 secret/myapp vault kv delete secret/myapp vault kv destroy -versions=1,2 secret/myapp
👤
Auth Methods
AppRole (CI/CD / Automatisierung)
vault auth enable approle vault write auth/approle/role/myapp \ secret_id_ttl=10m \ token_ttl=20m \ policies="myapp-policy" # Credentials abrufen: vault read auth/approle/role/myapp/role-id vault write -f auth/approle/role/myapp/secret-id # Login: vault write auth/approle/login \ role_id=ROLE_ID secret_id=SECRET_ID
Kubernetes Auth
vault auth enable kubernetes vault write auth/kubernetes/config \ kubernetes_host=https://k8s-api:6443 vault write auth/kubernetes/role/myapp \ bound_service_account_names=myapp \ bound_service_account_namespaces=default \ policies=myapp-policy ttl=1h
Userpass
vault auth enable userpass vault write auth/userpass/users/alice \ password=secret \ policies=admin
📜
Policies
Policy schreiben
# myapp-policy.hcl: path "secret/data/myapp/*" { capabilities = ["read", "list"] } path "secret/data/myapp/config" { capabilities = ["create", "update", "read"] } path "pki/issue/myapp" { capabilities = ["create", "update"] }
Policy verwalten
vault policy write myapp-policy myapp-policy.hcl vault policy list vault policy read myapp-policy vault policy delete myapp-policy
🔏
PKI — Zertifikate
PKI Engine einrichten
vault secrets enable -path=pki pki vault secrets tune -max-lease-ttl=87600h pki # Root CA generieren: vault write pki/root/generate/internal \ common_name="my-ca.example.com" \ ttl=87600h
Intermediate CA
vault secrets enable -path=pki_int pki vault write pki_int/intermediate/generate/internal \ common_name="my-int-ca" ttl=43800h vault write pki/root/sign-intermediate \ csr=@int-ca.csr format=pem_bundle
Zertifikat-Rolle & Ausstellung
vault write pki_int/roles/myapp \ allowed_domains="example.com" \ allow_subdomains=true max_ttl=72h # Zertifikat ausstellen: vault write pki_int/issue/myapp \ common_name="api.example.com"
🔄
Dynamic Secrets — Datenbanken
Database Engine (PostgreSQL)
vault secrets enable database vault write database/config/mydb \ plugin_name=postgresql-database-plugin \ connection_url="postgresql://{{username}}:{{password}}@db:5432/mydb" \ allowed_roles="myapp-role" \ username="vault" password="vaultpass" vault write database/roles/myapp-role \ db_name=mydb \ creation_statements="CREATE ROLE \"{{name}}\" WITH LOGIN PASSWORD '{{password}}' VALID UNTIL '{{expiration}}';" \ default_ttl=1h max_ttl=24h # Credentials abrufen: vault read database/creds/myapp-role
🔑
Tokens & Audit
Token verwalten
vault token lookup vault token create \ -policy=myapp-policy \ -ttl=1h vault token renew vault token revoke TOKEN
Audit Logging
vault audit enable file \ file_path=/var/log/vault/audit.log vault audit list vault audit disable file/
Terraform Integration
# In Terraform: provider "vault" { address = "https://vault:8200" } data "vault_kv_secret_v2" "myapp" { mount = "secret" name = "myapp" } output "password" { value = data.vault_kv_secret_v2.myapp.data["password"] sensitive = true }
