📋
GlideRecord — Abfragen
Einfache Abfrage
var gr = new GlideRecord('incident'); gr.addQuery('state', 1); // state = 1 (New) gr.addQuery('category', 'network'); gr.query(); while (gr.next()) { gs.info(gr.getValue('number')); }
get() — Single Record
var gr = new GlideRecord('incident'); if (gr.get('sys_id', 'abc123...')) { gs.info(gr.short_description); } // oder per Feld-Wert-Paar: gr.get('number', 'INC0012345');
Encoded Query & Operatoren
var gr = new GlideRecord('incident'); // Encoded Query (direkt aus Filter kopieren): gr.addEncodedQuery('state=1^priority=1^active=true'); // OR-Bedingung: var qc = gr.addQuery('state', 1); qc.addOrCondition('state', 2); gr.query();
Sortierung & Limit
gr.orderBy('sys_created_on'); gr.orderByDesc('priority'); gr.setLimit(100); gr.chooseWindow(0, 50); // Pagination
✏️
GlideRecord — Schreiben
Insert (Neu erstellen)
var gr = new GlideRecord('incident'); gr.initialize(); gr.setValue('short_description', 'Network down'); gr.setValue('category', 'network'); gr.setValue('priority', 1); var sysId = gr.insert(); gs.info('Created: ' + sysId);
Update
var gr = new GlideRecord('incident'); gr.get('number', 'INC0012345'); gr.setValue('state', 6); // Resolved gr.setValue('close_code', 'Solved'); gr.update(); // Bulk-Update ohne next(): gr.addQuery('state', 1); gr.setValue('assigned_to', gs.getUserID()); gr.updateMultiple();
Delete & Reference Fields
// Löschen (Vorsicht!): gr.deleteRecord(); gr.deleteMultiple(); // Alle gematchten Records // Reference-Feld lesen: gr.getValue('assigned_to'); // sys_id gr.assigned_to.getDisplayValue(); // Name gr.getDisplayValue('assigned_to'); // Name

GlideSystem (gs)
Logging & User
gs.info('Message'); // Info-Log gs.warn('Warning'); gs.error('Error occurred'); gs.debug('Debug: ' + value); gs.getUserID(); // sys_id des aktuellen Users gs.getUserName(); // Login-Name gs.getUser().getFullName(); // Vollständiger Name
Utilities
gs.now(); // Aktuelle Zeit (UTC) gs.nowDateTime(); // DateTime-String gs.tableExists('u_custom'); // Tabelle existiert? gs.hasRole('admin'); // Rolle vorhanden? gs.getProperty('glide.smtp.host'); gs.setProperty('my.prop', 'value');
REST Message / HTTP
var rm = new sn_ws.RESTMessageV2(); rm.setEndpoint('https://api.example.com/v1/data'); rm.setHttpMethod('POST'); rm.setRequestHeader('Content-Type', 'application/json'); rm.setRequestBody(JSON.stringify({key: 'value'})); var resp = rm.execute(); gs.info(resp.getStatusCode() + ': ' + resp.getBody());
🌐
Table API (REST)
Auth & Basis-URL
export SN=https://instance.service-now.com export AUTH="admin:password" # Incidents abrufen: curl -su "$AUTH" \ "$SN/api/now/table/incident" \ -H "Accept: application/json" \ -G --data-urlencode 'sysparm_limit=10' \ --data-urlencode 'sysparm_query=state=1'
Wichtige Query-Parameter
sysparm_query=state=1^priority=2 # Encoded Query sysparm_fields=number,short_description,state sysparm_limit=100 sysparm_offset=0 sysparm_display_value=true # Display-Werte statt IDs sysparm_exclude_reference_link=true
Create / Update / Delete
# Neuen Incident anlegen: curl -su "$AUTH" -X POST \ "$SN/api/now/table/incident" \ -H "Content-Type: application/json" \ -d '{"short_description":"Disk full","category":"hardware"}' # Update per sys_id: curl -su "$AUTH" -X PATCH \ "$SN/api/now/table/incident/SYS_ID" \ -H "Content-Type: application/json" \ -d '{"state":"6","close_code":"Solved"}'
OAuth: Für Produktions-Integrationen OAuth 2.0 (Credentials Flow) statt Basic Auth nutzen — unter System OAuth → Application Registry konfigurieren.
