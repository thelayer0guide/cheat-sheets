LVM-Architektur
Konzept PV → VG → LV
# Schichten: PV (Physical Volume) = Disk/Partition VG (Volume Group) = Pool aus PVs LV (Logical Volume) = Virtuelles Block-Gerät # Überblick: pvs / pvscan / pvdisplay vgs / vgscan / vgdisplay lvs / lvscan / lvdisplay
🔧
Physical Volumes (PV)
PV erstellen & verwalten
pvcreate /dev/sdb /dev/sdc pvcreate /dev/sdb1 # auf Partition pvdisplay /dev/sdb pvremove /dev/sdb pvmove /dev/sdb # Daten verschieben pvmove /dev/sdb /dev/sdd # auf Ziel-PV
📦
Volume Groups (VG)
VG erstellen & erweitern
vgcreate vg_data /dev/sdb /dev/sdc vgextend vg_data /dev/sdd # PV hinzufügen vgreduce vg_data /dev/sdb # PV entfernen vgremove vg_data
VG exportieren / importieren
vgexport vg_data vgimport vg_data vgrename vg_data vg_newname
📊
Logical Volumes (LV)
LV erstellen
# Mit Größenangabe: lvcreate -L 50G -n lv_data vg_data lvcreate -l 100%FREE -n lv_data vg_data lvcreate -l 50%VG -n lv_home vg_data
LV vergrößern / verkleinern
# Vergrößern + Filesystem erweitern: lvextend -L +20G /dev/vg_data/lv_data lvextend -l +100%FREE /dev/vg_data/lv_data # Mit Resize-Filesystem: lvextend -r -L +20G /dev/vg_data/lv_data # Manuell ext4: resize2fs /dev/vg_data/lv_data # Verkleinern (erst FS, dann LV!): resize2fs /dev/vg_data/lv_home 30G lvreduce -L 30G /dev/vg_data/lv_home
LV löschen & umbenennen
lvremove /dev/vg_data/lv_data lvrename vg_data lv_old lv_new
📸
Snapshots
Snapshot erstellen & nutzen
# Snapshot von lv_data: lvcreate -s -n lv_snap -L 10G \ /dev/vg_data/lv_data # Snapshot mounten (Read-Only): mount -o ro /dev/vg_data/lv_snap /mnt/snap # Snapshot zurückspielen: umount /dev/vg_data/lv_data lvconvert --merge /dev/vg_data/lv_snap
🔄
Thin Provisioning
Thin Pool & Thin Volume
# Thin Pool erstellen: lvcreate --type thin-pool \ -L 100G -n thin_pool vg_data # Thin Volume (virtuell 200G auf 100G Pool): lvcreate --type thin \ -V 200G --thinpool thin_pool \ -n lv_thin vg_data # Pool-Status: lvdisplay --maps vg_data/thin_pool
🔁
RAID & Mirroring
LVM-eigenes RAID
# RAID1 (Mirror): lvcreate --type raid1 -m 1 \ -L 50G -n lv_mirror vg_data # RAID5 (4+ Disks, -i 3 Stripes + 1 Parität): lvcreate --type raid5 -i 3 \ -L 100G -n lv_raid5 vg_data # RAID-Status: lvs -a -o name,copy_percent,devices
Filesystem erstellen & mounten
mkfs.ext4 /dev/vg_data/lv_data mkfs.xfs /dev/vg_data/lv_data # fstab-Eintrag: /dev/vg_data/lv_data /data ext4 defaults 0 2 # Device-Pfade: /dev/vg_data/lv_data /dev/mapper/vg_data-lv_data
💡 lvextend -r erweitert LV und Filesystem in einem Schritt (ext4, xfs)
