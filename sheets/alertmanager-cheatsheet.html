Grundkonfiguration
alertmanager.yml Grundstruktur
global: smtp_smarthost: 'smtp.example.com:587' smtp_from: 'alerts@example.com' slack_api_url: 'https://hooks.slack.com/...' route: receiver: default group_by: [alertname, cluster] group_wait: 30s group_interval: 5m repeat_interval: 4h routes: - matchers: [severity="critical"] receiver: pagerduty receivers: - name: default email_configs: [...] - name: pagerduty pagerduty_configs: [...]
📨
Receiver-Typen
Slack
slack_configs: - api_url: 'https://hooks.slack.com/...' channel: '#alerts' title: '[{{ .Status | toUpper }}] {{ .GroupLabels.alertname }}' text: '{{ range .Alerts }}{{ .Annotations.summary }}{{ end }}' send_resolved: true
Email
email_configs: - to: 'ops@example.com' from: 'alertmanager@example.com' smarthost: 'smtp.example.com:587' auth_username: 'user' auth_password: 'pass' send_resolved: true
PagerDuty / Webhook
pagerduty_configs: - routing_key: 'your-key' description: '{{ .GroupLabels.alertname }}' webhook_configs: - url: 'http://localhost:5001/webhook' send_resolved: true

Routing
Matcher-Syntax
# Neue Syntax (v0.22+): matchers: - alertname = "HighCPU" - severity =~ "critical|warning" - env != "dev" # Alte Syntax (YAML-Map): match: severity: critical match_re: service: "^(db|api)$"
Verschachtelte Routes
route: receiver: default routes: - matchers: [team="frontend"] receiver: frontend-slack routes: - matchers: [severity="critical"] receiver: frontend-pager continue: false
Timing
group_wait: 30s # warten vor erster Notification group_interval: 5m # Intervall für neue Alerts in Gruppe repeat_interval: 4h # Wiederholungsintervall

Alerts verwalten
amtool alert query amtool alert query severity=critical amtool alert query --active amtool alert query --silenced
🚫
Inhibition
Inhibition Rules
inhibit_rules: - source_matchers: - alertname = "NodeDown" target_matchers: - severity = "warning" equal: [cluster, node] # NodeDown unterdrückt Warning-Alerts # auf demselben Node
Praxis: Kritische Alerts (DB down) inhibieren, damit nachgelagerte App-Alerts nicht spammen.
⚡
Prometheus Alert Rules
Rule-Syntax
groups: - name: example rules: - alert: HighCPU expr: cpu_usage > 0.9 for: 5m labels: severity: critical team: ops annotations: summary: "CPU {{ $value | humanizePercentage }}" runbook: "https://wiki/runbook/high-cpu"
Rule-Dateien prüfen
promtool check rules alerts.yml amtool check-config alertmanager.yml
