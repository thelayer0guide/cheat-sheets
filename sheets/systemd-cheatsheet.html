Service Management
Service steuern
systemctl start app.service systemctl stop app.service systemctl restart app.service systemctl reload app.service systemctl status app.service
Autostart konfigurieren
systemctl enable app.service # Aktivieren systemctl disable app.service # Deaktivieren systemctl is-enabled app.service
Units sperren (Masking)
systemctl mask app.service # Link zu /dev/null systemctl unmask app.service
💡 daemon-reload nach Änderungen an Unit-Files ausführen.
📜
Journal / Logs
Logs anzeigen
journalctl # Alle Logs journalctl -u app.service # Nur für Unit journalctl -f # Follow (live) journalctl -xe # Ende + Details
Zeitliche Filter
journalctl --since "1 hour ago" journalctl --since "2024-03-27" journalctl --until "15:00"
Filter & Output
journalctl -p err # Priorities (err, warning, ...) journalctl -b # Logs vom aktuellen Boot journalctl -o json-pretty # Formatauflösung
📄
Unit Files
Speicherorte
/etc/systemd/system/ # User-definierte Units (höchste Prio) /lib/systemd/system/ # Paket-Units ~/.config/systemd/user/ # User-spezifische Units
Unit Struktur (.service)
[Unit] Description=My App After=network.target [Service] Type=simple ExecStart=/usr/bin/app Restart=always [Install] WantedBy=multi-user.target
Editieren & Reload
systemctl edit app.service # Erstellt Drop-in systemctl edit --full app # Kopiert Original systemctl daemon-reload # Konfig neu laden
💡 Drop-ins überschreiben nur Teile der Original-Unit (über .d/ Ordner).
🛠️
System Analysis
Boot Performance
systemd-analyze # Boot-Zeit systemd-analyze blame # Langsamste Units systemd-analyze critical-chain # Abhängigkeits-Pfad
Abhängigkeiten & Baum
systemctl list-dependencies # Baum-Ansicht systemctl cat app.service # Inhalt anzeigen systemctl show app.service # Low-level Properties
🔧
Tools & State
System Settings
hostnamectl # Hostname & OS-Info timedatectl # Zeit & Zeitzone localectl # Tastatur & Locale
User Sessions & Power
loginctl list-sessions # Aktive User systemctl reboot # Neustart systemctl poweroff # Herunterfahren systemctl suspend # Standby
🔍
Unit Listing
Units auflisten
systemctl list-units # Nur aktive systemctl list-units --all # Alle im Speicher systemctl list-units --failed # Nur fehlgeschlagene
Unit Files (Filesystem)
systemctl list-unit-files systemctl list-unit-files --type=service
Listen filtern
systemctl --type=mount systemctl --state=running
⚠️ list-unit-files zeigt den Installationsstatus (enabled/disabled), list-units den Laufzeitstatus.
👤
User Mode
Management
systemctl --user start app systemctl --user status app journalctl --user -u app
Linger (Dauerhaftigkeit)
# Units laufen lassen, wenn User ausgeloggt ist loginctl enable-linger $USER loginctl show-user $USER
