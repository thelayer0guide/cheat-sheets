CRUD — Insert
Einzelnes Dokument einfügen
db.users.insertOne({ name: "Alice", age: 30, role: "admin" })
Mehrere Dokumente einfügen
db.users.insertMany([ { name: "Bob", age: 25 }, { name: "Carol", age: 28 } ], { ordered: false }) # bei Fehler weitermachen
_id: Wird automatisch als ObjectId vergeben. Eigene _id-Werte sind erlaubt und müssen eindeutig sein.
🔍
CRUD — Find / Lesen
Erstes / alle Dokumente lesen
db.users.findOne({ name: "Alice" }) db.users.find() # alle db.users.find({ age: 30 }) # Filter
Projection, Sort, Limit, Skip
db.users.find( { age: { $gte: 25 } }, { name: 1, age: 1, _id: 0 } # Projection ).sort({ age: -1 }) .limit(10) .skip(20)
Anzahl & Distinct
db.users.countDocuments({ role: "admin" }) db.users.distinct("role")
✏️
CRUD — Update
Einzelnes / mehrere Dokumente aktualisieren
db.users.updateOne( { name: "Alice" }, { $set: { age: 31 } } ) db.users.updateMany( { role: "user" }, { $set: { active: true } } )
Upsert & Replace
db.users.updateOne( { email: "bob@example.com" }, { $set: { name: "Bob" } }, { upsert: true } ) db.users.replaceOne( { name: "Carol" }, { name: "Carol", age: 29, role: "user" } )
findOneAndUpdate (Dokument zurückgeben)
db.orders.findOneAndUpdate( { status: "pending" }, { $set: { status: "processing" } }, { returnDocument: "after", sort: { createdAt: 1 } } )
🗑️
CRUD — Delete
Einzelnes / mehrere Dokumente löschen
db.users.deleteOne({ name: "Bob" }) db.users.deleteMany({ active: false }) db.users.deleteMany({}) # alle löschen
findOneAndDelete
db.sessions.findOneAndDelete( { userId: "u123", expired: true } )
Vorsicht: deleteMany({}) ohne Filter löscht alle Dokumente der Collection. Collection selbst bleibt bestehen.

Query-Operatoren
Vergleich
$eq $ne # gleich / ungleich $gt $gte # größer / größer-gleich $lt $lte # kleiner / kleiner-gleich $in $nin # in / nicht in Array { age: { $gte: 18, $lt: 65 } }
Logisch & Element
{ $and: [{ age: { $gt: 20 } }, { active: true }] } { $or: [{ role: "admin" }, { role: "mod" }] } { $not: { status: "banned" } } { email: { $exists: true } } { score: { $type: "double" } }
Regex & Array
{ name: { $regex: /^alice/i } } { tags: { $all: ["db", "nosql"] } } { scores: { $elemMatch: { $gt: 90 } } } { tags: { $size: 3 } }
🔧
Update-Operatoren
Felder setzen / entfernen / umbenennen
$set: { field: value } $unset: { field: "" } $rename: { oldName: "newName" } $inc: { counter: 1, score: -5 } $mul: { price: 1.1 } $min: { score: 100 } # nur wenn kleiner $max: { score: 0 } # nur wenn größer
Array-Operatoren
$push: { tags: "mongodb" } $pull: { tags: "old-tag" } $addToSet: { tags: "unique" } $pop: { items: 1 } # 1=letztes, -1=erstes $push: { scores: { $each: [85, 90], $slice: -10 } }
Datum & Aktueller Zeitstempel
db.logs.updateOne( { _id: id }, { $currentDate: { updatedAt: true } } )

Aggregation Pipeline
Basis-Pipeline
db.orders.aggregate([ { $match: { status: "shipped" } }, { $group: { _id: "$customerId", total: { $sum: "$amount" } } }, { $sort: { total: -1 } }, { $limit: 5 }, { $skip: 0 } ])
$project, $unwind, $lookup
# $project — Felder auswählen/transformieren { $project: { name: 1, total: 1, _id: 0, year: { $year: "$createdAt" } } } # $unwind — Array-Felder auffalten { $unwind: "$items" } # $lookup — Left Outer Join { $lookup: { from: "products", localField: "pid", foreignField: "_id", as: "product" } }
Nützliche Accumulator-Operatoren
$sum $avg $min $max $count $first $last $push $addToSet $mergeObjects
$match früh platzieren, damit Index-Nutzung möglich ist und spätere Stages weniger Daten verarbeiten.

Indexierung
Index erstellen
db.users.createIndex({ email: 1 }, { unique: true }) db.users.createIndex({ age: -1, name: 1 }) # compound db.articles.createIndex({ body: "text" }) # text db.sessions.createIndex( { createdAt: 1 }, { expireAfterSeconds: 3600 } # TTL )
Indizes anzeigen & löschen
db.users.getIndexes() db.users.dropIndex("email_1") db.users.dropIndexes() # alle außer _id
explain — Abfrageplan analysieren
db.users.find({ age: { $gt: 25 } }) .explain("executionStats") # Werte: "queryPlanner" | "executionStats" | "allPlansExecution" db.users.find({}).hint({ email: 1 }) # Index erzwingen
Compound-Index-Reihenfolge: Equality → Sort → Range. Ein Index auf {a:1, b:1} unterstützt Queries auf a allein, aber nicht b allein.

Backup & Restore
mongodump / mongorestore (BSON)
mongodump --uri "mongodb://localhost:27017" \ --db mydb --out /backup/dump/ mongorestore --uri "mongodb://localhost:27017" \ --db mydb /backup/dump/mydb/ # einzelne Collection mongodump --db mydb --collection users --out /tmp/
mongoexport / mongoimport (JSON/CSV)
mongoexport --db mydb --collection users \ --out users.json --jsonArray mongoimport --db mydb --collection users \ --file users.json --jsonArray # CSV-Export mongoexport --db mydb --collection orders \ --type csv --fields _id,status,amount \ --out orders.csv
mongodump erzeugt BSON-Binärdateien (schnell, vollständig). mongoexport erzeugt JSON/CSV (lesbar, aber kein BSON-Typ-Fidelity).

Replica Set
Status & Konfiguration
rs.status() # Übersicht aller Member rs.conf() # aktuelle Konfiguration db.hello() # welcher Primary? (isMaster entfernt ab 6.0)
Member hinzufügen / entfernen
rs.add("mongo2:27017") rs.add({ host: "mongo3:27017", priority: 0, hidden: true }) rs.remove("mongo2:27017")
Failover & Re-Config
rs.stepDown() # Primary zurücktreten lassen rs.freeze(60) # Election verhindern (60 s) # Priority eines Members ändern let cfg = rs.conf() cfg.members[1].priority = 2 rs.reconfig(cfg)
Read Preference: primary · primaryPreferred · secondary · secondaryPreferred · nearest — konfigurierbar per Connection-String oder Session.

Benutzerverwaltung
Benutzer anlegen
use admin db.createUser({ user: "appuser", pwd: "s3cret!", roles: [ { role: "readWrite", db: "mydb" }, { role: "dbAdmin", db: "mydb" } ] })
Rollen & Passwort ändern
db.grantRolesToUser("appuser", [ { role: "read", db: "reporting" } ]) db.revokeRolesFromUser("appuser", [ { role: "dbAdmin", db: "mydb" } ]) db.changeUserPassword("appuser", "newPass!")
Benutzer anzeigen & löschen
db.getUsers() db.dropUser("appuser")
Built-in Rollen: read · readWrite · dbAdmin · dbOwner · userAdmin · clusterAdmin · root
