Login & Setup
Verbinden
argocd login argocd.example.com argocd login argocd.example.com --insecure argocd login argocd.example.com \ --username admin --password secret argocd logout argocd.example.com
Cluster registrieren
argocd cluster add my-context argocd cluster list argocd cluster get https://k8s.example.com
Admin-Passwort zurücksetzen
kubectl -n argocd get secret argocd-initial-admin-secret \ -o jsonpath="{.data.password}" | base64 -d argocd account update-password
📦
Apps verwalten
App erstellen
argocd app create myapp \ --repo https://github.com/org/repo \ --path k8s/overlays/prod \ --dest-server https://kubernetes.default.svc \ --dest-namespace production \ --revision main \ --sync-policy automated
App Status
argocd app list argocd app get myapp argocd app history myapp
App löschen
argocd app delete myapp argocd app delete myapp --cascade # auch K8s-Ressourcen
🔄
Sync & Diff
Sync ausführen
argocd app sync myapp argocd app sync myapp --force # erzwingen argocd app sync myapp --prune # extra Ressourcen löschen argocd app sync myapp --dry-run # simulieren argocd app sync myapp --revision v1.2 # spezifische Version
Diff & Logs
argocd app diff myapp # Git vs. Cluster argocd app logs myapp -c mycontainer argocd app wait myapp --health argocd app wait myapp --sync
Rollback
argocd app rollback myapp 3 # History-ID argocd app history myapp # IDs anzeigen
⚙️
Application YAML
App-Manifest
apiVersion: argoproj.io/v1alpha1 kind: Application metadata: name: myapp namespace: argocd spec: project: default source: repoURL: https://github.com/org/repo targetRevision: main path: k8s/overlays/prod destination: server: https://kubernetes.default.svc namespace: production syncPolicy: automated: prune: true selfHeal: true syncOptions: - CreateNamespace=true

Projects & RBAC
Projekt verwalten
argocd proj create myproject argocd proj list argocd proj get myproject argocd proj add-source myproject https://github.com/org/repo argocd proj add-destination myproject \ https://kubernetes.default.svc staging
RBAC Policies
# argocd-rbac-cm ConfigMap: policy.default: role:readonly policy.csv: | p, role:dev, applications, sync, myproject/*, allow g, team-dev, role:dev
Tipp: AppProject-Scoped Repositories nutzen, um Zugriff auf spezifische Repos einzuschränken.
