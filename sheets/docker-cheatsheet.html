Container Lifecycle
Starten & Ausführen
# Interaktiv mit TTY docker run -it ubuntu bash # Detached (Hintergrund) docker run -d --name webserver nginx # Port-Mapping: host:container docker run -d -p 8080:80 nginx
Container steuern
docker start webserver docker stop webserver # SIGTERM docker kill webserver # SIGKILL docker restart webserver docker pause webserver docker unpause webserver
Container auflisten
docker ps # laufende docker ps -a # alle inkl. gestoppte docker ps -q # nur IDs docker ps --filter "status=exited"
Entfernen
docker rm webserver docker rm -f webserver # erzwungen # Alle gestoppten Container löschen docker container prune # Nützlich: Auto-Cleanup docker run --rm ubuntu echo "done"
run — wichtige Flags
-d # detached -it # interaktiv + TTY --rm # auto-remove nach Stop --name NAME # Container-Name -p HOST:CONT # Port-Mapping -v VOL:/pfad # Volume-Mount -e VAR=wert # Env-Variable --env-file .env # Env-Datei --network NET # Netzwerk --restart always # Restart-Policy --cpus "1.5" # CPU-Limit --memory 512m # Speicher-Limit -u 1000:1000 # User:Group
💡 --restart unless-stopped startet den Container automatisch neu, aber nicht nach manuellem Stop.

Images
Suchen & Pullen
docker search nginx # Docker Hub docker pull nginx # latest docker pull nginx:1.25-alpine # bestimmter Tag
Images auflisten & entfernen
docker images # alle Images docker images -q # nur IDs docker rmi nginx:1.25 docker image prune # dangling images docker image prune -a # alle ungenutzten
Image inspizieren
docker inspect nginx # JSON-Details docker history nginx # Layer-History docker image ls --digests # mit SHA256
Taggen & Exportieren
docker tag nginx myreg/nginx:1.0 docker save nginx -o nginx.tar docker load -i nginx.tar
💡 alpine-Varianten sind deutlich kleiner. Für Production-Images bevorzugen.

Dockerfile
Wichtige Anweisungen
FROM node:20-alpine # Basis-Image LABEL maintainer="you" WORKDIR /app # Arbeitsverzeichnis COPY package*.json ./ # Dateien kopieren ADD src.tar.gz /app/ # + entpacken RUN npm ci --omit=dev COPY . . ENV NODE_ENV=production # Env-Variable EXPOSE 3000 # Doku-Port USER 1001 # Non-root! ENTRYPOINT ["node"] # Haupt-Prozess CMD ["server.js"] # Default-Args
Multi-Stage Build
# Stage 1: Kompilieren FROM golang:1.22 AS builder WORKDIR /src COPY . . RUN go build -o /app # Stage 2: Minimales Finale-Image FROM gcr.io/distroless/static COPY --from=builder /app /app ENTRYPOINT ["/app"]
.dockerignore
# .dockerignore Beispiel .git node_modules *.log .env dist/ coverage/
💡 ENTRYPOINT = was läuft immer. CMD = überschreibbare Default-Args bei docker run.

Build
Image bauen
docker build -t myapp:1.0 . docker build -t myapp -f Dockerfile.prod . docker build --no-cache -t myapp .
BuildKit & Build-Args
# BuildKit (Standard ab Docker 23) DOCKER_BUILDKIT=1 docker build -t myapp . # Build-Argumente übergeben docker build --build-arg VERSION=1.0 -t myapp . # In Dockerfile: ARG VERSION
Multi-Platform Build
docker buildx build \ --platform linux/amd64,linux/arm64 \ -t myapp:latest --push .
💡 Layer-Cache nutzen: COPY package.json vor RUN npm install — nur bei Änderungen neu bauen.

Debugging & Inspection
Logs
docker logs webserver docker logs -f webserver # follow docker logs --tail 100 webserver docker logs --since 5m webserver
In Container ausführen
# Shell im laufenden Container docker exec -it webserver bash docker exec -it webserver sh # Einzelner Befehl docker exec webserver cat /etc/nginx/nginx.conf
Ressourcen & Details
docker stats # Live-Ressourcen docker stats --no-stream # Einmalig docker top webserver # Prozesse docker inspect webserver # vollständige JSON-Info docker port webserver # Port-Mappings docker diff webserver # Dateisystem-Änderungen
Dateien kopieren
# Host → Container docker cp config.json webserver:/etc/app/ # Container → Host docker cp webserver:/var/log/app.log ./
⚠️ docker exec setzt voraus, dass der Container läuft. Für einen gestoppten Container: docker start zuerst.

Netzwerk
Netzwerke verwalten
docker network ls docker network create mynet docker network create --driver overlay mynet docker network rm mynet docker network inspect mynet
Container verbinden
# Beim Start docker run --network mynet nginx # Nachträglich docker network connect mynet webserver docker network disconnect mynet webserver
Netzwerk-Treiber
bridge # Standard, isoliertes Subnetz host # Teilt Host-Netzwerk-Stack none # Kein Netzwerk overlay # Multi-Host (Swarm) macvlan # Eigene MAC-Adresse
💡 Container im selben Bridge-Netzwerk kommunizieren über Container-Namen als DNS-Hostnamen.

Volumes & Bind Mounts
Volumes verwalten
docker volume create mydata docker volume ls docker volume inspect mydata docker volume rm mydata docker volume prune # ungenutzte löschen
Mount-Typen
# Named Volume (empfohlen für Daten) docker run -v mydata:/var/lib/data nginx # Bind Mount (Host-Pfad → Container) docker run -v /host/pfad:/container/pfad nginx # Read-only docker run -v mydata:/data:ro nginx # tmpfs (RAM, nicht persistent) docker run --tmpfs /tmp nginx
--mount Syntax (expliziter)
docker run \ --mount type=volume,src=mydata,dst=/data \ nginx docker run \ --mount type=bind,src=/host/path,dst=/app,readonly \ nginx
💡 Named Volumes überleben Container-Neustarts und Upgrades. Bind Mounts sind gut für Entwicklung.

Registry & Push
Registry Login & Push
docker login # Docker Hub docker login registry.example.com docker logout # Image taggen und pushen docker tag myapp:1.0 user/myapp:1.0 docker push user/myapp:1.0
Lokale Registry (GHCR / ECR)
# GitHub Container Registry echo $GH_TOKEN | docker login ghcr.io \ -u USERNAME --password-stdin docker tag myapp ghcr.io/USER/myapp:1.0 docker push ghcr.io/USER/myapp:1.0
Lokale Registry starten
# Private Registry auf Port 5000 docker run -d -p 5000:5000 --name registry \ -v registry-data:/var/lib/registry registry:2 docker tag myapp localhost:5000/myapp docker push localhost:5000/myapp

Docker Compose
compose.yml Beispiel
services: web: build: . ports: ["8080:80"] environment: - DB_HOST=db depends_on: [db] networks: [appnet] volumes: [./src:/app/src] db: image: postgres:16 volumes: [pgdata:/var/lib/postgresql/data] env_file: .env volumes: pgdata: networks: appnet:
Compose Befehle
docker compose up # Start (fg) docker compose up -d # detached docker compose up --build # immer neu bauen docker compose down # Stop + rm docker compose down -v # inkl. Volumes docker compose restart web docker compose pull # Images aktualisieren
Compose Status & Logs
docker compose ps docker compose logs -f web docker compose exec web bash docker compose run --rm web npm test
Mehrere Compose-Dateien
# Override Datei für Dev vs Prod docker compose \ -f compose.yml \ -f compose.override.yml up -d
💡 depends_on wartet nur auf Container-Start, nicht auf Readiness. Für Health-Checks: healthcheck definieren.
🧹
System & Cleanup
Aufräumen
# Gestoppte Container docker container prune # Ungenutzte Images (dangling) docker image prune # Alle ungenutzten Images docker image prune -a # Ungenutzte Netzwerke docker network prune # Ungenutzte Volumes docker volume prune
Alles auf einmal
# Container, Images, Netzwerke (nicht Volumes) docker system prune # Inkl. Volumes — VORSICHT! docker system prune -a --volumes # Disk-Nutzung anzeigen docker system df docker system df -v
Alle laufenden stoppen
# Alle Container stoppen docker stop $(docker ps -q) # Alle Container entfernen docker rm $(docker ps -aq)
⚠️ system prune --volumes löscht persistente Daten unwiderruflich. Kein Backup → kein Mitleid.
