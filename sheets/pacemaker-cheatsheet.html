OpenStack HA
🩺
Cluster Status
Überblick
pcs status pcs status --full crm_mon -1 # Einmalige Ausgabe crm_mon -r # Mit inaktiven Ressourcen
Cluster & Node Status
pcs cluster status pcs node status corosync-cfgtool -s corosync-quorumtool -l
Ressourcen-Status
pcs resource status pcs resource show res-vip crm_resource --list crm_resource --locate --resource res-vip
🔧
Ressourcen verwalten
Virtual IP erstellen
pcs resource create res-vip \ ocf:heartbeat:IPaddr2 \ ip=192.168.1.100 \ cidr_netmask=24 \ op monitor interval=30s
Service-Ressource (systemd)
pcs resource create res-nginx \ systemd:nginx \ op monitor interval=30s # Oder mit LSB: pcs resource create res-haproxy \ lsb:haproxy \ op monitor interval=20s
Ressource steuern
pcs resource enable res-vip pcs resource disable res-vip pcs resource restart res-vip pcs resource cleanup res-vip pcs resource move res-vip node2 pcs resource clear res-vip # Move aufheben
Ressource löschen / ändern
pcs resource delete res-vip pcs resource update res-vip ip=192.168.1.101
Resource Groups
pcs resource group add grp-web res-vip res-nginx pcs resource group list pcs resource group remove grp-web res-nginx
🌐
Node Management
Cluster starten / stoppen
pcs cluster start --all pcs cluster stop --all pcs cluster start node1 pcs cluster enable --all # Autostart
Node in Standby
pcs node standby node1 # Ressourcen weg pcs node unstandby node1 pcs node standby --all # Wartungsmodus
Node hinzufügen / entfernen
pcs cluster node add node3 pcs cluster node remove node3
⛓
Constraints
Location (Bevorzugter Node)
pcs constraint location res-vip \ prefers node1=100 pcs constraint location res-vip \ avoids node2
Order (Startreihenfolge)
pcs constraint order res-vip \ then res-nginx pcs constraint order start res-db \ then start res-app
Colocation (Zusammen auf einem Node)
pcs constraint colocation add \ res-nginx with res-vip # Alle Constraints anzeigen: pcs constraint list --full pcs constraint remove CONSTRAINT_ID
⚡
STONITH / Fencing
STONITH-Geräte auflisten
pcs stonith list pcs stonith describe fence_ipmilan
IPMI-Fencing einrichten
pcs stonith create fence-node1 \ fence_ipmilan \ ipaddr=10.0.0.10 \ login=admin passwd=secret \ pcmk_host_list=node1 \ op monitor interval=60s
STONITH testen & deaktivieren
pcs stonith fence node1 pcs property set stonith-enabled=false # Nur für Test! In Produktion immer aktiviert lassen
⚠️ STONITH niemals dauerhaft deaktivieren — Split-Brain-Gefahr!
⚙️
Corosync & Cluster-Setup
Cluster erstellen
pcs cluster auth node1 node2 node3 pcs cluster setup --name mycluster \ node1 node2 node3 pcs cluster start --all
Cluster-Eigenschaften
pcs property list pcs property set no-quorum-policy=freeze pcs property set default-resource-stickiness=100 pcs property set migration-threshold=3
Quorum-Policy
# 2-Node-Cluster: pcs quorum update two_node=1 corosync-quorumtool -s # Quorum erzwingen (Notfall): corosync-quorumtool -e 1

# Virtual IP für OpenStack API: pcs resource create res-os-vip \ ocf:heartbeat:IPaddr2 \ ip=10.0.0.100 cidr_netmask=24 # HAProxy als Ressource: pcs resource create res-haproxy \ systemd:haproxy \ op monitor interval=30s # Galera (MariaDB Cluster): pcs resource create res-galera \ ocf:heartbeat:galera \ wsrep_cluster_address="gcomm://node1,node2,node3" \ op monitor interval=20s
