vrrp_instance — Minimalkonfiguration
vrrp_instance VI_1 { state MASTER # MASTER | BACKUP interface eth0 virtual_router_id 51 # 1-255, unique! priority 100 # BACKUP: 99 advert_int 1 # Hello-Interval Sek. virtual_ipaddress { 10.0.0.100/24 dev eth0 # VIP } }
🔐
VRRP — Authentication & Tracking
Authentication
vrrp_instance VI_1 { ... authentication { auth_type PASS # PASS | AH auth_pass secret123 # max 8 Zeichen! } }
track_interface — Interface-Monitoring
vrrp_instance VI_1 { ... track_interface { eth0 # priority -= 10 wenn eth0 down eth1 weight -20 } }
track_script — Script-Monitoring
vrrp_script chk_haproxy { script "/usr/bin/killall -0 haproxy" interval 2 # alle 2 Sekunden prüfen weight -20 # priority -= 20 bei Fehler fall 3 # 3x Fehler → Failover rise 2 # 2x OK → wieder MASTER } vrrp_instance VI_1 { ... track_script { chk_haproxy } }
📡
Unicast — Kein Multicast-Netz
Unicast-Konfiguration (kein IGMP nötig)
vrrp_instance VI_1 { state MASTER interface eth0 virtual_router_id 51 priority 100 advert_int 1 unicast_src_ip 10.0.0.1 # eigene IP unicast_peer { 10.0.0.2 # IP des anderen Nodes 10.0.0.3 } virtual_ipaddress { 10.0.0.100/24 } }
💡 Unicast empfohlen für Cloud-Umgebungen (AWS, OpenStack) wo Multicast geblockt ist
📧
Notify-Skripte & State-Callbacks
notify auf Statuswechsel reagieren
vrrp_instance VI_1 { ... notify_master "/etc/keepalived/notify.sh MASTER" notify_backup "/etc/keepalived/notify.sh BACKUP" notify_fault "/etc/keepalived/notify.sh FAULT" notify "/etc/keepalived/notify.sh" }
notify.sh Beispiel
#!/bin/bash TYPE=$1 # INSTANCE | GROUP NAME=$2 # Name der vrrp_instance STATE=$3 # MASTER | BACKUP | FAULT logger "Keepalived transition: $TYPE $NAME to $STATE" case $STATE in MASTER) systemctl start haproxy ;; BACKUP) systemctl stop haproxy ;; esac

HAProxy + Keepalived — HA-Kombi
Komplette HA-Konfiguration (Node 1)
vrrp_script chk_haproxy { script "/bin/bash -c 'kill -0 $(cat /run/haproxy.pid)'" interval 2 weight -20 fall 2 rise 2 } vrrp_instance VI_PUBLIC { state MASTER interface eth0 virtual_router_id 51 priority 101 advert_int 1 unicast_src_ip 10.0.0.1 unicast_peer { 10.0.0.2 } track_script { chk_haproxy } virtual_ipaddress { 10.0.0.100/24 } }
💡 Node 2: state BACKUP, priority 100 — sonst identisch

Betrieb & Debugging
Systemctl & Konfiguration prüfen
systemctl start keepalived systemctl enable keepalived systemctl status keepalived keepalived --config-test -f /etc/keepalived/keepalived.conf
Debugging & Logs
journalctl -u keepalived -f journalctl -u keepalived --since "5 min ago" # VIP prüfen (muss auf MASTER-Node erscheinen) ip addr show eth0 | grep 10.0.0.100 # VRRP-Pakete beobachten tcpdump -i eth0 -n 'proto vrrp' # Debug-Modus starten keepalived --log-detail -l -n
Manuelles Failover testen
# HAProxy stoppen → Script schlägt fehl → BACKUP übernimmt systemctl stop haproxy # oder: keepalived Priorität temporär senken: kill -SIGTERM $(cat /run/keepalived.pid) # auf MASTER
