Chains & Targets
🛡
iptables — Grundstruktur
Tables und Chains
# Tables: filter, nat, mangle, raw, security # filter-Chains: INPUT, OUTPUT, FORWARD # nat-Chains: PREROUTING, OUTPUT, POSTROUTING # Regeln anzeigen: iptables -L -n -v --line-numbers iptables -t nat -L -n
Targets (Aktionen)
ACCEPT — Paket annehmen DROP — Paket still verwerfen REJECT — Paket ablehnen + ICMP senden LOG — Loggen, dann weiter MASQUERADE — NAT für dyn. IP DNAT — Ziel-IP ändern SNAT — Quell-IP ändern
✅
Regeln — Zulassen / Sperren
Eingehende Ports öffnen
# SSH erlauben: iptables -A INPUT -p tcp --dport 22 -j ACCEPT # HTTP + HTTPS: iptables -A INPUT -p tcp -m multiport \ --dports 80,443 -j ACCEPT # Established Traffic: iptables -A INPUT -m state \ --state ESTABLISHED,RELATED -j ACCEPT
IP / Subnetz sperren
# IP blockieren: iptables -A INPUT -s 192.168.1.100 -j DROP # Subnetz erlauben: iptables -A INPUT -s 10.0.0.0/8 -j ACCEPT # Interface: iptables -A INPUT -i lo -j ACCEPT
Default Policy / Drop All
# Alles blockieren (Standard): iptables -P INPUT DROP iptables -P FORWARD DROP iptables -P OUTPUT ACCEPT # Loopback + Established immer erlauben! iptables -A INPUT -i lo -j ACCEPT iptables -A INPUT -m conntrack \ --ctstate ESTABLISHED,RELATED -j ACCEPT
Regel einfügen / löschen
# Regel an Position 1 einfügen: iptables -I INPUT 1 -s 10.0.0.0/8 -j ACCEPT # Regel per Nummer löschen: iptables -D INPUT 3 # Alle Regeln löschen: iptables -F iptables -t nat -F
🔀
NAT & Port-Forwarding
Masquerade / SNAT (Outbound-NAT)
iptables -t nat -A POSTROUTING \ -o eth0 -j MASQUERADE # Mit fixer IP (SNAT): iptables -t nat -A POSTROUTING \ -o eth0 -j SNAT --to-source 1.2.3.4 # IP-Forwarding aktivieren: echo 1 > /proc/sys/net/ipv4/ip_forward
DNAT / Port Forwarding
# Extern-Port 80 → intern 192.168.1.10:8080: iptables -t nat -A PREROUTING \ -p tcp --dport 80 \ -j DNAT --to-destination 192.168.1.10:8080 iptables -A FORWARD -p tcp \ -d 192.168.1.10 --dport 8080 -j ACCEPT
💾
Persistenz
Regeln speichern/wiederherstellen
iptables-save > /etc/iptables/rules.v4 iptables-restore < /etc/iptables/rules.v4 # systemd-Einheit (Debian/Ubuntu): apt install iptables-persistent netfilter-persistent save
🔥
nftables — Modern Firewall
Grundbefehle
nft list ruleset nft list tables nft list table ip filter nft flush ruleset nft -f /etc/nftables.conf
Tabellen & Chains erstellen
nft add table ip filter nft add chain ip filter input \ { type filter hook input priority 0 \; policy drop \; } nft add chain ip filter output \ { type filter hook output priority 0 \; policy accept \; }
Regeln hinzufügen
# SSH erlauben: nft add rule ip filter input \ tcp dport 22 accept # Established: nft add rule ip filter input \ ct state established,related accept # Loopback: nft add rule ip filter input \ iif lo accept
nftables NAT
nft add table ip nat nft add chain ip nat postrouting \ { type nat hook postrouting priority 100 \; } nft add rule ip nat postrouting \ oif eth0 masquerade
nftables Config-Datei (/etc/nftables.conf)
#!/usr/sbin/nft -f flush ruleset table inet filter { chain input { type filter hook input priority 0; policy drop; iif lo accept ct state established,related accept tcp dport { 22, 80, 443 } accept log prefix "DROPPED: " drop } chain forward { type filter hook forward priority 0; policy drop; } chain output { type filter hook output priority 0; policy accept; } }
💡 nftables ist der Nachfolger von iptables (seit Kernel 3.13) — bevorzugt ab RHEL 8/Debian 10
🔍
Rate Limiting & Log
Brute-Force-Schutz (iptables)
iptables -A INPUT -p tcp --dport 22 \ -m recent --name SSH --update \ --seconds 60 --hitcount 5 -j DROP iptables -A INPUT -p tcp --dport 22 \ -m recent --name SSH --set -j ACCEPT
Logging
iptables -A INPUT -j LOG \ --log-prefix "IPTABLES-DROP: " \ --log-level 4 # nftables-Logging: nft add rule ip filter input \ log prefix "DROPPED: " level warn drop
⚠ Immer SSH-Zugang absichern BEVOR Policy auf DROP gesetzt wird!
