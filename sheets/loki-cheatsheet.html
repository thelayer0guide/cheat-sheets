LogQL Grundlagen
Log-Stream Selektor
{app="nginx"} {app="nginx", env="prod"} {namespace="default", pod=~"api-.*"} -- Operator: = != =~ !~
Log Pipeline
{app="nginx"} |= "error" -- enthält != "health" -- enthält nicht |~ "ERR[0-9]+" -- regex match !~ "bot|crawler" -- regex nicht
Parser
{app="myapp"} | json {app="myapp"} | json level, msg {app="nginx"} | logfmt {app="nginx"} | pattern '<ip> - - [<_>] "<method> <path> <_>" <status>' {app="nginx"} | regexp `(?P<ip>\d+\.\d+\.\d+\.\d+)`
📊
Filter & Transformationen
Label Filter (nach Parse)
{app="myapp"} | json | level="error" {app="myapp"} | json | status >= 500 {app="myapp"} | json | duration > 1s
Line Format
{app="myapp"} | json | line_format "{{.level}} - {{.msg}}"
Label Transformationen
{app="myapp"} | json | label_format level=lvl | drop msg
📈
Metric Queries
Rate & Count
rate({app="nginx"}[5m]) count_over_time({app="nginx"}[1h]) bytes_rate({app="nginx"}[5m]) bytes_over_time({app="nginx"}[1h])
Aggregation
-- Error-Rate per App: sum(rate({job="app"} |= "error" [5m])) by (app) -- HTTP Status-Verteilung: sum(count_over_time({app="nginx"} | logfmt | status > 399 [1m])) by (status)
Unwrapped Metrics
-- Extrahierte Zahlen als Metriken: avg_over_time({app="api"} | json | unwrap duration [5m]) max_over_time({app="api"} | json | unwrap resp_size [5m])
Aggregationsfunktionen
sum() avg() min() max() count() topk(k,…) quantile_over_time(0.99, …)

logcli CLI
Grundbefehle
export LOKI_ADDR=http://loki:3100 logcli query '{app="nginx"}' logcli query '{app="nginx"}' --since=1h logcli query '{app="nginx"}' --from=2024-01-01T00:00:00Z logcli query '{app="nginx"}' --limit=100 logcli query '{app="nginx"}' --output=jsonl
Labels anzeigen
logcli labels logcli labels app logcli series '{app="nginx"}'
Instanter Query
logcli instant-query 'sum(rate({app="nginx"}[5m]))'

Promtail Konfiguration
Grundstruktur
server: http_listen_port: 9080 clients: - url: http://loki:3100/loki/api/v1/push scrape_configs: - job_name: system static_configs: - targets: [localhost] labels: job: varlogs __path__: /var/log/*.log
Pipeline Stages
pipeline_stages: - json: expressions: level: level msg: message - labels: level: - timestamp: source: time format: RFC3339
Tipp: In Kubernetes Promtail als DaemonSet deployen und Pod-Labels automatisch als Loki-Labels übernehmen.
