Installation & Setup
Installieren (Linux)
# Offizielles Installationsskript curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash # Über Package Manager snap install helm --classic brew install helm
Version & Info
helm version helm env # Helm-Umgebungsvariablen
Shell-Completion aktivieren
helm completion bash >> ~/.bashrc helm completion zsh >> ~/.zshrc
📡
Repositories
Repo hinzufügen & aktualisieren
helm repo add bitnami https://charts.bitnami.com/bitnami helm repo update # Alle Repos aktualisieren helm repo update bitnami # Nur einen Repo
Repos anzeigen & entfernen
helm repo list # Alle konfigurierten Repos helm repo remove bitnami # Repo entfernen
Charts suchen
helm search repo nginx # In lokalen Repos helm search hub nginx # Auf Artifact Hub helm search repo nginx --versions # Alle Versionen
💡 helm repo list -o json — JSON-Ausgabe für Scripting.
⚓
Install & Upgrade
Chart installieren
helm install my-release bitnami/nginx # Mit Namespace (wird erstellt falls nötig) helm install my-release bitnami/nginx \ --namespace production --create-namespace
Bestimmte Chart-Version
helm install my-release bitnami/nginx \ --version 15.0.2
Installieren mit Values-Überschreibung
helm install my-release bitnami/nginx \ --set replicaCount=3 \ --set service.type=LoadBalancer # Aus einer values.yaml Datei helm install my-release bitnami/nginx \ -f values-prod.yaml
Upgrade eines bestehenden Release
helm upgrade my-release bitnami/nginx # Bei nicht existentem Release installieren helm upgrade --install my-release bitnami/nginx \ --atomic --timeout 5m
Dry-run (kein Deployment)
helm install my-release bitnami/nginx \ --dry-run --debug
💡 --atomic — Rollback automatisch bei fehlgeschlagenem Upgrade.
📋
Releases verwalten
Releases auflisten
helm list # Im Default-Namespace helm list -A # Alle Namespaces helm list -n production # Nur production helm list --failed # Nur fehlerhafte
Release Status & Details
helm status my-release helm status my-release -n production
History eines Release
helm history my-release # Alle Revisionen helm history my-release --max 5
Rollback
helm rollback my-release # Zur vorherigen Revision helm rollback my-release 2 # Zu Revision 2
Release deinstallieren
helm uninstall my-release helm uninstall my-release --keep-history

Values & Konfiguration
Default-Values eines Charts einsehen
helm show values bitnami/nginx helm show values bitnami/nginx > values.yaml
Gesetzte Values eines Release
helm get values my-release # User-defined Values helm get values my-release --all # Inkl. Defaults
Verschachtelte Werte mit --set
# Punkt-Notation für Verschachtelung --set image.tag="1.25" --set ingress.enabled=true # Listen mit geschweiften Klammern --set "hosts={a.example.com,b.example.com}" # Strings erzwingen mit --set-string --set-string serviceAccount.annotations.role-arn="arn:..."
Mehrere Values-Files kombinieren
helm upgrade my-release ./mychart \ -f values.yaml \ -f values-prod-overrides.yaml
💡 Spätere -f Dateien überschreiben frühere. Reihenfolge ist wichtig!
📦
Chart Development
Neues Chart erstellen
helm create mychart # Scaffold-Verzeichnis
Chart-Struktur
mychart/ ├── Chart.yaml # Metadaten ├── values.yaml # Default-Werte ├── charts/ # Abhängige Charts └── templates/ ├── deployment.yaml ├── service.yaml ├── _helpers.tpl # Wiederverwendbare Snippets └── NOTES.txt # Post-Install Ausgabe
Chart validieren & analysieren
helm lint ./mychart # Syntax-Check helm lint ./mychart -f values.yaml
Abhängigkeiten verwalten
# In Chart.yaml definieren, dann: helm dependency update ./mychart helm dependency list ./mychart helm dependency build ./mychart
Chart paketieren
helm package ./mychart # Erzeugt .tgz helm package ./mychart --sign --key "My Key"
📄
Templating
Templates rendern (lokal)
helm template my-release ./mychart helm template my-release bitnami/nginx \ -f values.yaml
Wichtige Built-in Objekte
# In templates/*.yaml verfügbar: {{ .Release.Name }} # Release-Name {{ .Release.Namespace }} # Namespace {{ .Chart.Name }} # Chart-Name {{ .Chart.Version }} # Chart-Version {{ .Values.replicaCount }} # Aus values.yaml
Häufige Template-Funktionen
{{ include "mychart.labels" . }} {{ toYaml .Values.resources | indent 10 }} {{ default "latest" .Values.image.tag | quote }} {{ tpl .Values.configTemplate . }} {{ required "image.tag is required!" .Values.image.tag }}
Bedingte Logik
{{- if .Values.ingress.enabled }} # Ingress YAML hier {{- end }} {{- range .Values.env }} - name: {{ .name }} value: {{ .value | quote }} {{- end }}
💡 Der _helpers.tpl-Prefix-Unterstrich sorgt dafür, dass die Datei nicht als K8s-Manifest gerendert wird.

Debugging & Troubleshooting
Generierte Manifeste einsehen
helm get manifest my-release helm get manifest my-release --revision 2
Helm Notes einsehen
helm get notes my-release
Alle Release-Infos
helm get all my-release
Verbose-Ausgabe
helm install my-release bitnami/nginx \ --debug --dry-run # Dry-run mit Debug
Chart-Info anzeigen
helm show chart bitnami/nginx # Chart.yaml helm show readme bitnami/nginx # README.md helm show all bitnami/nginx # Alles
⚠️ Fehler? helm status + kubectl describe pod kombinieren.

OCI & Registry
Registry-Login
helm registry login registry.example.com \ --username user --password pass helm registry logout registry.example.com
Chart pushen (OCI)
helm push mychart-1.0.0.tgz \ oci://registry.example.com/charts
Chart aus OCI installieren
helm install my-release \ oci://registry.example.com/charts/mychart \ --version 1.0.0
OCI Chart pullen
helm pull oci://registry.example.com/charts/mychart \ --version 1.0.0 --untar
💡 OCI-Registry-Support ist seit Helm 3.8.0 stabil (kein HELM_EXPERIMENTAL_OCI mehr nötig).

Plugins
Plugin verwalten
helm plugin list helm plugin install https://github.com/helm/helm-2to3 helm plugin update 2to3 helm plugin uninstall 2to3
Nützliche Plugins
# helm-diff — Diff vor Upgrades helm plugin install \ https://github.com/databus23/helm-diff helm diff upgrade my-release bitnami/nginx # helm-secrets — Secrets verschlüsseln helm plugin install \ https://github.com/jkroepke/helm-secrets
💡 helm-diff ist unverzichtbar für sichere Upgrades in Production.

Häufige Flags & Tipps
Wichtige globale Flags
-n, --namespace # Ziel-Namespace -o, --output # json | yaml | table --kube-context # Kubeconfig-Kontext --kubeconfig # Alternativer Kubeconfig-Pfad --timeout # Default: 5m0s --wait # Wartet auf Ready-Status --debug # Verbose-Ausgabe
Lokales Chart-Verzeichnis nutzen
# Direkt aus lokalem Verzeichnis installieren helm install my-release ./mychart helm upgrade my-release ./mychart -f values.yaml
Chart downloaden & inspizieren
helm pull bitnami/nginx --untar # Entpackt helm pull bitnami/nginx --version 15.0.2
Release-Daten als JSON/YAML ausgeben
helm list -A -o json helm get values my-release -o yaml
💡 helm upgrade --install ist ideal für CI/CD-Pipelines: install oder upgrade in einem Befehl.
