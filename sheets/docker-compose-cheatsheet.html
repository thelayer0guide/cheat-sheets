CLI Befehle
Starten / Stoppen
docker compose up -d # detached docker compose down # stoppen + entfernen docker compose down -v # inkl. Volumes docker compose stop # nur stoppen docker compose start
Build & Pull
docker compose build docker compose build --no-cache docker compose pull docker compose up -d --build # build + start
Status & Logs
docker compose ps docker compose logs -f docker compose logs -f web # Service-spez. docker compose top
Skalierung & Exec
docker compose up --scale web=3 docker compose exec web bash docker compose run --rm web pytest docker compose restart web
📝
compose.yml Grundstruktur
Minimales Beispiel
services: web: image: nginx:alpine ports: - "8080:80" volumes: - ./html:/usr/share/nginx/html restart: unless-stopped
Mit Build-Context
app: build: context: ./app dockerfile: Dockerfile.prod args: NODE_ENV: production image: myapp:latest
Dateinamen
compose.yml # bevorzugt (v2) docker-compose.yml # legacy, noch gültig docker compose -f custom.yml up
⚙️
Service-Konfiguration
Environment
app: environment: DATABASE_URL: postgres://db/mydb DEBUG: "false" env_file: - .env - .env.local
Ports & Expose
ports: - "127.0.0.1:8080:80" # bind an localhost - "3000" # random Host-Port - "5432:5432/tcp" expose: # nur intern sichtbar - "5432"
Abhängigkeiten
depends_on: db: condition: service_healthy cache: condition: service_started
Ressourcenlimits
deploy: resources: limits: cpus: "0.5" memory: 512M reservations: memory: 128M
🌐
Netzwerke
Custom Networks
services: web: networks: [frontend, backend] api: networks: [backend] networks: frontend: backend: internal: true # kein ext. Zugriff
Externes Netzwerk
networks: proxy: external: true name: nginx-proxy
Service-DNS
# Services sind intern über ihren Namen erreichbar postgres://db:5432/mydb # "db" = service name http://api:3000/health
💾
Volumes
Named Volumes
services: db: volumes: - db-data:/var/lib/postgresql/data - ./init.sql:/docker-entrypoint-initdb.d/init.sql volumes: db-data: # Docker managed
Bind Mounts (kurz & lang)
# Kurz: - ./src:/app/src # Lang: - type: bind source: ./src target: /app/src read_only: true
Externes Volume
volumes: nfs-data: driver: local driver_opts: type: nfs o: addr=10.0.0.1,rw device: ":/exports/data"
🏥
Health Checks
Service Health Check
db: image: postgres:16 healthcheck: test: ["CMD-SHELL", "pg_isready -U postgres"] interval: 10s timeout: 5s retries: 5 start_period: 30s
HTTP Health Check
healthcheck: test: ["CMD", "curl", "-f", "http://localhost/health"] interval: 30s timeout: 10s retries: 3
🔒
Secrets & Configs
Secrets (Datei-basiert)
services: app: secrets: [db_password] secrets: db_password: file: ./secrets/db_password.txt
Configs
services: nginx: configs: - source: nginx_conf target: /etc/nginx/nginx.conf configs: nginx_conf: file: ./nginx.conf
Tipp: Secrets werden in /run/secrets/NAME gemountet. Nie Passwörter in environment-Variablen — secrets verwenden!
🔄
Profile & Overrides
Profile (optionale Services)
debug: image: busybox profiles: [debug] docker compose --profile debug up
Override-Dateien
# compose.yml + compose.override.yml # werden automatisch gemerged docker compose -f compose.yml \ -f compose.prod.yml up
Nützliche Variablen
COMPOSE_PROJECT_NAME=myapp COMPOSE_FILE=compose.yml:compose.prod.yml DOCKER_BUILDKIT=1
