LogQL — Loki-Abfragen
Log-Abfragen
# Stream-Selektor {job="nova-compute"} {job="nova-compute", host="compute01"} # Filter-Operatoren {job="nova"} |= "ERROR" # enthält {job="nova"} != "DEBUG" # enthält nicht {job="nova"} |~ "error|warning" # Regex {job="nova"} !~ "healthcheck" # Parser + Extraktion {job="nginx"} | logfmt | status="500" {job="app"} | json | level="error" {job="app"} | pattern "<ip> - <user>"
Metrische Abfragen (LogQL)
# Fehlerate pro Minute rate({job="nova"} |= "ERROR" [1m]) # Anzahl Zeilen pro Zeitfenster count_over_time({job="nova"}[5m]) # Bytes verarbeitet bytes_rate({job="nova"}[1m]) # Top 5 IPs aus Access Log topk(5, sum by (remote_addr) ( rate({job="nginx"} | logfmt [1m]) ))
💡 {job=~"nova.*"} — Regex-Selektor für alle Nova-Jobs

REST API — Häufige Operationen
API-Grundlagen
# API-Key erstellen (UI: Configuration → API Keys) # Oder Service-Account-Token nutzen GRAFANA="http://admin:pass@grafana:3000" # Alle Datasources curl $GRAFANA/api/datasources # Alle Dashboards suchen curl $GRAFANA/api/search?query=nova # Org-Benutzer auflisten curl $GRAFANA/api/org/users # Healthcheck curl $GRAFANA/api/health
Benutzer & Teams (RBAC)
# Benutzer anlegen curl -X POST -H "Content-Type: application/json" \ $GRAFANA/api/admin/users \ -d '{"name":"Max","email":"m@x.de","login":"max","password":"pass"}' # Rollen: Viewer | Editor | Admin # Ordner-Permissions setzen curl -X POST $GRAFANA/api/folders/UID/permissions \ -d '{"items":[{"role":"Viewer","permission":1}]}'
