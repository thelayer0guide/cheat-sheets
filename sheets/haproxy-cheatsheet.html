Grundstruktur — haproxy.cfg
Minimale Konfigurationsstruktur
global log /dev/log local0 maxconn 4096 user haproxy group haproxy daemon defaults mode http timeout connect 5s timeout client 30s timeout server 30s log global option httplog
Config testen & neu laden
haproxy -f /etc/haproxy/haproxy.cfg -c systemctl reload haproxy haproxy -f haproxy.cfg -sf $(cat /var/run/haproxy.pid)

Frontend
HTTP Frontend
frontend http-in bind *:80 default_backend webservers # Mit ACL-Routing: use_backend api-servers if { path_beg /api/ }
HTTPS Frontend (SSL-Termination)
frontend https-in bind *:443 ssl crt /etc/haproxy/certs/ bind *:443 ssl crt /etc/ssl/cert.pem http-request redirect scheme https unless { ssl_fc } default_backend webservers
TCP Frontend (L4 Load Balancing)
frontend mysql-in bind *:3306 mode tcp default_backend mysql-cluster
🖥
Backend
HTTP Backend mit Health Check
backend webservers balance roundrobin option httpchk GET /health http-check expect status 200 server web1 10.0.0.1:80 check server web2 10.0.0.2:80 check server web3 10.0.0.3:80 check backup
Server-Optionen
server web1 10.0.0.1:80 \ check inter 2s rise 2 fall 3 \ weight 10 maxconn 100 server web2 10.0.0.2:80 \ check ssl verify none
Load-Balancing Algorithmen
balance roundrobin # Standard balance leastconn # Wenigste Verbindungen balance source # IP-Hash (sticky) balance uri # URI-Hash balance first # Erster verfügbarer
Sticky Sessions (Cookie)
backend webservers cookie SERVERID insert indirect nocache server web1 10.0.0.1:80 check cookie web1 server web2 10.0.0.2:80 check cookie web2

ACLs & Conditions
ACL Definitionen
# In frontend/listen: acl is_api path_beg /api/ acl is_static path_end .jpg .png .css .js acl is_post method POST acl is_local src 192.168.0.0/24 acl is_host hdr(host) -i api.example.com
ACL-basiertes Routing
use_backend api-servers if is_api use_backend static-servers if is_static use_backend api-servers if is_host http-request deny if !is_local http-request redirect location https://%[hdr(Host)]%[req.uri]
Header manipulieren
http-request set-header X-Forwarded-Proto https http-request add-header X-Real-IP %[src] http-response del-header Server http-response set-header Strict-Transport-Security \ "max-age=31536000"

SSL/TLS
SSL Konfiguration
bind *:443 ssl crt /etc/haproxy/cert.pem \ alpn h2,http/1.1 \ ssl-min-ver TLSv1.2 ssl-default-bind-ciphers ECDHE+AESGCM:ECDHE+CHACHA20 ssl-default-bind-options no-sslv3 no-tlsv10 no-tlsv11
Let's Encrypt / Multi-Cert
# Alle .pem aus Ordner laden: bind *:443 ssl crt /etc/haproxy/certs/ # Zertifikat per SNI auswählen: bind *:443 ssl crt default.pem \ crt-list /etc/haproxy/crt-list.txt
SSL Passthrough (TCP-Mode)
frontend ssl-passthrough mode tcp tcp-request inspect-delay 5s tcp-request content accept if { req.ssl_hello_type 1 } use_backend ssl-backend if { req.ssl_sni -i secure.example.com }

Stats & Runtime API
Stats-Seite aktivieren
listen stats bind *:8404 stats enable stats uri /stats stats refresh 10s stats auth admin:secret stats show-node stats show-legends
Runtime API (UNIX Socket)
# In global: stats socket /var/run/haproxy.sock mode 600 level admin # Befehle via socat: echo "show stat" | socat stdio /var/run/haproxy.sock echo "show info" | socat stdio /var/run/haproxy.sock
Server dynamisch steuern
# Server deaktivieren: echo "disable server webservers/web1" | \ socat stdio /var/run/haproxy.sock # Server aktivieren: echo "enable server webservers/web1" | \ socat stdio /var/run/haproxy.sock # Server-Gewicht ändern: echo "set weight webservers/web1 50" | \ socat stdio /var/run/haproxy.sock
haproxyctl / hatop
hatop -s /var/run/haproxy.sock # Interaktiv haproxyctl show info haproxyctl show stat

Logging & Debug
Log-Konfiguration
# global: log /dev/log local0 info log /dev/log local1 notice # defaults: option httplog option dontlognull log-format "%ci:%cp [%t] %ft %b/%s %Tq/%Tw/%Tc/%Tr/%Ta %ST %B %tsc %ac/%fc/%bc/%sc/%rc %{+Q}r"
Logs anzeigen
journalctl -u haproxy -f tail -f /var/log/haproxy.log
Nützliche Optionen (defaults)
option forwardfor # X-Forwarded-For option http-server-close # Verbindung trennen option redispatch # Bei Ausfall neuer Server retries 3 errorfile 503 /etc/haproxy/errors/503.http
