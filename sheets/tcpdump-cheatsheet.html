tcpdump — Grundbefehle
Grundlegende Capture
tcpdump -i eth0 # Interface tcpdump -i any # Alle Interfaces tcpdump -n # Keine DNS-Auflösung tcpdump -nn # Keine DNS + Port-Namen tcpdump -v / -vv / -vvv # Verbose tcpdump -c 100 # 100 Pakete erfassen
In Datei schreiben / lesen
tcpdump -i eth0 -w capture.pcap tcpdump -r capture.pcap tcpdump -r capture.pcap -n port 80 # Mit Rotation: tcpdump -i eth0 -w cap_%Y%m%d_%H%M%S.pcap \ -G 3600 -z gzip
🔍
BPF-Filter
Host & Port-Filter
tcpdump -n host 192.168.1.1 tcpdump -n src host 192.168.1.1 tcpdump -n dst host 192.168.1.1 tcpdump -n port 80 tcpdump -n src port 80 tcpdump -n portrange 8000-8080
Protokoll & Netz-Filter
tcpdump tcp tcpdump udp tcpdump icmp tcpdump net 10.0.0.0/8 tcpdump src net 192.168.0.0/16
Logische Verknüpfungen
tcpdump host 1.2.3.4 and port 443 tcpdump port 80 or port 443 tcpdump not port 22 # Komplex: tcpdump '(host 1.2.3.4 and port 80) or (host 5.6.7.8 and port 443)'
Praktische Beispiele
# HTTP-Traffic: tcpdump -i eth0 -A port 80 # DNS-Anfragen: tcpdump -n port 53 # SYN-Pakete (neue Verbindungen): tcpdump 'tcp[tcpflags] & tcp-syn != 0' # ARP-Traffic: tcpdump arp # Payload anzeigen (-A ASCII, -X Hex+ASCII): tcpdump -A -s 0 port 80
🔌
ss — Socket-Statistiken
Socket-Überblick
ss -tlnp # TCP, listen, no-resolve, proc ss -ulnp # UDP listening ss -tnp # Established TCP ss -anp # Alle Sockets ss -s # Zusammenfassung
Filter
ss -tnp dst :443 # Verbindungen zu Port 443 ss -tnp src :80 # Verbindungen von Port 80 ss -tnp dport = :443 ss -tnp sport = :22 ss -tnp state established
Erweiterte Infos
ss -tnpe # Extended: uid, timer ss -tnpm # Memory-Info ss -tnpo # Timer-Info # Prozess + Port finden: ss -tlnp | grep :8080
🌐
ip — Netzwerk-Konfiguration
Interfaces & Adressen
ip addr show ip addr show eth0 ip link show ip link set eth0 up|down ip addr add 192.168.1.10/24 dev eth0 ip addr del 192.168.1.10/24 dev eth0
Routing
ip route show ip route get 8.8.8.8 # Route zu Host ip route add 10.0.0.0/8 via 192.168.1.1 ip route del 10.0.0.0/8 ip route add default via 192.168.1.1
Neighbor / ARP
ip neigh show # ARP-Tabelle ip neigh flush all # ARP-Cache leeren ip neigh show dev eth0
🔎
Nützliche Kombis
Wer horcht auf Port X?
ss -tlnp | grep :8080 lsof -i :8080 fuser 8080/tcp
Traffic live auf Wireshark remote
# Remote-Capture → lokales Wireshark: ssh user@host 'tcpdump -i eth0 -w - -n port 80' \ | wireshark -k -i - # Alternativ mit Pipe: ssh root@server 'tcpdump -w - not port 22' \ | tshark -r -
Offene Verbindungen zählen
ss -tn state established | wc -l ss -tn | awk 'NR>1{print $4}' | \ sort | uniq -c | sort -rn
💡 tcpdump -s 0 erfasst vollständige Pakete (Standard: 262144 Bytes)
