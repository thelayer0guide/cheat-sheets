JQL Grundlagen
Syntax
-- Feld Operator Wert [AND|OR|NOT ...] project = "MYPROJ" status = "In Progress" assignee = currentUser() priority in (High, Critical) labels is EMPTY
Verknüpfung
project = MYPROJ AND status != Done project = A OR project = B NOT status = Closed (status = Open OR status = Reopened) AND assignee = currentUser()
Sortierung & Limit
ORDER BY created DESC ORDER BY priority ASC, updated DESC
📋
Felder & Operatoren
Wichtige Felder
project -- Projekt-Key oder Name issuetype -- Bug, Story, Task, Epic status -- Open, In Progress, Done assignee -- Username oder currentUser() reporter -- Username oder currentUser() priority -- Lowest, Low, Medium, High, Highest labels -- Label-Name oder EMPTY component -- Komponenten-Name fixVersion -- Release-Version sprint -- Sprint-Name oder openSprints() "Epic Link" -- Epic-Key (Classic; in Quotes wegen Leerzeichen) summary -- Titel (nur ~) text -- Volltext (summary + desc + comments)
Operatoren
= != -- gleich / ungleich ~ !~ -- enthält / enthält nicht (Textfelder) < <= > >= -- Vergleich (Datum, Priorität) in not in -- Listenwert (mehrere Werte) is is not -- NULL / EMPTY-Prüfung was was in -- historischer Feldwert was not -- historisch nie in diesem Status changed -- Feld hat sich geändert
📅
Datums-Abfragen
Relative Zeitangaben
created >= "-7d" -- letzte 7 Tage updated >= "-1w" -- letzte Woche duedate <= "2d" -- fällig in 2 Tagen created >= "-30d" AND created <= "-1d"
Funktionen
startOfDay() -- Tagesbeginn endOfDay() -- Tagesende startOfWeek() -- Wochenbeginn startOfMonth() -- Monatsbeginn startOfYear() -- Jahresbeginn updated >= startOfWeek() AND updated <= endOfDay()
Status-Datum
status changed to "Done" after "-7d" status changed from "Open" to "In Progress" status was "In Progress" during ("-5d", "-1d")
🏃
Sprint & Versionen
Sprint-Abfragen
sprint in openSprints() sprint in closedSprints() sprint in futureSprints() sprint = "Sprint 42" sprint is EMPTY -- nicht im Sprint
Versionen
fixVersion in unreleasedVersions() fixVersion in releasedVersions() affectedVersion = "v1.2.0" fixVersion in latestReleasedVersion(PROJ)
Epics & Verlinkung
issue in childIssuesOf("PROJ-42") issue in linkedIssues("PROJ-10") issue in linkedIssues("PROJ-10", "is blocked by") parent = "PROJ-42"
⭐
Nützliche JQL-Patterns
Meine offenen Tickets
assignee = currentUser() AND status not in (Done, Closed, Resolved) ORDER BY priority DESC
Überfällige Tickets
duedate < startOfDay() AND status not in (Done, Closed) ORDER BY duedate ASC
Bugs im aktuellen Sprint
project = MYPROJ AND issuetype = Bug AND sprint in openSprints() AND status != Done ORDER BY priority DESC
Kürzlich aktualisiert
project = MYPROJ AND updated >= "-1d" ORDER BY updated DESC
Tipp: Gespeicherte Filter in Jira (Filters → View all filters) können als Dashboard-Gadget-Quelle oder per REST-ID abgerufen werden.

Jira REST API
Auth & Basis
# Basic Auth mit API-Token: export JIRA=https://your-domain.atlassian.net export AUTH="user@example.com:API_TOKEN" # Alle Projekte: curl -su "$AUTH" "$JIRA/rest/api/3/project"
Issues suchen (JQL via REST)
curl -su "$AUTH" \ "$JIRA/rest/api/3/search" \ -G \ --data-urlencode 'jql=project=MYPROJ AND status="In Progress"' \ --data-urlencode 'fields=summary,status,assignee' \ --data-urlencode 'maxResults=50' | jq '.issues[].fields.summary'
Issue abrufen & anlegen
# Issue abrufen: curl -su "$AUTH" "$JIRA/rest/api/3/issue/PROJ-123" # Issue anlegen: curl -su "$AUTH" -X POST \ "$JIRA/rest/api/3/issue" \ -H "Content-Type: application/json" \ -d '{"fields":{"project":{"key":"PROJ"}, "summary":"Bug found","issuetype":{"name":"Bug"}}}'
Issue-Transitions
# Verfügbare Transitions: curl -su "$AUTH" "$JIRA/rest/api/3/issue/PROJ-1/transitions" # Status-Übergang (31 = Done): curl -su "$AUTH" -X POST \ "$JIRA/rest/api/3/issue/PROJ-1/transitions" \ -H "Content-Type: application/json" \ -d '{"transition":{"id":"31"}}'

atlas CLI
Setup
# Installation (npm): npm install -g @atlassian/atlas atlas login # Oder über curl: curl -L https://dac-static.atlassian.com/... | sh
Jira Issues
atlas issue list --project MYPROJ atlas issue list --jql "assignee = currentUser()" atlas issue view PROJ-123 atlas issue create \ --project MYPROJ \ --summary "New bug found" \ --type Bug atlas issue transition PROJ-123 --status "In Progress"
Confluence
atlas confluence page list --space MYSPACE atlas confluence page view PAGE-ID atlas confluence page create \ --space MYSPACE \ --title "New Page" \ --body page.html
📊
Confluence REST API
Seiten abfragen
# Seiten in einem Space: curl -su "$AUTH" \ "$JIRA/wiki/rest/api/content" \ -G --data-urlencode 'spaceKey=MYSPACE' \ --data-urlencode 'type=page' | jq '.results[].title' # Seite per Titel suchen: curl -su "$AUTH" \ "$JIRA/wiki/rest/api/content?title=MyPage&spaceKey=SPACE"
CQL (Confluence Query Language)
# CQL-Suche: curl -su "$AUTH" \ "$JIRA/wiki/rest/api/search" \ -G --data-urlencode \ 'cql=space=MYSPACE AND type=page AND text~"terraform"' # Nützliche CQL-Felder: space type title text label creator lastModified created ancestor
API-Token: Unter id.atlassian.com → Security → API Tokens erstellen. Niemals Passwort direkt verwenden — API-Tokens sind widerrufbar.
