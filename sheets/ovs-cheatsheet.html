Bridge Management
Überblick
ovs-vsctl show ovs-vsctl list-br
Bridge anlegen / löschen
ovs-vsctl add-br br-int ovs-vsctl del-br br-int
Bridge-Eigenschaften
ovs-vsctl set bridge br-int \ fail_mode=secure ovs-vsctl get bridge br-int fail_mode ovs-vsctl set bridge br-int \ other_config:disable-in-band=true
Controller setzen
ovs-vsctl set-controller br-int \ tcp:127.0.0.1:6633 ovs-vsctl del-controller br-int ovs-vsctl get-controller br-int
OVSDB direkt abfragen
ovs-vsctl list bridge ovsdb-client dump
🔌
Port Management
Ports auflisten
ovs-vsctl list-ports br-int ovs-vsctl list interface ovs-vsctl list port
Port hinzufügen / entfernen
ovs-vsctl add-port br-int eth1 ovs-vsctl del-port br-int eth1
Internes Port (TAP/TUN)
ovs-vsctl add-port br-int tap0 \ -- set interface tap0 type=internal
Port-Details & Statistiken
ovs-vsctl get port eth1 tag ovs-ofctl dump-ports br-int ovs-ofctl dump-ports-desc br-int
Interface-Typ & Optionen
ovs-vsctl set interface eth1 \ type=dpdk options:dpdk-devargs=0000:00:07.0 ovs-vsctl get interface eth1 ofport
🏷
VLAN & Tunneling
Access Port (VLAN-Tag)
ovs-vsctl set port eth1 tag=100 # Tag entfernen: ovs-vsctl remove port eth1 tag 100
Trunk Port
ovs-vsctl set port eth1 \ trunks=100,200,300 # Alle VLANs erlauben: ovs-vsctl set port eth1 trunks=[]
VXLAN-Tunnel
ovs-vsctl add-port br-tun vxlan-remote \ -- set interface vxlan-remote \ type=vxlan \ options:remote_ip=10.0.0.2 \ options:key=1000
GRE-Tunnel
ovs-vsctl add-port br-tun gre-remote \ -- set interface gre-remote \ type=gre \ options:remote_ip=10.0.0.3 \ options:key=100
Geneve-Tunnel (OVN)
ovs-vsctl add-port br-int geneve0 \ -- set interface geneve0 \ type=geneve \ options:remote_ip=flow \ options:key=flow
Tunnel-Verbindungen anzeigen
ovs-vsctl find interface type=vxlan ovs-vsctl find interface type=gre
💡 OpenStack Neutron nutzt standardmäßig VXLAN oder GRE für Overlay-Netzwerke
⚡
OpenFlow — Flows
Flows anzeigen
ovs-ofctl dump-flows br-int ovs-ofctl dump-flows br-int table=0 ovs-ofctl show br-int
Flow hinzufügen
ovs-ofctl add-flow br-int \ "table=0,priority=100,ip,nw_dst=10.0.0.1,actions=output:2" ovs-ofctl add-flow br-int \ "in_port=1,dl_vlan=100,actions=strip_vlan,output:2"
Flow löschen / ändern
ovs-ofctl del-flows br-int table=0 ovs-ofctl del-flows br-int "ip,nw_dst=10.0.0.1" ovs-ofctl mod-flows br-int \ "table=0,ip,actions=drop"
Paket-Tracing (Trace)
ovs-appctl ofproto/trace br-int \ "in_port=1,dl_src=00:11:22:33:44:55,dl_dst=ff:ff:ff:ff:ff:ff" # Mit IP: ovs-appctl ofproto/trace br-int \ "in_port=1,ip,nw_src=192.168.1.1,nw_dst=10.0.0.1"
Group-Tabellen
ovs-ofctl dump-groups br-int ovs-ofctl dump-group-stats br-int
💡 Mit ovs-ofctl -O OpenFlow13 OpenFlow 1.3 nutzen (Neutron-Standard)

OVS-Prozesse prüfen

Coverages & Statistiken

OVN — Open Virtual Network
OVN-Überblick
ovn-nbctl show ovn-sbctl show ovn-nbctl list Logical_Switch
Logische Switches
ovn-nbctl ls-add ls-web ovn-nbctl ls-list ovn-nbctl ls-del ls-web
Logische Ports
ovn-nbctl lsp-add ls-web port-vm1 ovn-nbctl lsp-set-addresses port-vm1 \ "aa:bb:cc:dd:ee:ff 192.168.1.10" ovn-nbctl lsp-set-port-security port-vm1 \ "aa:bb:cc:dd:ee:ff 192.168.1.10" ovn-nbctl lsp-list ls-web
Logische Router
ovn-nbctl lr-add lr-main ovn-nbctl lrp-add lr-main lrp-web \ "aa:bb:cc:00:00:01" "192.168.1.1/24" ovn-nbctl lsp-set-type port-to-router router ovn-nbctl lsp-set-options port-to-router \ router-port=lrp-web
ACLs (Firewall-Regeln)
ovn-nbctl acl-add ls-web \ to-lport 1000 "tcp.dst == 80" allow ovn-nbctl acl-add ls-web \ to-lport 900 "ip4" drop ovn-nbctl acl-list ls-web ovn-nbctl acl-del ls-web
Chassis & Binding
ovn-sbctl list Chassis ovn-sbctl list Port_Binding ovn-sbctl dump-flows ls-web
NAT (Floating IPs in OVN)
ovn-nbctl lr-nat-add lr-main \ dnat_and_snat \ 203.0.113.10 192.168.1.10 ovn-nbctl lr-nat-list lr-main ovn-nbctl lr-nat-del lr-main \ dnat_and_snat 203.0.113.10
💡 OVN ersetzt den klassischen OVS-Agent in modernen OpenStack Neutron-Deployments

OpenStack Neutron Integration
Typische OVS-Bridges (OVS-Agent)
# br-int — Integration Bridge (interne VMs) # br-ex — External Bridge (Provider/Floating) # br-tun — Tunnel Bridge (VXLAN/GRE zwischen Nodes) ovs-vsctl show
Physical Interface zu br-ex hinzufügen
ovs-vsctl add-br br-ex ovs-vsctl add-port br-ex eth0 # In neutron.conf / ml2_conf.ini: # bridge_mappings = physnet1:br-ex
OVS-Neutron-Agent Status
openstack network agent list \ --agent-type Open vSwitch agent neutron agent-list --agent-type ovs
OVN-Backend für Neutron prüfen
# OVN Northbound DB: ovn-nbctl show # Neutron-Port ↔ OVN-Port: ovn-nbctl list Logical_Switch_Port | \ grep NEUTRON_PORT_ID
Paketweg durch Neutron verfolgen
ovs-appctl ofproto/trace br-int \ "in_port=PORT_NUM,ip,nw_src=VM_IP,nw_dst=TARGET" # OVN Trace: ovn-trace ls-web \ "inport==\"port-vm1\", eth.src=aa:bb:cc:dd:ee:ff, ip4.src=192.168.1.10"
Neutron ML2 OVN Config (ml2_conf.ini)
[ml2] mechanism_drivers = ovn type_drivers = geneve,vlan,flat tenant_network_types = geneve [ml2_type_geneve] vni_ranges = 1:65536 [ovn] ovn_nb_connection = tcp:127.0.0.1:6641 ovn_sb_connection = tcp:127.0.0.1:6642

DPDK & Performance
DPDK aktivieren
ovs-vsctl set Open_vSwitch . \ other_config:dpdk-init=true ovs-vsctl set Open_vSwitch . \ other_config:dpdk-socket-mem="1024,1024"
PMD-Threads & CPU-Pinning
ovs-vsctl set Open_vSwitch . \ other_config:pmd-cpu-mask=0xc # CPUs 2+3 ovs-appctl dpif-netdev/pmd-stats-show ovs-appctl dpif-netdev/pmd-rxq-show
Hardware Offload (TC)
ovs-vsctl set Open_vSwitch . \ other_config:hw-offload=true ovs-appctl dpctl/offload-stats-show
💡 DPDK erfordert Hugepages: hugepages=1G im Kernel-Cmdline
